## 3. 系统测试

### 3.1 端到端测试

**测试场景：完整数据采集流程**
```
1. 调用 Tushare API 获取股票列表
2. 获取 K 线数据
3. 解析 RSS 新闻
4. 数据清洗和去重
5. 存储到数据库
6. 验证数据完整性
```

### 3.2 定时任务测试

**测试步骤:**
1. 配置测试用 Cron 表达式
2. 等待定时任务触发
3. 验证数据采集成功
4. 验证数据存储正确

## 4. 性能测试

### 4.1 性能指标

| 指标 | 要求 | 测试方法 |
|------|------|----------|
| API 调用延迟 | < 2 秒/次 | 单次调用计时 |
| 批量采集速度 | > 100 条/分钟 | 批量采集计时 |
| 数据存储延迟 | < 5 分钟 | 采集到存储完成 |

### 4.2 压力测试

- 连续采集 24 小时
- 验证无内存泄漏
- 验证数据完整性
