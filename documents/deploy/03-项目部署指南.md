# 项目部署指南

---

## 1. 部署方式

### 方式一：Docker Compose 一键部署（推荐）

```bash
# 1. 克隆项目
cd /opt
git clone <repository-url> stock-trading
cd stock-trading

# 2. 启动所有服务
docker-compose up -d --build

# 3. 查看服务状态
docker-compose ps

# 4. 查看日志
docker-compose logs -f
```

### 方式二：分阶段部署

```bash
# 1. 先启动基础设施（MySQL、Redis、MongoDB）
docker-compose up -d mysql redis mongo

# 2. 验证基础设施
docker-compose ps

# 3. 启动应用服务
docker-compose up -d backend frontend stock-service
```

## 2. 服务配置

### 2.1 配置文件

项目配置文件位于 `docker-compose.yml`：

```yaml
services:
  mysql:
    environment:
      - MYSQL_ROOT_PASSWORD=Root.123456
      - MYSQL_DATABASE=stock_trading
    ports:
      - "6033:3306"

  redis:
    command: redis-server --requirepass Root.123456 --appendonly yes
    ports:
      - "6379:6379"

  mongo:
    environment:
      - MONGO_INITDB_ROOT_USERNAME=admin
      - MONGO_INITDB_ROOT_PASSWORD=Root.123456
    ports:
      - "27017:27017"

  backend:
    environment:
      - SPRING_PROFILES_ACTIVE=prod
      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/stock_trading...
      - SPRING_DATA_MONGODB_URI=mongodb://admin:Root.123456@mongo:27017...
      - SPRING_REDIS_PASSWORD=Root.123456

  frontend:
    ports:
      - "8000:80"

  stock-service:
    ports:
      - "8001:8001"
```

### 2.2 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| SPRING_PROFILES_ACTIVE | Spring Profile | prod |
| MYSQL_HOST | MySQL地址 | mysql |
| REDIS_HOST | Redis地址 | redis |
| MONGO_HOST | MongoDB地址 | mongo |

## 3. 数据库初始化

### 3.1 导入SQL

```bash
# 进入 MySQL 容器
docker exec -it stock-mysql mysql -u root -pRoot.123456

# 创建数据库
CREATE DATABASE stock_trading CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 导入表结构
docker exec -i stock-mysql mysql -u root -pRoot.123456 stock_trading < stock-backend/src/main/resources/sql/found_trading.sql
```

### 3.2 验证数据库

```bash
# 登录 MySQL
docker exec -it stock-mysql mysql -u root -pRoot.123456

# 查看数据库
SHOW DATABASES;

# 使用数据库
USE stock_trading;

# 查看表
SHOW TABLES;
```

## 5. 日志管理

### 5.1 查看日志

```bash
# 查看所有服务日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend

# 查看最近100行
docker-compose logs --tail=100 backend
```

### 5.2 日志轮转

日志配置已在 daemon.json 中设置：
- 最大文件大小: 100MB
- 最大文件数: 3

### 5.3 清理日志

```bash
# 清理所有容器日志
docker-compose down

# 清理日志文件
sudo rm -rf /var/lib/docker/containers/*/*-json.log

# 清理未使用的镜像
docker image prune -a
```

## 6. 服务管理

### 6.1 启动/停止

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启特定服务
docker-compose restart backend

# 重启所有服务
docker-compose restart
```

### 6.2 扩缩容

```bash
# 扩展后端服务实例
docker-compose up -d --scale backend=2

# 注意：需要配置负载均衡
```

### 6.3 更新服务

```bash
# 拉取最新镜像
docker-compose pull

# 重新构建并启动
docker-compose up -d --build

# 重启特定服务
docker-compose up -d --no-deps backend
```

## 7. 数据备份

### 7.1 备份数据库

```bash
# 备份 MySQL
docker exec stock-mysql mysqldump -u root -pRoot.123456 stock_trading > backup/mysql_$(date +%Y%m%d).sql

# 备份 Redis
docker exec stock-redis redis-cli -a Root.123456 SAVE

# 备份 MongoDB
docker exec stock-mongo mongodump --out=/backup --username admin --password Root.123456 --authenticationDatabase admin
```

### 7.2 恢复数据

```bash
# 恢复 MySQL
docker exec -i stock-mysql mysql -u root -pRoot.123456 stock_trading < backup/mysql_20260219.sql
```

## 8. 常见问题

### 8.1 容器启动失败

```bash
# 查看错误日志
docker-compose logs <service-name>

# 检查配置文件
docker-compose config
```

### 8.2 连接被拒绝

```bash
# 检查端口占用
netstat -tlnp | grep <port>

# 检查防火墙
sudo ufw status
```

### 8.3 内存不足

```bash
# 查看内存使用
docker stats

# 限制内存
# 编辑 docker-compose.yml 添加:
deploy:
  resources:
    limits:
      memory: 2G
```
