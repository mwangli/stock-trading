# 数据采集模块设计

## 模块说明

数据采集模块负责数据采集、存储和分发，是整个系统的基础数据层。

## 包含功能

| 序号 | 功能 | 说明 |
|------|------|------|
| 1 | 股票列表同步 | Tushare Pro API获取A股股票列表 |
| 2 | 历史K线采集 | Tushare Pro获取历史数据 |
| 3 | 每日K线同步 | Tushare Pro每日K线同步(LSTM训练数据) |
| 4 | RSS新闻采集 | ROME抓取财经新闻 |
| 5 | 数据存储 | MySQL + MongoDB |

## 核心类

| 类名 | 说明 |
|------|------|
| TushareClient | Tushare Pro HTTP客户端 |
| RomeClient | ROME RSS客户端 |
| StockCollector | 股票数据采集器 |
| NewsCollector | 新闻采集器 |
| StockRepository | MyBatis数据仓储 |
| PriceRepository | MongoDB价格仓储 |
| NewsRepository | MySQL新闻仓储 |

## 技术栈

- OkHttp (HTTP客户端)
- ROME (RSS抓取)
- MyBatis-Plus (MySQL)
- Spring Data MongoDB

## 一、数据源方案

### 1.1 Tushare Pro 数据源

- **接口**: Tushare Pro API
- **数据内容**: 
  - 股票列表（基本信息）
  - 历史日线数据
- **频控策略**: 120积分档位50次/分钟，需间隔1.2秒
- **更新频率**: 每日（收盘后）

### 1.2 RSS新闻数据源

#### ROME方案优势

- 纯Java实现：Maven引入即用，与Spring无缝集成
- 多源聚合：可同时抓取新浪、东财、证券时报等多个源
- 完全可控：数据存本地，不受第三方API限制
- 自动编码：XmlReader自动处理中文乱码

#### 推荐RSS源

| 源名称 | RSS地址 | 稳定性 |
|--------|---------|--------|
| 新浪财经 | http://feed.sina.com.cn/finance | 较稳定 |
| 东方财富 | http://finance.eastmoney.com/rss/ | 一般 |
| 中国证券报 | http://www.cs.com.cn/rss/ | 稳定 |
| 证券时报 | http://www.stcn.com/rss/ | 稳定 |

## 二、数据采集流水线

### 2.1 股票数据采集流程

```
Tushare Pro API → TushareClient → StockCollector → StockRepository(MySQL)
                                              ↓
                                        PriceRepository(MongoDB)
```

### 2.2 新闻数据采集流程

```
RSS源 → RomeClient → NewsCollector → 清洗去重 → NewsRepository(MySQL)
                                            ↓
                                    实体识别(AI模型模块)
                                            ↓
                                    情感分析(AI模型模块)
```

## 三、数据存储设计

### 3.1 存储策略

| 数据类型 | 存储位置 | 说明 |
|----------|----------|------|
| 股票信息 | MySQL | stock_info表 |
| K线数据 | MongoDB | stock_prices集合 |
| 新闻数据 | MySQL | news表 |

### 3.2 MongoDB集合设计

```javascript
// 股票价格集合
db.createCollection("stock_prices");

// 使用日期和股票代码作为分区键（复合索引）
db.stock_prices.createIndex({"date": -1, "code": 1}, {unique: true});

// 文档结构
{
    "code": "600519",
    "date": "2026-02-27",
    "price1": 1850.00,  // 开盘价
    "price2": 1870.00,  // 最高价
    "price3": 1840.00,  // 最低价
    "price4": 1865.00,  // 收盘价
    "price5": 1000000,  // 成交量
    "amount": 1850000000 // 成交额
}
```

## 六、数据质量保障

### 6.1 ST股票过滤

- 采集时过滤名称包含"ST"、"*ST"的股票
- 定期检查并标记已退市股票

### 6.2 市场分类

| 市场 | 代码前缀 | 判断逻辑 |
|------|----------|----------|
| 上海 | 6开头 | code.startsWith("6") |
| 深圳 | 0/3开头 | code.startsWith("0") \|\| code.startsWith("3") |

### 6.3 数据去重

- 新闻URL唯一性检查
- 标题+发布时间联合去重

### 6.4 新闻数据保留策略

- **保留策略**: 所有历史新闻永久保留，用于FinBERT模型训练和微调
- **语料积累**: 每日增量采集，持续积累训练语料
- **关联处理**: 通过实体识别关联股票代码，生成训练样本

## 七、性能与可用性设计

### 7.1 性能指标

| 指标 | 要求 | 说明 |
|------|------|------|
| 股票列表采集 | < 30秒 | 5000只股票 |
| 历史数据采集 | < 10秒/股票 | 60天数据 |
| RSS新闻抓取 | < 30秒 | 单源 |
| 并发采集 | 10线程 | 批量采集 |

### 7.2 可用性保障

- **SLA**: 每日数据采集成功率 > 99%
- **故障处理**: 采集失败自动重试3次
- **数据保留**: 永久保留历史数据，用于模型训练
