## 2. 概述

### 2.1 模块定位

风控管理模块是股票交易系统的安全保障组件，负责在交易执行前进行风险检查，包括止损检查、熔断检查、仓位检查等，确保交易风险可控。

### 2.2 功能摘要

- 止损规则检查（日内止损、累计止损）
- 仓位规则检查（单票仓位、总仓位）
- 熔断机制（日熔断、月熔断）
- 风控状态管理

### 2.3 上下游依赖

| 上游模块 | 接口 | 数据 |
|----------|------|------|
| 决策引擎 | checkBeforeTrade() | 交易信号 |

| 下游模块 | 接口 | 数据 |
|----------|------|------|
| 交易执行 | executeTrade() | 风控结果 |

### 3.2 功能详情

#### F-001: 止损检查

**功能描述**: 检查持仓是否触发止损条件。

**输入**: 持仓信息

**处理逻辑**:
1. 计算日内亏损比例
2. 计算累计亏损比例
3. 判断是否触发止损

**输出**: 止损检查结果

**业务规则**:
- 日内止损: -3%（单日亏损超过3%）
- 累计止损: -10%（累计亏损超过10%清仓）
- 止盈: +10%

#### F-002: 仓位检查

**功能描述**: 检查买入后仓位是否超限。

**输入**: 交易请求

**处理逻辑**:
1. 计算单票仓位比例
2. 计算总仓位比例
3. 判断是否超限

**输出**: 仓位检查结果

**业务规则**:
- 单票仓位: ≤30%
- 总仓位: ≤80%
- 累计亏损-10%时禁止开仓

#### F-003: 熔断检查

**功能描述**: 检查是否触发熔断条件。

**输入**: 账户盈亏数据

**处理逻辑**:
1. 计算日亏损比例
2. 计算月亏损比例
3. 判断是否触发熔断

**输出**: 熔断检查结果

**业务规则**:
- 日熔断: 日亏损≥5%，停止当日买入
- 月熔断: 月亏损≥10%，停止本周交易

### 6.3 熔断机制

#### 6.3.1 日熔断

```
触发条件: 日亏损 >= 5%
执行动作: 
  1. 停止当日所有买入交易
  2. 允许卖出（止损）
  3. 记录熔断日志
恢复条件: 次日自动恢复
```

#### 6.3.2 月熔断

```
触发条件: 月亏损 >= 10%
执行动作:
  1. 停止本周所有交易
  2. 发送告警通知
恢复条件: 下周一自动恢复
```

#### 6.4.3 核心风控查询SQL示例

**查询账户总仓位**:
```sql
-- 计算当前总仓位比例
SELECT 
    a.total_amount,
    a.available_amount,
    a.used_amount,
    ROUND((a.used_amount / a.total_amount) * 100, 2) as position_ratio
FROM account_info a
WHERE id = 1;
```

**查询单票仓位**:
```sql
-- 计算指定股票的仓位比例
SELECT 
    p.stock_code,
    p.stock_name,
    p.quantity,
    p.market_value,
    a.total_amount,
    ROUND((p.market_value / a.total_amount) * 100, 2) as stock_position_ratio
FROM position p
CROSS JOIN account_info a
WHERE p.stock_code = '600519'
  AND p.status = 'HOLDING';
```

**查询当日盈亏**:
```sql
-- 计算当日已实现盈亏
SELECT 
    SUM(CASE 
        WHEN type = '卖出' THEN amount 
        WHEN type = '买入' THEN -amount 
        ELSE 0 
    END) as today_pnl
FROM order_info
WHERE date = CURDATE()
  AND status = '已成交';
```

**查询累计盈亏**:
```sql
-- 计算累计收益率
SELECT 
    SUM(income) as total_income,
    SUM(buy_amount) as total_buy,
    ROUND((SUM(income) / SUM(buy_amount)) * 100, 2) as total_pnl_percent
FROM trading_record
WHERE sold = '1';
```

**查询当日交易次数**:
```sql
-- 统计当日交易次数(用于日熔断判断)
SELECT 
    COUNT(*) as order_count,
    SUM(CASE WHEN status = '已成交' THEN 1 ELSE 0 END) as filled_count
FROM order_info
WHERE date = CURDATE();
```

**查询持仓盈亏**:
```sql
-- 计算持仓浮动盈亏
SELECT 
    p.stock_code,
    p.avg_cost,
    p.current_price,
    p.quantity,
    p.unrealized_pnl,
    p.unrealized_pnl_ratio
FROM position p
WHERE p.status = 'HOLDING';
```

**查询本月盈亏**:
```sql
-- 计算本月已实现盈亏
SELECT 
    SUM(income) as month_income,
    SUM(buy_amount) as month_buy_amount,
    ROUND((SUM(income) / NULLIF(SUM(buy_amount), 0)) * 100, 2) as month_pnl_percent
FROM trading_record
WHERE sold = '1'
  AND sale_date >= DATE_FORMAT(CURDATE(), '%Y-%m-01');
```

      daily: -0.05        # 日熔断 5%

## 7. 错误处理

### 7.1 错误码定义

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| RISK_001 | 账户状态异常 | 账户不可用 |
| RISK_002 | 交易时间不允许 | 非交易时间 |
| RISK_003 | 股票停牌 | 不可交易 |
| RISK_004 | 仓位超限 | 仓位检查未通过 |
| RISK_005 | 熔断触发 | 熔断中禁止交易 |

### 7.2 异常处理策略

| 异常类型 | 处理策略 |
|----------|----------|

| 熔断触发 | 返回拒绝，发送告警 |
| 仓位超限 | 返回拒绝，建议调整 |
