# 风控策略设计

## 1. 模块概述

风控管理模块负责在交易执行前进行风险检查，包括止损检查、熔断检查、仓位检查等。

## 2. 风控规则

### 2.1 止损规则

| 规则 | 阈值 | 说明 |
|------|------|------|
| 日内止损 | -3% | 单日亏损超过3%则止损 |
| 累计止损 | -10% | 累计亏损超过10%则清仓 |

### 2.2 仓位规则

| 规则 | 阈值 | 说明 |
|------|------|------|
| 单票仓位 | 30% | 单只股票不超过30%仓位 |
| 总仓位 | 80% | 总仓位不超过80% |
| 空仓信号 | -10% | 累计亏损10%时禁止开仓 |

### 2.3 熔断规则

| 规则 | 触发条件 | 说明 |
|------|----------|------|
| 日熔断 | 日亏损>=5% | 停止当日交易 |
| 月熔断 | 月亏损>=15% | 停止本周交易 |

## 3. 风控流程

```
交易信号产生
           │
           ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 基本检查                                           │
│  ┌─────────────────────────────────────────────────────┐  │
│  │ • 账户状态正常?                                      │  │
│  │ • 交易时间允许?                                      │  │
│  │ • 股票未停牌?                                        │  │
│  └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
           │
           ▼ (通过)
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 买入风控 (仅买入信号)                              │
│  ┌─────────────────────────────────────────────────────┐  │
│  │ • 累计亏损 <= 10%?                                  │  │
│  │ • 单票仓位 + 本次 <= 30%?                           │  │
│  │ • 总仓位 + 本次 <= 80%?                             │  │
│  │ • 日熔断未触发?                                      │  │
│  └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
           │
           ▼ (通过)
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 卖出风控 (仅卖出信号)                               │
│  ┌─────────────────────────────────────────────────────┐  │
│  │ • 止损条件触发?                                      │  │
│  │ • 止盈条件触发?                                      │  │
│  │ • T+1 可卖出?                                        │  │
│  └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
           │
           ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 返回风控结果                                       │
│  - 通过: 返回交易许可                                       │
│  - 未通过: 返回拒绝原因                                     │
└─────────────────────────────────────────────────────────────┘
```

## 4. 数据结构

### 4.1 风控请求

```java
public class RiskCheckRequest {
    private String signalType;           // buy/sell
    private String stockCode;            // 股票代码
    private BigDecimal quantity;         // 数量
    private BigDecimal price;           // 价格
    private Account account;            // 账户信息
}
```

### 4.2 风控结果

```java
public class RiskCheckResult {
    private boolean passed;              // 是否通过
    private String rejectReason;         // 拒绝原因 (如果未通过)
    private List<RiskRule> checkedRules; // 检查的规则列表
    
    // 详细检查结果
    private BigDecimal currentPositionRatio;  // 当前仓位
    private BigDecimal todayPnlPercent;       // 日盈亏比例
    private BigDecimal monthPnlPercent;        // 月盈亏比例
    private BigDecimal totalPnlPercent;        // 累计盈亏比例
    private boolean dayCircuitBreaker;         // 日熔断
    private boolean monthCircuitBreaker;       // 月熔断
    
    private LocalDateTime checkedAt;    // 检查时间
}
```

### 4.3 风控规则

```java
public class RiskRule {
    private String ruleId;              // 规则ID
    private String ruleName;            // 规则名称
    private String description;         // 规则描述
    private boolean passed;              // 是否通过
    private BigDecimal threshold;       // 阈值
    private BigDecimal actualValue;     // 实际值
}
```

## 5. 风控配置

### 5.1 默认配置

```yaml
risk:
  management:
    # 止损规则
    stop_loss:
      daily: -0.03        # 日止损 3%
      cumulative: -0.10   # 累计止损 10%
      take_profit: 0.10   # 止盈 10%
    
    # 仓位规则
    position:
      single_stock: 0.30  # 单票 30%
      total: 0.80         # 总仓位 80%
      min_cash: 10000     # 最低保留现金
    
    # 熔断规则
    circuit_breaker:
      daily: -0.05        # 日熔断 5%
      monthly: -0.15       # 月熔断 15%
```

## 6. 熔断机制

### 6.1 日熔断

```
触发条件: 日亏损 >= 5%
执行动作: 
  1. 停止当日所有买入交易
  2. 允许卖出（止损）
  3. 记录熔断日志
恢复条件: 次日自动恢复
```

### 6.2 月熔断

```
触发条件: 月亏损 >= 15%
执行动作:
  1. 停止本周所有交易
  2. 发送告警通知
恢复条件: 下周一自动恢复
```

### 7.2 服务调用

```java
@Service
public class RiskManagementService {
    
    public RiskCheckResult checkBeforeTrade(RiskCheckRequest request) {
        RiskCheckResult result = new RiskCheckResult();
        List<RiskRule> rules = new ArrayList<>();
        
        // 1. 基本检查
        rules.add(checkBasicConditions());
        
        // 2. 根据信号类型检查
        if ("buy".equals(request.getSignalType())) {
            rules.add(checkCumulativeLoss());
            rules.add(checkSinglePosition(request));
            rules.add(checkTotalPosition(request));
            rules.add(checkDayCircuitBreaker());
        } else if ("sell".equals(request.getSignalType())) {
            rules.add(checkT1Policy(request));
        }
        
        // 3. 汇总结果
        result.setPassed(rules.stream().allMatch(RiskRule::isPassed));
        result.setCheckedRules(rules);
        
        return result;
    }
}
```
