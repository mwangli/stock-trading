## 1. 模块概述

数据采集模块是整个系统的基础数据层，负责股票数据、财经新闻的采集、存储和分发。

## 3. 核心组件

### 3.1 TushareClient (Tushare API 客户端)

- **职责**: HTTP 请求 Tushare Pro API，获取股票列表和 K 线数据
- **技术**: OkHttp + FastJSON2
- **频控**: 50 次/分钟，请求间隔 1.2 秒
- **数据**: 股票基本信息、日线 K 线数据

### 3.2 RomeClient (RSS 解析器)

- **职责**: 解析财经 RSS 源，提取新闻标题、内容、发布时间
- **技术**: ROME 框架 1.18.0
- **RSS 源**: 新浪财经、东方财富、中国证券报等
- **特性**: 自动编码识别，支持多种 RSS 格式

### 3.3 JsoupClient (网页抓取器)

- **职责**: 作为 RSS 的补充，直接抓取财经网站新闻
- **技术**: Jsoup 1.15.3
- **场景**: RSS 不可用时的备用方案
- **特性**: 自定义 CSS 选择器，反爬虫处理

### 3.4 DataCollector (数据采集器)

- **职责**: 协调各采集客户端，执行数据采集流水线
- **功能**: 数据清洗、去重、验证、存储
- **调度**: Spring Scheduler 定时任务

## 4. 数据流程

### 4.1 股票数据采集流程

```
Tushare API → TushareClient → StockCollector → 数据清洗 → 存储
                                           ↓
                                    MySQL (stock_info)
                                    MongoDB (stock_prices)
```

### 4.2 新闻数据采集流程

```
RSS 源/网页 → RomeClient/Jsoup → NewsCollector → 清洗去重 → 存储
                                              ↓
                                       MySQL (news)
                                              ↓
                                       AI 模块 (情感分析)
```

## 5. 数据存储设计

### 5.1 MySQL 表结构

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| stock_info | 股票信息 | stock_code, stock_name, list_date, industry |
| news | 新闻数据 | id, title, content, source, url, pub_time, create_time |

### 5.2 MongoDB 集合

| 集合名 | 说明 | 主要字段 |
|--------|------|----------|
| stock_prices | K 线数据 | stock_code, trade_date, open, high, low, close, volume |

## 6. 定时任务

| 任务名 | Cron 表达式 | 说明 |
|--------|------------|------|
| syncStockList | 0 0 1 * * * | 每日凌晨同步股票列表 |
| syncDailyKline | 0 30 18 * * 1-5 | 交易日收盘后同步 K 线 |
| collectNews | 0 */30 9-15 * * 1-5 | 交易时间每 30 分钟采集新闻 |

## 7. 技术栈

- **HTTP 客户端**: OkHttp 4.12
- **RSS 解析**: ROME 1.18.0
- **HTML 解析**: Jsoup 1.15.3
- **ORM**: MyBatis-Plus 3.5.5 (MySQL)
- **文档存储**: Spring Data MongoDB
- **工具类**: Hutool 5.8, FastJSON2 2.0

## 8. 关键设计决策

### 8.1 混合数据源策略

- **主方案**: RSS 源解析 (ROME)
- **备用方案**: 网页抓取 (Jsoup)
- **原因**: RSS 源可能失效，需要备用方案保证数据可用性

### 8.2 存储分离设计

- **MySQL**: 结构化数据 (股票信息、新闻)
- **MongoDB**: 时序数据 (K 线价格)
- **原因**: K 线数据量大，MongoDB 更适合时序数据存储和查询

### 8.3 频控与容错

- **API 频控**: 严格遵守 Tushare 积分限制
- **重试机制**: 网络失败自动重试 3 次
- **异常隔离**: 单个数据源失败不影响其他采集任务

## 10. 与其他模块的关系

- **AI 模型模块**: 提供新闻数据用于情感分析
- **策略模块**: 提供股票列表和 K 线数据用于选股
- **执行模块**: 不直接交互
