## 1. 模块概述

数据采集模块是整个系统的基础数据层，负责从证券平台获取股票数据，并存储到数据库中。

**重要变更 (v2.0)**:
- ✅ 移除 TuShare、ROME 等第三方数据源
- ✅ 统一使用证券平台接口获取数据
- ✅ 数据采集接口不需要 Token，交易接口才需要 Token

## 3. 核心组件

### 3.1 SecuritiesPlatformClient (证券平台客户端)

- **职责**: 调用证券平台 HTTP 接口获取股票数据
- **技术**: Hutool HttpUtil + FastJSON2
- **特点**: 
  - 不需要 Token 即可调用数据采集接口
  - 自动处理请求参数和响应解析
  - 超时时间：10 秒
- **接口列表**:
  - `getDataList()`: 获取所有 A 股股票列表
  - `getHistoryPrices(String code)`: 获取股票历史 K 线数据
  - `getNowPrice(String code)`: 获取股票实时价格

### 3.2 StockCollector (股票采集器)

- **职责**: 调用证券平台客户端，执行数据采集流水线
- **功能**: 
  - 采集股票列表并保存到 MySQL
  - 采集历史 K 线数据并保存到 MongoDB
  - 增量更新（检测已存在数据）
- **调度**: Spring Scheduler 定时任务

### 3.3 定时任务

#### SyncSecuritiesDataJob (股票列表同步)
- **执行时间**: 周一到周五 09:00
- **职责**: 同步最新股票列表
- **是否需要 Token**: ❌ 不需要

#### SyncHistoryDataJob (历史数据同步)
- **执行时间**: 周一到周五 18:00
- **职责**: 同步所有股票的历史 K 线数据
- **是否需要 Token**: ❌ 不需要

#### RefreshTokenJob (Token 刷新)
- **执行频率**: 每 30 分钟
- **职责**: 刷新交易接口的 Token
- **是否需要 Token**: ✅ 需要 (但会自动获取)

## 4. 数据流程

### 4.1 股票数据采集流程

```
证券平台接口
    ↓
SecuritiesPlatformClient.getDataList()
    ↓
StockCollector.collectStockList()
    ↓
数据清洗和验证
    ↓
MySQL (stock_info)
    ↓
检测新增/更新
    ↓
保存或更新
```

### 4.2 历史 K 线采集流程

```
证券平台接口
    ↓
SecuritiesPlatformClient.getHistoryPrices(code)
    ↓
StockCollector.collectHistoricalData(code, days)
    ↓
数据转换 (除以 100 转换为实际价格)
    ↓
MongoDB (stock_prices)
    ↓
检测是否已存在
    ↓
新增或更新
```

## 5. 接口详细说明

### 5.1 获取股票列表

**方法**: `getDataList()`

**不需要 Token**

**请求参数**:
```java
{
    "c.funcno": 21000,
    "c.version": 1,
    "c.sort": 1,
    "c.order": 1,
    "c.type": "0:2:9:18",  // 市场类型
    "c.curPage": 1,
    "c.rowOfPage": 5000,   // 每页数量
    "c.field": "1:2:22:23:24:3:8:16:21:31"  // 返回字段
}
```

**返回数据**:
- 股票代码
- 股票名称
- 当前价格
- 涨跌幅
- 市场类型

### 5.2 获取历史 K 线

**方法**: `getHistoryPrices(String code)`

**不需要 Token**

**请求参数**:
```java
{
    "c.funcno": 20009,
    "c.version": 1,
    "c.stock_code": "000001",
    "c.type": "day",      // 日线
    "c.count": "100"      // 返回数量
}
```

**返回数据**:
- 日期
- 开盘价 (price1)
- 收盘价 (price2)
- 最高价 (price3)
- 最低价 (price4)

**注意**: 返回的价格需要除以 100 才是实际价格

### 5.3 获取实时价格

**方法**: `getNowPrice(String code)`

**不需要 Token**

**请求参数**:
```java
{
    "stockcode": "000001",
    "action": 33,
    "Level": 1,
    "UseBPrice": 1
}
```

**返回数据**:
- 当前价格

## 6. 数据存储设计

### 6.1 MySQL 表结构

**stock_info** (股票信息表)
```sql
CREATE TABLE stock_info (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    code VARCHAR(10) NOT NULL,      -- 股票代码
    name VARCHAR(50) NOT NULL,      -- 股票名称
    market VARCHAR(20),             -- 市场类型
    price DECIMAL(18,4),            -- 当前价格
    increase DECIMAL(18,4),         -- 涨跌幅
    deleted BOOLEAN DEFAULT FALSE,  -- 是否已删除
    create_time DATETIME,
    update_time DATETIME,
    UNIQUE KEY uk_code (code)
);
```

### 6.2 MongoDB 集合

**stock_prices** (股票价格数据)
```javascript
{
    _id: ObjectId,
    code: String,           -- 股票代码
    name: String,           -- 股票名称
    date: String,           -- 日期 (yyyy-MM-dd)
    price1: Double,         -- 开盘价
    price2: Double,         -- 收盘价
    price3: Double,         -- 最高价
    price4: Double,         -- 最低价
    createTime: Date,
    updateTime: Date
}
```

**索引**:
```javascript
db.stock_prices.createIndex({code: 1, date: 1}, {unique: true})
```

## 8. 关键设计决策

### 8.1 数据源统一

- **选择**: 统一使用证券平台接口
- **原因**:
  - 数据更准确可靠
  - 无需维护多个数据源
  - 接口稳定性更好

### 8.2 Token 分离设计

- **数据采集接口**: 不需要 Token
- **交易接口**: 需要 Token
- **优势**: 数据采集更简单，无需处理 Token 刷新

### 8.3 增量更新

- **股票列表**: 检测新增和更新，避免重复插入
- **K 线数据**: 检查日期是否已存在，更新或插入

## 11. 常见问题

### Q1: 接口调用失败怎么办？
A: 检查网络连接，查看日志中的详细错误信息

### Q2: 数据返回为空？
A: 检查股票代码是否正确，接口参数是否正确

### Q3: 需要 Token 吗？
A: 数据采集接口不需要 Token，只有交易接口才需要
