### 3.1 实际代码结构


### 3.2 Controller 层

#### LstmTrainingController
- **位置**: `controller/LstmTrainingController.java`
- **职责**: 提供 LSTM 模型训练的 REST API
- **接口**:
  - `POST /api/lstm/train`: 启动模型训练
  - `GET /api/lstm/status`: 获取训练状态
  - `POST /api/lstm/predict`: 执行预测

#### SentimentController
- **位置**: `controller/SentimentController.java`
- **职责**: 提供情感分析的 REST API
- **接口**:
  - `POST /api/sentiment/train`: 启动情感模型训练
  - `POST /api/sentiment/analyze`: 分析文本情感
  - `GET /api/sentiment/status`: 获取训练状态

### 3.3 Service 层

#### LstmTrainerService
- **位置**: `service/LstmTrainerService.java`
- **职责**: LSTM 模型训练和更新
- **功能**: 
  - 数据预处理 (归一化、序列化)
  - 模型训练
  - 模型评估
  - 模型保存和加载
- **状态管理**: TrainingStatus (IDLE, TRAINING, COMPLETED, FAILED)

#### SentimentTrainerService
- **位置**: `service/SentimentTrainerService.java`
- **职责**: 情感分析模型训练
- **功能**:
  - 新闻数据预处理
  - BERT 模型微调
  - 模型评估

#### SentimentDataPreprocessor
- **位置**: `service/SentimentDataPreprocessor.java`
- **职责**: 情感分析数据预处理
- **功能**:
  - 数据清洗
  - 数据集划分 (训练/验证/测试)
  - 数据增强

### 3.4 Model 层

#### StockLSTM
- **位置**: `model/StockLSTM.java`
- **职责**: LSTM 模型架构定义
- **结构**:
  - 输入层：(batch_size, 60, 5)
  - LSTM 层 1: 128 units, dropout=0.2
  - LSTM 层 2: 64 units, dropout=0.2
  - 全连接层 1: 32 units, ReLU
  - Dropout: 0.3
  - 全连接层 2: 1 unit, Sigmoid
  - 输出：涨的概率 (0-1)

### 3.5 Inference 层

#### LstmInference
- **位置**: `inference/LstmInference.java`
- **职责**: LSTM 模型推理
- **功能**:
  - 加载预训练模型
  - 执行预测
  - 结果缓存

#### SentimentInference
- **位置**: `inference/SentimentInference.java`
- **职责**: 情感分析推理
- **功能**:
  - 加载预训练模型
  - 文本情感分类
  - 结果输出 (SentimentResult)

## 4. 数据流程

### 4.1 LSTM 预测流程

```
从 MongoDB 读取 60 日 K 线
    ↓
数据预处理 (归一化、填充)
    ↓
转换为 Tensor
    ↓
LSTM 模型推理
    ↓
输出层计算概率
    ↓
缓存结果 (Redis)
    ↓
输出：涨的概率、预测变化率
```

### 4.2 情感分析流程

```
从 MySQL 读取新闻
    ↓
文本清洗和分词
    ↓
BERT Tokenizer 编码
    ↓
BERT 模型推理
    ↓
情感分类和得分计算
    ↓
实体识别 (股票代码)
    ↓
输出：情感得分、关联股票
```

### 4.3 模型训练流程

```
接收训练请求 (Controller)
    ↓
参数验证和配置加载
    ↓
数据加载和预处理 (Service)
    ↓
模型构建 (Model)
    ↓
训练循环
    ↓
模型评估
    ↓
保存模型
    ↓
返回训练结果
```

## 5. 模型架构

### 5.1 LSTM 模型结构

```
输入层：(batch_size, 60, 5)
    ↓
LSTM 层 1: 128 units, dropout=0.2
    ↓
LSTM 层 2: 64 units, dropout=0.2
    ↓
全连接层 1: 32 units, ReLU
    ↓
Dropout: 0.3
    ↓
全连接层 2: 1 unit, Sigmoid
    ↓
输出：涨的概率 (0-1)
```

**输入特征** (5 维):
- 开盘价 (归一化)
- 最高价 (归一化)
- 最低价 (归一化)
- 收盘价 (归一化)
- 成交量 (归一化)

### 5.2 情感分析模型

- **预训练模型**: bert-base-chinese
- **微调数据**: 金融新闻标注数据
- **输出层**: 3 分类 (利好/利空/中性)
- **映射**: 利好=1, 中性=0, 利空=-1

## 8. 关键设计决策

### 8.1 模型选择

- **LSTM**: 适合时间序列预测
- **FinBERT**: 金融领域预训练，效果好
- **DJL**: Java 原生支持，部署简单

### 8.2 缓存策略

- **预测结果缓存**: 1 天有效期
- **减少重复计算**: 相同输入直接返回缓存
- **缓存失效**: 模型更新后清空缓存

### 8.3 性能优化

- **批量推理**: 支持多股票同时预测
- **模型预热**: 启动时加载模型到内存
- **异步推理**: 后台线程执行预测

## 9. 对外接口

### 9.1 REST API

```java
// LSTM 训练接口
POST /api/lstm/train
Request: TrainingRequest {
    String stockCode;
    LocalDate startDate;
    LocalDate endDate;
    int epochs;
    int batchSize;
    double learningRate;
}
Response: TrainingResponse {
    String modelId;
    TrainingStatus status;
    double accuracy;
    double loss;
}

// LSTM 预测接口
POST /api/lstm/predict
Request: { String stockCode; }
Response: { double riseProbability; double predictedChange; }

// 情感分析接口
POST /api/sentiment/analyze
Request: { String text; }
Response: SentimentAnalysisResult {
    double sentimentScore;
    String sentimentLabel;
    List<String> relatedStocks;
}
```

### 9.2 服务接口

```java
// LSTM 训练服务
LstmTrainerService
  - train(TrainingRequest request): TrainingResponse
  - getStatus(): TrainingStatus
  - getTrainingResult(): TrainingResult

// LSTM 推理服务
LstmInference
  - predict(String stockCode): LstmPrediction
  - batchPredict(List<String> stockCodes): Map<String, LstmPrediction>

// 情感分析服务
SentimentInference
  - analyze(String text): SentimentResult
  - batchAnalyze(List<String> texts): List<SentimentResult>
```
