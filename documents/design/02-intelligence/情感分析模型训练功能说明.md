# 情感分析模型训练功能实现

本次实现为 AI 股票交易系统添加了完整的情感分析模型训练、下载和使用功能，支持基于**中文 BERT 模型**的财经新闻情感分析。

## 重要更新

- ✅ **已切换为中文 BERT 模型**（哈工大 `chinese-bert-wwm-ext`）
- ✅ 支持中文文本情感分析
- ✅ 情感标签：负面、中性、正面
- ✅ 针对中文财经新闻优化

## 实现的功能模块

### 1. 配置管理 (`SentimentTrainingConfig`)
- 位置：`backend/src/main/java/com/stock/models/config/SentimentTrainingConfig.java`
- 功能：集中管理情感分析训练的所有超参数
- 配置项：
  - `pretrained-model`: 预训练模型名称（默认 `hfl/chinese-bert-wwm-ext`，哈工大中文 BERT）
  - `max-sequence-length`: 最大序列长度（默认 128）
  - `epochs`: 训练轮次（默认 5）
  - `batch-size`: 批次大小（默认 16）
  - `learning-rate`: 学习率（默认 2e-5）
  - `train-ratio`: 训练集比例（默认 0.8）
  - `num-labels`: 标签数量（默认 3：负面、中性、正面）
  - `labels`: 情感标签列表

### 2. 数据预处理服务 (`SentimentDataPreprocessor`)
- 位置：`backend/src/main/java/com/stock/models/service/SentimentDataPreprocessor.java`
- 功能：
  - 从 MongoDB 加载财经新闻数据
  - 文本清洗：移除 HTML 标签、URL、特殊字符
  - 自动标注：基于情感关键词规则自动标注训练数据
  - 数据集构建和划分（训练集/验证集）
  - 生成模拟数据（用于测试）

### 3. 模型训练服务 (`SentimentTrainerService`)
- 位置：`backend/src/main/java/com/stock/models/service/SentimentTrainerService.java`
- 功能：
  - 训练流程编排
  - 预训练模型下载（从 HuggingFace/DJL model zoo）
  - 模型加载和推理
  - 训练进度追踪
  - 模型保存与管理

### 4. 推理组件增强 (`SentimentInference`)
- 位置：`backend/src/main/java/com/stock/models/inference/SentimentInference.java`
- 新增功能：
  - 集成真实 BERT 模型推理
  - 支持模型热加载/重新加载
  - 模型状态查询
  - 降级方案：模型未加载时使用规则推理

### 5. REST API 控制器 (`SentimentController`)
- 位置：`backend/src/main/java/com/stock/web/controller/SentimentController.java`
- 接口：
  - `POST /api/models/sentiment/train` - 启动训练
  - `GET /api/models/sentiment/status/{trainingId}` - 查询训练状态
  - `POST /api/models/sentiment/analyze` - 分析单条文本情感
  - `POST /api/models/sentiment/analyze/batch` - 批量分析文本情感
  - `GET /api/models/sentiment/health` - 健康检查
  - `POST /api/models/sentiment/reload` - 重新加载模型
  - `POST /api/models/sentiment/download` - 下载预训练模型

### 6. 数据传输对象
- 位置：`backend/src/main/java/com/stock/models/dto/SentimentTrainingRequest.java`
- DTO 类：
  - `SentimentTrainingRequest` - 训练请求参数
  - `SentimentTrainingResponse` - 训练响应
  - `SentimentAnalysisResult` - 情感分析结果
  - `TrainingSample` - 训练样本

## 使用方法

### 1. 下载预训练模型（可选）

```bash
curl -X POST http://localhost:8080/api/models/sentiment/download
```

### 2. 启动训练

```bash
curl -X POST http://localhost:8080/api/models/sentiment/train \
  -H "Content-Type: application/json" \
  -d '{
    "dataSource": "news",
    "numSamples": 500,
    "epochs": 5,
    "batchSize": 16,
    "learningRate": 0.00002,
    "autoLabel": true
  }'
```

### 3. 查询训练状态

```bash
curl http://localhost:8080/api/models/sentiment/status/sentiment_training_1234567890
```

### 4. 分析单条文本

```bash
curl -X POST http://localhost:8080/api/models/sentiment/analyze \
  -H "Content-Type: application/json" \
  -d '{
    "text": "公司业绩大幅增长，净利润创新高"
  }'
```

响应：
```json
{
  "success": true,
  "label": "positive",
  "score": 0.85,
  "confidence": 0.92,
  "probabilities": {
    "negative": 0.03,
    "neutral": 0.05,
    "positive": 0.92
  },
  "text": "公司业绩大幅增长，净利润创新高",
  "modelLoaded": true
}
```

### 5. 批量分析

```bash
curl -X POST http://localhost:8080/api/models/sentiment/analyze/batch \
  -H "Content-Type: application/json" \
  -d '{
    "texts": [
      "公司业绩大幅增长",
      "股票价格暴跌",
      "市场表现平稳"
    ]
  }'
```

### 6. 健康检查

```bash
curl http://localhost:8080/api/models/sentiment/health
```

    ↓ 数据集划分
    ↓

    ↓

    ↓ 保存模型
    ↓
模型文件 (models/sentiment-analysis/)
    ↓

    ↓ 模型加载
    ↓ 情感分析推理
```

## 文本预处理流程

1. **转小写**：统一文本格式
2. **移除 HTML**：清理 `<tag>` 等 HTML 标签
3. **移除 URL**：删除 http:// 链接
4. **清理特殊字符**：保留中文、英文、数字和标点
5. **压缩空格**：多个空格合并为一个

## 自动标注规则

基于情感关键词匹配：

**正面关键词**（+1 分）：
- 增长、盈利、上涨、突破、利好、推荐、买入、收益、业绩、向好、创新高、大涨、飙升、激增、超预期、优秀、强劲、复苏、回暖

**负面关键词**（-1 分）：
- 下跌、亏损、风险、减持、利空、卖出、警告、业绩下滑、暴跌、缩水、下滑、衰退、恶化、承压、下调、警惕、低迷、疲软、危机

**标注规则**：
- 正面分 > 负面分 + 1 → 正面 (label=2)
- 负面分 > 正面分 + 1 → 负面 (label=0)
- 其他 → 中性 (label=1)

在 `application.yml` 中已添加以下配置：

models:

    model-path: models/sentiment-analysis
    pretrained-model: distilbert-base-uncased
    max-sequence-length: 128
    epochs: 5
    batch-size: 16
    learning-rate: 0.00002
    train-ratio: 0.8
    num-labels: 3
    download-pretrained: true
    cache-dir: models/cache
    labels:
      - negative
      - neutral
      - positive
```

## 模型架构

使用 **哈工大中文 BERT**（HFL/Chinese-BERT-wwm-ext）：
- 哈工大讯飞联合实验室发布的中文 BERT 模型
- 使用 Whole Word Masking (wwm) 技术
- 在中文 NLP 任务上表现优异
- 适合中文文本情感分析
- 模型大小：约 400MB

**其他可选中文模型**：
- `hfl/chinese-roberta-wwm-ext`：RoBERTa 中文，更准确但更大
- `hfl/chinese-macbert-base`：MacBERT 中文，针对性优化
- `bert-base-chinese`：Google 官方 BERT 中文

```
输入中文文本
    ↓
Tokenizer (中文分词 + BPE)
    ↓
Chinese-BERT Encoder (12 layers)
    ↓
Pooler + Dropout
    ↓
Linear Classifier
    ↓
Softmax
    ↓
[负面，中性，正面] 概率
```

## 注意事项

1. **首次运行**：
   - 中文 BERT 模型较大（约 400MB），下载需要几分钟
   - 首次使用时需要下载或手动放置模型文件
   - 确保网络连接正常

2. **中文模型下载方式**：
   
   **方式 1：使用 HuggingFace（推荐）**
   ```bash
   # 使用 git clone 下载
   cd models/sentiment-analysis
   git lfs install
   git clone https://huggingface.co/hfl/chinese-bert-wwm-ext
   ```
   
   **方式 2：手动下载**
   - 访问：https://huggingface.co/hfl/chinese-bert-wwm-ext
   - 下载文件：`pytorch_model.bin`, `config.json`, `vocab.txt`
   - 放到：`models/sentiment-analysis/` 目录

3. **数据要求**：
   - 建议至少 100 条中文新闻数据进行训练
   - 自动标注适用于快速原型，建议后期使用人工标注数据

4. **训练时间**：
   - CPU：约 5-10 分钟（5 epochs，500 样本）
   - GPU：推荐用于大规模训练（速度快 10-20 倍）

5. **模型保存**：
   - 模型保存在 `models/sentiment-analysis/` 目录
   - 包含模型配置 JSON 和参数文件

6. **内存使用**：
   - 中文 BERT 约占用 800MB-1.5GB 内存
   - 批次大小可根据内存调整（默认 16）

7. **中文文本处理**：
   - 自动支持中文分词（使用 BERT tokenizer）
   - 无需额外分词工具
   - 支持中英文混合文本

## 与 LSTM 训练功能对比

| 功能 | LSTM 价格预测 | 情感分析 |
|------|--------------|----------|
| 模型类型 | 回归 | 分类 |
| 输入数据 | 数值序列（价格、成交量） | 文本（新闻、评论） |
| 特征维度 | 5（开高低收量） | 可变（文本长度） |
| 输出 | 预测价格（连续值） | 情感标签（类别） |
| 评估指标 | MSE、MAE | Accuracy、F1 |
| 预训练模型 | 无（从头训练） | DistilBERT（迁移学习） |
| 训练时间 | 较长 | 较短（微调） |
