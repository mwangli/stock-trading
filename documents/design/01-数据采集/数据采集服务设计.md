## 重要变更说明

**架构变更**: 从Java调用Python HTTP接口，改为Python服务直接采集并写入数据库

**变更原因**:
1. 减少HTTP调用链路，降低系统复杂度
2. Python直接操作数据库，数据一致性更好
3. 避免Java层中转，提升性能
4. 定时任务在Python层执行，职责更清晰

## 2. 概述

### 2.1 模块定位

数据采集模块是整个股票交易系统的数据源头，负责从外部数据源采集A股行情数据、财经新闻数据，为其他模块提供数据支撑。

### 2.2 功能摘要

- 采集全市场股票基本信息（代码、名称、行业等）
- 采集个股历史K线数据用于模型训练和历史分析
- 采集实时行情数据用于交易决策
- 采集财经新闻用于情感分析

### 2.3 上下游依赖

| 上游模块 | 接口 | 数据 |
|----------|------|------|
| 无 | - | 外部数据源(AKShare/新浪财经/东方财富) |

**下游数据消费者**:

| 下游模块 | 数据来源 | 数据 |
|----------|----------|------|
| LSTM预测 | MongoDB (stock_prices) | 历史K线数据 |
| 情感分析 | MySQL/MongoDB (news) | 财经新闻 |
| 决策引擎 | MySQL/Redis (stock_info) | 实时行情 |
| 交易执行 | MySQL (stock_info) | 股票信息 |
| Java后端 | MySQL/MongoDB | 所有数据（只读查询） |

**重要**: Java后端不再调用Python服务API，而是直接从数据库读取数据

### 3.2 功能详情

#### F-001: 股票列表同步

**功能描述**: 从AKTools API获取全市场股票列表，排除ST、停牌等不可交易股票，维护本地股票信息表。

**输入**: 无

**处理逻辑**:
1. 调用AKTools股票列表接口
2. 解析返回的股票数据
3. 过滤不可交易股票（ST、停牌、退市）
4. 批量更新到本地数据库

**输出**: 股票信息列表 (List<StockInfo>)

**业务规则**:
- 每日盘前同步一次
- 只保留可交易股票
- 同步后更新最新日期戳

#### F-002: 历史行情获取

**功能描述**: 获取指定股票的历史K线数据，用于模型训练和历史分析。

**输入**: 
- stockCode: 股票代码
- startDate: 开始日期
- endDate: 结束日期

**处理逻辑**:
1. 检查本地数据库是否有足够数据
2. 如需增量获取，调用AKTools K线接口
3. 解析并存储到数据库

**输出**: 历史价格列表 (List<StockPrices>)

**业务规则**:
- 优先从本地数据库读取
- 增量同步时跳过已有数据
- 保留OHLCV全部字段

#### F-003: 实时行情获取

**功能描述**: 获取个股实时行情数据，用于交易决策。

**输入**: stockCode: 股票代码

**处理逻辑**:
1. 调用AKTools实时行情接口
2. 解析返回的行情数据
3. 返回当前价格和涨跌幅

**输出**: 实时价格 (Double)

**业务规则**:
- 交易时间段内每分钟更新
- 非交易时间返回昨日收盘价

#### F-005: 财经新闻采集

**功能描述**: 从新浪财经、东方财富等平台采集财经新闻。

**输入**: stockCode: 股票代码（可选）

**处理逻辑**:
1. 并行请求多个新闻源
2. 解析新闻标题、内容、发布时间
3. 新闻去重（基于URL或标题哈希）
4. 存储到MongoDB

**输出**: 新闻列表 (List<News>)

**业务规则**:
- 每30分钟采集一次
- 保留最近7天新闻
- 去重后保留最新版本

### 5.4 AKShare直接集成方案（新架构）

#### 5.4.1 架构演进

| 阶段 | 架构 | 说明 |
|------|------|------|
| **旧架构** | Java → HTTP → Python API → 返回数据 → Java写入数据库 | 链路长、复杂度高 |
| **新架构** | Python → 直接采集 → 直接写入 → MySQL/MongoDB/Redis | 链路短、职责清晰 |

#### 5.4.2 新架构核心组件

| 组件 | 技术 | 职责 |
|------|------|------|
| **DataCollectionService** | Python/FastAPI | 数据采集、处理、写入数据库 |
| **APScheduler** | Python | 定时执行数据采集任务 |
| **AKShare** | Python库 | 从东方财富/新浪财经获取数据 |
| **SQLAlchemy** | Python ORM | MySQL数据写入 |
| **PyMongo** | Python驱动 | MongoDB数据写入 |
| **Redis-py** | Python客户端 | Redis缓存更新 |

#### 5.4.3 数据流向

```
外部数据源(AKShare)
        │
        ▼
┌──────────────────────┐
│ DataCollectionService│  ← Python直接调用AKShare
│   (数据采集)          │
└──────────┬───────────┘
           │
    ┌──────┼──────┐
    ▼      ▼      ▼
┌──────┐ ┌──────┐ ┌──────┐
│ MySQL│ │Mongo │ │ Redis│
└──┬───┘ └──┬───┘ └──┬───┘
   │        │        │
   ▼        ▼        ▼
┌──────────────────────────┐
│    Java Backend          │  ← 只读查询
│  (StockDataQueryService) │
└──────────────────────────┘
```

#### 5.4.4 新架构优势

1. **减少调用链路**: 从3层(Java-Python-DB)减少到2层(Python-DB)
2. **提升性能**: 避免HTTP序列化和Java层中转
3. **数据一致性**: Python直接写入，避免Java层数据处理错误
4. **职责分离**: 数据采集(Python)与业务查询(Java)完全分离
5. **降低复杂度**: Java后端只负责查询，代码更简洁

### 7.2 Python异常处理策略

```python
import asyncio
from tenacity import retry, stop_after_attempt, wait_exponential

class DataCollectionService:
    
    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=1, max=10),
        reraise=True
    )
    async def fetch_and_save_stock_list(self) -> int:
        """带重试机制的股票列表同步"""
        try:
            # 1. 获取数据
            stocks = await self._fetch_from_akshare()
            # 2. 写入数据库
            count = await self._save_to_mysql(stocks)
            return count
        except Exception as e:
            logger.error(f"同步股票列表失败: {e}")
            raise
    
    async def _save_to_mysql(self, stocks: List[Dict]) -> int:
        """保存到MySQL，带事务处理"""
        try:
            with self.db_session.begin():
                for stock in stocks:
                    # 检查是否存在
                    existing = self._get_by_code(stock['code'])
                    if existing:
                        self._update(stock)
                    else:
                        self._insert(stock)
            return len(stocks)
        except Exception as e:
            logger.error(f"写入MySQL失败: {e}")
            raise DataWriteException("DATA_006", str(e))
```
