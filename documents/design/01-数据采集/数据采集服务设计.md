## 2. 概述

### 2.1 模块定位

数据采集模块是整个股票交易系统的数据源头，负责从外部数据源采集A股行情数据、财经新闻数据，为其他模块提供数据支撑。

### 2.2 功能摘要

- 采集全市场股票基本信息（代码、名称、行业等）
- 采集个股历史K线数据用于模型训练和历史分析
- 采集实时行情数据用于交易决策
- 采集财经新闻用于情感分析

### 2.3 上下游依赖

| 上游模块 | 接口 | 数据 |
|----------|------|------|
| 无 | - | 外部数据源 |

| 下游模块 | 接口 | 数据 |
|----------|------|------|
| LSTM预测 | getStockPrices() | 历史K线数据 |
| 情感分析 | getNews() | 财经新闻 |
| 决策引擎 | getRealtimeQuote() | 实时行情 |
| 交易执行 | getStockInfo() | 股票信息 |

### 3.2 功能详情

#### F-001: 股票列表同步

**功能描述**: 从AKTools API获取全市场股票列表，排除ST、停牌等不可交易股票，维护本地股票信息表。

**输入**: 无

**处理逻辑**:
1. 调用AKTools股票列表接口
2. 解析返回的股票数据
3. 过滤不可交易股票（ST、停牌、退市）
4. 批量更新到本地数据库

**输出**: 股票信息列表 (List<StockInfo>)

**业务规则**:
- 每日盘前同步一次
- 只保留可交易股票
- 同步后更新最新日期戳

#### F-002: 历史行情获取

**功能描述**: 获取指定股票的历史K线数据，用于模型训练和历史分析。

**输入**: 
- stockCode: 股票代码
- startDate: 开始日期
- endDate: 结束日期

**处理逻辑**:
1. 检查本地数据库是否有足够数据
2. 如需增量获取，调用AKTools K线接口
3. 解析并存储到数据库

**输出**: 历史价格列表 (List<StockPrices>)

**业务规则**:
- 优先从本地数据库读取
- 增量同步时跳过已有数据
- 保留OHLCV全部字段

#### F-003: 实时行情获取

**功能描述**: 获取个股实时行情数据，用于交易决策。

**输入**: stockCode: 股票代码

**处理逻辑**:
1. 调用AKTools实时行情接口
2. 解析返回的行情数据
3. 返回当前价格和涨跌幅

**输出**: 实时价格 (Double)

**业务规则**:
- 交易时间段内每分钟更新
- 非交易时间返回昨日收盘价

#### F-005: 财经新闻采集

**功能描述**: 从新浪财经、东方财富等平台采集财经新闻。

**输入**: stockCode: 股票代码（可选）

**处理逻辑**:
1. 并行请求多个新闻源
2. 解析新闻标题、内容、发布时间
3. 新闻去重（基于URL或标题哈希）
4. 存储到MongoDB

**输出**: 新闻列表 (List<News>)

**业务规则**:
- 每30分钟采集一次
- 保留最近7天新闻
- 去重后保留最新版本

#### 6.1.2 核心服务接口

```java
public interface StockDataService {
    
    /**
     * 获取A股全市场股票列表
     * @return 可交易股票列表（排除ST、停牌等）
     */
    List<StockInfo> fetchStockList();
    
    /**
     * 获取个股历史K线数据
     * @param stockCode 股票代码
     * @param startDate 开始日期
     * @param endDate 结束日期
     * @return 历史价格列表
     */
    List<StockPrices> fetchHistoricalData(String stockCode, LocalDate startDate, LocalDate endDate);
    
    /**
     * 获取个股近N天历史数据
     * @param stockCode 股票代码
     * @param days 天数
     * @return 历史价格列表
     */
    List<StockPrices> fetchHistoricalData(String stockCode, int days);
    
    /**
     * 获取实时行情数据
     * @param stockCode 股票代码
     * @return 当前价格
     */
    Double fetchRealTimePrice(String stockCode);
    
    /**
     * 批量获取实时行情
     * @param stockCodes 股票代码列表
     * @return 股票代码到价格的映射
     */
    Map<String, Double> fetchRealTimePrices(List<String> stockCodes);
    
    /**
     * 获取财务报表数据
     * @param stockCode 股票代码
     * @return 财务数据JSON
     */
    String fetchFinancialReport(String stockCode);
}
```

## 7. 错误处理

### 7.1 错误码定义

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| DATA_001 | 数据源连接失败 | AKTools API不可用 |
| DATA_002 | API频率超限 | 请求过于频繁 |
| DATA_003 | 数据解析失败 | 返回数据格式异常 |
| DATA_004 | 股票不存在 | 股票代码无效 |
| DATA_005 | 网络请求超时 | 请求超时 |

### 7.2 异常处理策略

| 异常类型 | 处理策略 |
|----------|----------|
| DataSourceException | 切换备用源，重试 |
| RateLimitException | 等待后重试，加入队列缓冲 |
| ParseException | 记录日志，返回空结果 |
| NetworkException | 重试3次，失败后告警 |

```java
@Service
public class StockDataServiceImpl implements StockDataService {
    
    @Retryable(value = Exception.class, maxAttempts = 3, 
               backoff = @Backoff(delay = 1000))
    @Override
    public List<StockPrices> fetchHistoricalData(String stockCode, int days) {
        // 获取历史数据逻辑
    }
    
    @Recover
    public List<StockPrices> recover(Exception e, String stockCode, int days) {
        log.error("获取历史数据失败, stockCode={}, days={}", stockCode, days, e);
        // 返回空列表或缓存数据
        return Collections.emptyList();
    }
}
```
