## 2. 概述

### 2.1 模块定位

数据采集模块是整个股票交易系统的数据源头，负责从外部数据源采集A股行情数据、财经新闻数据，为其他模块提供数据支撑。

**架构模式**: Python定时任务直接写入数据库，Java服务直接查库

### 2.2 功能摘要

- Python定时任务调用AKShare获取原始数据
- Python直接写入MySQL/MongoDB/Redis
- Java服务直接从数据库查询，无需HTTP调用
- 定时任务在Python层调度执行（APScheduler）

### 2.3 上下游依赖

**上游数据源**:

| 数据源 | 用途 |
|--------|------|
| AKShare | 行情数据获取 |
| 新浪财经 | 财经新闻 |
| 东方财富 | 财经新闻 |

**下游数据消费者**:

| 下游模块 | 数据来源 | 数据 |
|----------|----------|------|
| LSTM预测 | MongoDB | 历史K线数据 |
| 情感分析 | MongoDB | 财经新闻 |
| 决策引擎 | MySQL/Redis | 实时行情 |
| 交易执行 | MySQL | 股票信息 |

### 3.2 功能详情

#### F-001: 股票列表同步

**功能描述**: 从AKTools API获取全市场股票列表，排除ST、停牌等不可交易股票，维护本地股票信息表。

**输入**: 无

**处理逻辑**:
1. 调用AKTools股票列表接口
2. 解析返回的股票数据
3. 过滤不可交易股票（ST、停牌、退市）
4. 批量更新到本地数据库

**输出**: 股票信息列表 (List<StockInfo>)

**业务规则**:
- 每日盘前同步一次
- 只保留可交易股票
- 同步后更新最新日期戳

#### F-002: 历史行情获取

**功能描述**: 获取指定股票的历史K线数据，用于模型训练和历史分析。

**输入**: 
- stockCode: 股票代码
- startDate: 开始日期
- endDate: 结束日期

**处理逻辑**:
1. 检查本地数据库是否有足够数据
2. 如需增量获取，调用AKTools K线接口
3. 解析并存储到数据库

**输出**: 历史价格列表 (List<StockPrices>)

**业务规则**:
- 优先从本地数据库读取
- 增量同步时跳过已有数据
- 保留OHLCV全部字段

#### F-003: 实时行情获取

**功能描述**: 获取个股实时行情数据，用于交易决策。

**输入**: stockCode: 股票代码

**处理逻辑**:
1. 调用AKTools实时行情接口
2. 解析返回的行情数据
3. 返回当前价格和涨跌幅

**输出**: 实时价格 (Double)

**业务规则**:
- 交易时间段内每分钟更新
- 非交易时间返回昨日收盘价

#### F-005: 财经新闻采集

**功能描述**: 从新浪财经、东方财富等平台采集财经新闻。

**输入**: stockCode: 股票代码（可选）

**处理逻辑**:
1. 并行请求多个新闻源
2. 解析新闻标题、内容、发布时间
3. 新闻去重（基于URL或标题哈希）
4. 存储到MongoDB

**输出**: 新闻列表 (List<News>)

**业务规则**:
- 每30分钟采集一次
- 保留最近7天新闻
- 去重后保留最新版本

@Service
@Slf4j
@RequiredArgsConstructor
public class StockDataServiceImpl implements StockDataService {
    
    private final DataCollectionClient dataCollectionClient;
    private final StockInfoMapper stockInfoMapper;
    private final MongoTemplate mongoTemplate;
    private final RedisTemplate<String, Object> redisTemplate;
    
    private static final String PYTHON_SERVICE_URL = "http://localhost:8001";
    
    @Override
    public List<StockInfo> fetchStockList() {
        log.info("Fetching stock list from Python service...");
        
        // 1. 调用Python API获取数据
        StockListResponse response = dataCollectionClient.getStockList(1, 5000);
        
        // 2. 处理数据
        List<StockInfo> stocks = new ArrayList<>();
        if (response != null && response.getData() != null) {
            List<Map<String, Object>> stocksData = response.getData().getStocks();
            for (Map<String, Object> stockData : stocksData) {
                StockInfo stock = new StockInfo();
                stock.setCode((String) stockData.get("code"));
                stock.setName((String) stockData.get("name"));
                stock.setMarket((String) stockData.get("market"));
                stock.setIsTradable((Integer) stockData.get("is_tradable") == 1);
                stocks.add(stock);
            }
        }
        
        // 3. 写入MySQL
        stockInfoMapper.saveBatch(stocks);
        
        log.info("Saved {} stocks to MySQL", stocks.size());
        return stocks;
    }
    
    @Override
    public List<StockPrices> fetchHistoricalData(String stockCode, int days) {
        log.info("Fetching historical data for {} from Python service", stockCode);
        
        // 1. 调用Python API获取数据
        LocalDate endDate = LocalDate.now();
        LocalDate startDate = endDate.minusDays(days);
        
        HistoricalDataResponse response = dataCollectionClient.getHistoricalData(
            stockCode, 
            startDate.toString(), 
            endDate.toString(), 
            days
        );
        
        // 2. 处理数据并写入MongoDB
        List<StockPrices> prices = new ArrayList<>();
        if (response != null && response.getData() != null) {
            List<Map<String, Object>> pricesData = response.getData().getPrices();
            for (Map<String, Object> priceData : pricesData) {
                StockPrices price = new StockPrices();
                price.setCode(stockCode);
                price.setDate(LocalDate.parse((String) priceData.get("date")));
                price.setPrice1(new BigDecimal(priceData.get("open").toString()));
                price.setPrice2(new BigDecimal(priceData.get("high").toString()));
                price.setPrice3(new BigDecimal(priceData.get("low").toString()));
                price.setPrice4(new BigDecimal(priceData.get("close").toString()));
                prices.add(price);
            }
        }
        
        // 3. 写入MongoDB
        mongoTemplate.insertAll(prices);
        
        log.info("Saved {} price records to MongoDB", prices.size());
        return prices;
    }
    
    @Override
    public Double fetchRealTimePrice(String stockCode) {
        log.info("Fetching realtime quote for {} from Python service", stockCode);
        
        // 1. 调用Python API获取数据
        QuoteResponse response = dataCollectionClient.getQuote(stockCode);
        
        // 2. 处理数据
        Double price = null;
        if (response != null && response.getData() != null) {
            price = response.getData().getPrice();
            
            // 3. 更新Redis缓存
            String cacheKey = "stock:quote:" + stockCode;
            redisTemplate.opsForValue().set(cacheKey, price, 60, TimeUnit.SECONDS);
            
            // 4. 更新MySQL
            stockInfoMapper.updatePrice(stockCode, price);
        }
        
        return price;
    }
}

## 7. 错误处理（Python层）

### 7.1 错误码定义

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| DATA_001 | 数据源连接失败 | AKShare API不可用 |
| DATA_002 | API频率超限 | 请求过于频繁 |
| DATA_003 | 数据解析失败 | 返回数据格式异常 |
| DATA_004 | 股票不存在 | 股票代码无效 |
| DATA_005 | 网络请求超时 | 请求超时 |
| DATA_006 | 数据库写入失败 | MySQL/MongoDB连接异常 |
| DATA_007 | 缓存更新失败 | Redis连接异常 |

### 7.2 Python异常处理策略

```python
import asyncio
from tenacity import retry, stop_after_attempt, wait_exponential

class DataCollectionService:
    
    @retry(
        stop=stop_after_attempt(3),
        wait=wait_exponential(multiplier=1, min=1, max=10),
        reraise=True
    )
    async def fetch_and_save_stock_list(self) -> int:
        """带重试机制的股票列表同步"""
        try:
            # 1. 获取数据
            stocks = await self._fetch_from_akshare()
            # 2. 写入数据库
            count = await self._save_to_mysql(stocks)
            return count
        except Exception as e:
            logger.error(f"同步股票列表失败: {e}")
            raise
    
    async def _save_to_mysql(self, stocks: List[Dict]) -> int:
        """保存到MySQL，带事务处理"""
        try:
            with self.db_session.begin():
                for stock in stocks:
                    # 检查是否存在
                    existing = self._get_by_code(stock['code'])
                    if existing:
                        self._update(stock)
                    else:
                        self._insert(stock)
            return len(stocks)
        except Exception as e:
            logger.error(f"写入MySQL失败: {e}")
            raise DataWriteException("DATA_006", str(e))
```

**Step 2: 实现 fetchStockList()**
```
1. 调用 akToolsClient.getStockList() 获取原始数据
2. 遍历数据，过滤 isST=false 且 isTradable=true 的股票
3. 转换字段（code, name, market, industry）
4. 批量保存到 MySQL (stockInfoMapper.saveBatch)
```

**Step 3: 实现 fetchHistoricalData()**
```
1. 检查本地MongoDB是否有足够数据
2. 如需增量，调用 akToolsClient.getKLine(stockCode, startDate, endDate)
3. 解析返回的 OHLCV 数据
4. 批量写入 MongoDB (stock_prices 集合)
5. 返回 List<StockPrices>
```

**Step 4: 实现 fetchRealTimePrice()**
```
1. 从 Redis 缓存读取 (key: "stock:quote:{code}")
2. 如缓存不存在，调用 akToolsClient.getQuote(stockCode)
3. 写入 Redis，TTL=60秒
4. 返回当前价格 Double
```

**Step 5: 实现 syncAllStocks()**
```
1. 调用 fetchStockList() 同步股票列表
2. 遍历股票列表，调用 fetchHistoricalData() 补充历史数据
3. 使用线程池并行处理（10线程）
4. 记录同步日志
```
