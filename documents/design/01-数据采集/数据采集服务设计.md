## 2. 概述

### 2.1 模块定位

数据采集模块是整个股票交易系统的数据源头，负责从外部数据源采集A股行情数据、财经新闻数据，为其他模块提供数据支撑。

### 2.2 功能摘要

- 采集全市场股票基本信息（代码、名称、行业等）
- 采集个股历史K线数据用于模型训练和历史分析
- 采集实时行情数据用于交易决策
- 采集财经新闻用于情感分析

### 2.3 上下游依赖

| 上游模块 | 接口 | 数据 |
|----------|------|------|
| 无 | - | 外部数据源 |

| 下游模块 | 接口 | 数据 |
|----------|------|------|
| LSTM预测 | getStockPrices() | 历史K线数据 |
| 情感分析 | getNews() | 财经新闻 |
| 决策引擎 | getRealtimeQuote() | 实时行情 |
| 交易执行 | getStockInfo() | 股票信息 |

### 3.2 功能详情

#### F-001: 股票列表同步

**功能描述**: 从AKTools API获取全市场股票列表，排除ST、停牌等不可交易股票，维护本地股票信息表。

**输入**: 无

**处理逻辑**:
1. 调用AKTools股票列表接口
2. 解析返回的股票数据
3. 过滤不可交易股票（ST、停牌、退市）
4. 批量更新到本地数据库

**输出**: 股票信息列表 (List<StockInfo>)

**业务规则**:
- 每日盘前同步一次
- 只保留可交易股票
- 同步后更新最新日期戳

#### F-002: 历史行情获取

**功能描述**: 获取指定股票的历史K线数据，用于模型训练和历史分析。

**输入**: 
- stockCode: 股票代码
- startDate: 开始日期
- endDate: 结束日期

**处理逻辑**:
1. 检查本地数据库是否有足够数据
2. 如需增量获取，调用AKTools K线接口
3. 解析并存储到数据库

**输出**: 历史价格列表 (List<StockPrices>)

**业务规则**:
- 优先从本地数据库读取
- 增量同步时跳过已有数据
- 保留OHLCV全部字段

#### F-003: 实时行情获取

**功能描述**: 获取个股实时行情数据，用于交易决策。

**输入**: stockCode: 股票代码

**处理逻辑**:
1. 调用AKTools实时行情接口
2. 解析返回的行情数据
3. 返回当前价格和涨跌幅

**输出**: 实时价格 (Double)

**业务规则**:
- 交易时间段内每分钟更新
- 非交易时间返回昨日收盘价

#### F-005: 财经新闻采集

**功能描述**: 从新浪财经、东方财富等平台采集财经新闻。

**输入**: stockCode: 股票代码（可选）

**处理逻辑**:
1. 并行请求多个新闻源
2. 解析新闻标题、内容、发布时间
3. 新闻去重（基于URL或标题哈希）
4. 存储到MongoDB

**输出**: 新闻列表 (List<News>)

**业务规则**:
- 每30分钟采集一次
- 保留最近7天新闻
- 去重后保留最新版本

### 5.4 AKShare集成方案

#### 5.4.1 AKShare简介

| 属性 | 说明 |
|------|------|
| 本质 | Python开源金融数据接口库 |
| 安装方式 | pip install akshare |
| 数据源 | 东方财富、新浪财经、金十数据等 |
| 授权 | 免费开源（MIT License） |

AKShare是面向Python用户的开源财经数据接口库，提供股票、期货、基金、宏观经济等数据。采用Python直接调用方式，无需额外部署HTTP服务。

#### 5.4.2 集成方案对比

| 方案 | 实现方式 | 服务数 | 优点 | 缺点 |
|------|----------|--------|------|------|
| **A: Java+AKTools HTTP** | 后端HTTP调用AKTools | 3个 | 统一架构 | 需要额外部署、调用链长 |
| **B: Python+akshare** | Python直接import | 2个 | 无需额外服务、调用快 | 数据分散在Python层 |

#### 5.4.3 推荐方案：方案B（Python直接集成）

**推荐理由**：

1. **最简部署** - 无需额外部署AKTools HTTP服务
2. **调用更快** - 直接import调用，无HTTP序列化开销
3. **架构简洁** - 减少一层服务调用链路
4. **复用依赖** - Python服务已有pandas等基础库
5. **统一维护** - 数据采集和AI分析在同一服务

**安装命令**：
```bash
pip install akshare
```

```java
public interface StockDataService {
    
    /**
     * 获取A股全市场股票列表
     * @return 可交易股票列表（排除ST、停牌等）
     */
    List<StockInfo> fetchStockList();
    
    /**
     * 获取个股历史K线数据
     * @param stockCode 股票代码
     * @param startDate 开始日期
     * @param endDate 结束日期
     * @return 历史价格列表
     */
    List<StockPrices> fetchHistoricalData(String stockCode, LocalDate startDate, LocalDate endDate);
    
    /**
     * 获取个股近N天历史数据
     * @param stockCode 股票代码
     * @param days 天数
     * @return 历史价格列表
     */
    List<StockPrices> fetchHistoricalData(String stockCode, int days);
    
    /**
     * 获取实时行情数据
     * @param stockCode 股票代码
     * @return 当前价格
     */
    Double fetchRealTimePrice(String stockCode);
    
    /**
     * 批量获取实时行情
     * @param stockCodes 股票代码列表
     * @return 股票代码到价格的映射
     */
    Map<String, Double> fetchRealTimePrices(List<String> stockCodes);
    
    /**
     * 获取财务报表数据
     * @param stockCode 股票代码
     * @return 财务数据JSON
     */
    String fetchFinancialReport(String stockCode);
}
```

### 7.2 异常处理策略

| 异常类型 | 处理策略 |
|----------|----------|
| DataSourceException | 切换备用源，重试 |
| RateLimitException | 等待后重试，加入队列缓冲 |
| ParseException | 记录日志，返回空结果 |
| NetworkException | 重试3次，失败后告警 |

```java
        log.error("获取历史数据失败, stockCode={}, days={}", stockCode, days, e);
```

#### T-004: StockDataServiceImpl实现 ⭐ 核心任务

**文件路径**: `stock-backend/src/main/java/online/mwang/stockTrading/modules/datacollection/impl/StockDataServiceImpl.java`

**状态**: 🔄 进行中

##### 实现步骤:

**Step 1: 创建实现类框架**
```java
@Service
@Slf4j
@RequiredArgsConstructor
public class StockDataServiceImpl implements StockDataService {
    
    private final StockInfoMapper stockInfoMapper;
    private final MongoTemplate mongoTemplate;
    private final RedisTemplate<String, Object> redisTemplate;
    private final AKToolsClient akToolsClient;
}
```

**Step 2: 实现 fetchStockList()**
```
1. 调用 akToolsClient.getStockList() 获取原始数据
2. 遍历数据，过滤 isST=false 且 isTradable=true 的股票
3. 转换字段（code, name, market, industry）
4. 批量保存到 MySQL (stockInfoMapper.saveBatch)
5. 返回 List<StockInfo>
```

**Step 3: 实现 fetchHistoricalData()**
```
1. 检查本地MongoDB是否有足够数据
2. 如需增量，调用 akToolsClient.getKLine(stockCode, startDate, endDate)
3. 解析返回的 OHLCV 数据
4. 批量写入 MongoDB (stock_prices 集合)
5. 返回 List<StockPrices>
```

**Step 4: 实现 fetchRealTimePrice()**
```
1. 从 Redis 缓存读取 (key: "stock:quote:{code}")
2. 如缓存不存在，调用 akToolsClient.getQuote(stockCode)
3. 写入 Redis，TTL=60秒
4. 返回当前价格 Double
```

**Step 5: 实现 syncAllStocks()**
```
1. 调用 fetchStockList() 同步股票列表
2. 遍历股票列表，调用 fetchHistoricalData() 补充历史数据
3. 使用线程池并行处理（10线程）
4. 记录同步日志
```
