## 2. 概述

### 2.1 模块定位

情感分析模块是股票交易系统的智能分析组件，负责对财经新闻进行情感分析，为综合选股模块提供情绪因子。该模块使用 FinBERT 模型（Financial BERT）进行金融领域的情感分析。

**V2.0架构更新**: 本模块采用Python定时任务架构，情感分析结果直接写入MySQL，Java服务直接从数据库查询，无需通过HTTP API调用。

### 2.1.1 什么是FinBERT

FinBERT是专门为金融文本情感分析设计的预训练Transformer模型。由ProsusAI团队开发，在大规模金融语料库（包括路透社财经新闻、10-K年报、SEC文件等）上进行预训练。

FinBERT基于BERT架构，采用双向Transformer编码器：

```
FinBERT 结构:
┌─────────────────────────────────────────┐
│              Input Text                  │
│  "贵州茅台发布财报，营收同比增长15%"      │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  Token Embedding + Position Embedding   │
│  + Segment Embedding                    │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  Transformer Encoder × 12层             │
│  (BERT-base架构)                        │
│  每层: Multi-Head Attention + FFN      │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  [CLS] Token Output                     │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  Dense + Softmax                        │
│  → positive / negative / neutral       │
└─────────────────────────────────────────┘
```

### 2.1.2 为什么选择FinBERT

金融文本情感分析的特殊性：

| 特点 | 说明 | 通用模型问题 |
|------|------|-------------|
| 专业术语 | 涨停、跌停、市盈率、市净率 | 理解不准确 |
| 语境依赖 | "增长"在财报中是正面，在风险提示中是中性 | 情感误判 |
| 数据稀缺 | 标注数据少 | 容易过拟合 |

FinBERT的优势对比：

| 模型 | 金融情感准确率 | 推理速度 |
|------|--------------|---------|
| 通用BERT | ~75% | 快 |
| FinBERT | ~90% | 快 |
| FinBERT-large | ~92% | 中等 |

### 2.1.3 本系统使用的FinBERT模型

| 属性 | 值 |
|------|-----|
| 模型名称 | ProsusAI/finbert |
| 基础架构 | bert-base-uncased |
| 参数量 | 110M |
| 层数 | 12层Transformer |
| 隐藏层维度 | 768 |
| 注意力头数 | 12 |
| 最大序列长度 | 512 tokens |
| 词表大小 | 30,522 |

### 2.1.4 模型推理流程

```
输入文本
    │
    ▼
┌─────────────────────────────┐
│ 1. 文本预处理               │
│    - 分词 (BertTokenizer)   │
│    - 添加[CLS]、[SEP]标记  │
│    - 截断或填充到512       │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 2. 模型推理                 │
│    - 输入编码为tensor       │
│    - 通过12层Transformer    │
│    - 提取[CLS]向量         │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 3. 情感分类                 │
│    - Dense层分类             │
│    - Softmax得到概率分布   │
└─────────────────────────────┘
    │
    ▼
输出: {label, score, confidence, probabilities}
```

### 3.2 功能详情

#### F-001: 定时新闻采集

**功能描述**: 每日15:30（A股收盘后）自动采集当日财经新闻数据。

**处理逻辑**:
1. 从东方财富等源获取新闻
2. 保存原始新闻到MongoDB
3. 记录任务执行日志

**执行时间**: 每日15:30

#### F-002: 定时情感分析

**功能描述**: 每日16:30自动执行情感分析任务，计算股票情感得分。

**处理逻辑**:
1. 从MongoDB获取近期新闻
2. 调用FinBERT进行批量情感分析
3. 计算每只股票的情感得分
4. 将结果写入MySQL（stock_sentiment表）
5. 记录任务执行日志

**执行时间**: 每日16:30

**数据写入表**:
- `stock_sentiment`: 股票情感得分汇总
- `news_sentiment`: 单条新闻情感详情

#### F-003: 新闻批量分析

**功能描述**: 分析新闻列表，返回带情感enriched的新闻列表。

**输入**: newsItems: 新闻列表 (List[Dict])

**处理逻辑**:
1. 批量调用FinBERT模型
2. 为每条新闻添加sentiment和sentimentScore字段
3. 返回 enriched 后的新闻列表

**输出**: 带情感标签的新闻列表

**业务规则**:
- 单次批量最大100条
- 保留原始新闻所有字段

#### F-004: 股票情感得分

**功能描述**: 计算单只股票的综合情感得分，用于选股排名。

**得分计算公式**:
```
情感得分 = Σ(news_i.sentiment × confidence_i) / Σ(confidence_i)

其中：
- sentiment: positive=1, neutral=0, negative=-1
- confidence: 模型输出置信度
```

**输入**: stockCode: 股票代码

**输出**: 情感得分 (float, -1到1)

**业务规则**:
- 无新闻时返回默认得分0
- 得分>0.3为偏正面，得分<-0.3为偏负面

## 7. 错误处理

### 7.1 错误码定义

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| SENT_001 | 模型加载失败 | FinBERT模型无法加载 |
| SENT_002 | 文本过长 | 超过512 tokens |
| SENT_003 | 文本为空 | 输入为空 |
| SENT_004 | 推理失败 | 模型推理异常 |
| SENT_005 | 新闻采集失败 | 无法获取新闻数据 |
| SENT_006 | 数据库写入失败 | MySQL写入异常 |

### 7.2 异常处理策略

| 异常类型 | 处理策略 |
|----------|----------|
| ModelLoadError | 返回默认值，记录告警 |
| TextTooLongError | 自动截断到512 tokens |
| EmptyTextError | 返回neutral |
| NewsCollectionError | 记录日志，跳过本次任务 |
| DatabaseWriteError | 重试3次，失败记录错误日志 |
| RuntimeError | 返回默认neutral，记录日志 |
