# 情感分析服务设计

## 1. 模块概述

情感分析模块负责对财经新闻进行情感分析，为选股系统提供情绪因子。该模块使用 FinBERT 模型（Financial BERT）进行金融领域的情感分析。

## 2. 技术架构

### 2.1 技术栈

| 组件 | 技术选型 | 版本 |
|------|---------|------|
| 模型 | ProsusAI/finbert | - |
| 深度学习框架 | PyTorch | 2.x |
| NLP框架 | Transformers | 4.x |
| Web框架 | FastAPI | 0.109.x |
| ASGI服务器 | Uvicorn | 0.27.x |

## 3. 服务设计

### 3.1 SentimentService 类

```python
class SentimentService:
    """FinBERT-based sentiment analysis service"""

    def __init__(self):
        self.model_name = "ProsusAI/finbert"
        self.max_length = 512
        self._model = None
        self._device = "cuda" if torch.cuda.is_available() else "cpu"

    def analyze(self, text: str) -> Dict:
        """分析单条文本情感"""
        pass

    def analyze_batch(self, texts: List[str]) -> List[Dict]:
        """批量分析文本情感"""
        pass

    def analyze_news_batch(self, news_items: List[Dict]) -> List[Dict]:
        """分析新闻批量情感"""
        pass

    def get_market_sentiment(self, news_items: List[Dict]) -> Dict:
        """计算市场整体情绪"""
        pass
```

### 3.2 核心方法

#### analyze(text)

单条文本情感分析。

| 参数 | 类型 | 说明 |
|------|------|------|
| text | str | 待分析文本 |

| 返回 | 类型 | 说明 |
|------|------|------|
| label | str | positive/negative/neutral |
| score | float | 置信度分数 (0-1) |
| confidence | float | 模型置信度 |
| probabilities | dict | 各分类概率分布 |

#### analyze_news_batch(news_items)

新闻批量分析，返回带情感 enriched 的新闻列表。

| 参数 | 类型 | 说明 |
|------|------|------|
| news_items | List[Dict] | 新闻列表，每项包含 title/content |

| 返回 | 类型 | 说明 |
|------|------|------|
| List[Dict] | Enriched新闻列表 | 包含 sentiment/sentiment_score 字段 |

#### get_market_sentiment(news_items)

计算市场整体情绪。

| 参数 | 类型 | 说明 |
|------|------|------|
| news_items | List[Dict] | 带情感的新闻列表 |

| 返回 | 类型 | 说明 |
|------|------|------|
| overall | str | overall/positive/negative |
| score | float | 加权平均分数 |
| positive_count | int | 正面新闻数量 |
| neutral_count | int | 中性新闻数量 |
| negative_count | int | 负面新闻数量 |
| total_count | int | 总新闻数量 |

## 4. 模型设计

### 4.1 模型选择

选用 ProsusAI/finbert 模型，该模型专门针对金融文本训练，在金融情感分析任务上表现优异。

### 4.2 输入输出

**输入**:
- 文本长度：最大 512 tokens
- 文本类型：财经新闻、财报、公司公告

**输出**:
- 分类：positive / negative / neutral
- 分数：0-1 之间的置信度

### 4.3 设备选择

```python
device = "cuda" if torch.cuda.is_available() else "cpu"
```

优先使用 GPU 加速推理，无 GPU 时使用 CPU。

## 5. 性能优化

### 5.1 模型缓存

- 模型采用懒加载模式，启动时不加载模型
- 首次调用时加载模型到内存/GPU
- 服务关闭时释放模型资源

### 5.2 批处理优化

- 支持批量请求，减少模型推理次数
- 单次批量最大 100 条

### 5.3 资源管理

| 资源 | 占用 |
|------|------|
| GPU显存 | ~1GB |
| 内存 | ~2GB |
| 磁盘 | ~400MB (模型) |

## 6. 错误处理

### 6.1 异常类型

| 异常 | 说明 | 处理 |
|------|------|------|
| ModelLoadError | 模型加载失败 | 返回默认值 |
| TextTooLongError | 文本超长 | 自动截断 |
| EmptyTextError | 空文本 | 返回 neutral |

### 6.2 默认返回值

```json
{
    "label": "neutral",
    "score": 0.0,
    "confidence": 0.0,
    "probabilities": {
        "positive": 0.33,
        "neutral": 0.34,
        "negative": 0.33
    }
}
```
