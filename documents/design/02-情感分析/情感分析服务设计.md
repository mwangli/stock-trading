## 2. 概述

### 2.1 模块定位

情感分析模块是股票交易系统的智能分析组件，负责对财经新闻进行情感分析，为综合选股模块提供情绪因子。该模块使用 FinBERT 模型（Financial BERT）进行金融领域的情感分析。

### 2.1.1 什么是FinBERT

FinBERT是专门为金融文本情感分析设计的预训练Transformer模型。由ProsusAI团队开发，在大规模金融语料库（包括路透社财经新闻、10-K年报、SEC文件等）上进行预训练。

FinBERT基于BERT架构，采用双向Transformer编码器：

```
FinBERT 结构:
┌─────────────────────────────────────────┐
│              Input Text                  │
│  "贵州茅台发布财报，营收同比增长15%"      │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  Token Embedding + Position Embedding   │
│  + Segment Embedding                    │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  Transformer Encoder × 12层             │
│  (BERT-base架构)                        │
│  每层: Multi-Head Attention + FFN      │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  [CLS] Token Output                     │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│  Dense + Softmax                        │
│  → positive / negative / neutral       │
└─────────────────────────────────────────┘
```

### 2.1.2 为什么选择FinBERT

金融文本情感分析的特殊性：

| 特点 | 说明 | 通用模型问题 |
|------|------|-------------|
| 专业术语 | 涨停、跌停、市盈率、市净率 | 理解不准确 |
| 语境依赖 | "增长"在财报中是正面，在风险提示中是中性 | 情感误判 |
| 数据稀缺 | 标注数据少 | 容易过拟合 |

FinBERT的优势对比：

| 模型 | 金融情感准确率 | 推理速度 |
|------|--------------|---------|
| 通用BERT | ~75% | 快 |
| FinBERT | ~90% | 快 |
| FinBERT-large | ~92% | 中等 |

### 2.1.3 本系统使用的FinBERT模型

| 属性 | 值 |
|------|-----|
| 模型名称 | ProsusAI/finbert |
| 基础架构 | bert-base-uncased |
| 参数量 | 110M |
| 层数 | 12层Transformer |
| 隐藏层维度 | 768 |
| 注意力头数 | 12 |
| 最大序列长度 | 512 tokens |
| 词表大小 | 30,522 |

### 2.1.4 模型推理流程

```
输入文本
    │
    ▼
┌─────────────────────────────┐
│ 1. 文本预处理               │
│    - 分词 (BertTokenizer)   │
│    - 添加[CLS]、[SEP]标记  │
│    - 截断或填充到512       │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 2. 模型推理                 │
│    - 输入编码为tensor       │
│    - 通过12层Transformer    │
│    - 提取[CLS]向量         │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 3. 情感分类                 │
│    - Dense层分类             │
│    - Softmax得到概率分布   │
└─────────────────────────────┘
    │
    ▼
输出: {label, score, confidence, probabilities}
```

### 3.2 功能详情

#### F-001: 单条文本分析

**功能描述**: 对单条财经文本进行情感分析，返回情感分类和得分。

**输入**: text: 待分析文本 (string)

**处理逻辑**:
1. 文本预处理（去重、截断）
2. 调用FinBERT模型进行推理
3. 解析模型输出，返回情感结果

**输出**: 
```json
{
  "label": "positive",
  "score": 0.85,
  "confidence": 0.92,
  "probabilities": {
    "positive": 0.85,
    "neutral": 0.10,
    "negative": 0.05
  }
}
```

**业务规则**:
- score范围: -1到1（negative为负，positive为正，neutral接近0）
- confidence范围: 0到1
- 空文本返回neutral

#### F-003: 新闻批量分析

**功能描述**: 分析新闻列表，返回带情感enriched的新闻列表。

**输入**: newsItems: 新闻列表 (List[Dict])

**处理逻辑**:
1. 批量调用FinBERT模型
2. 为每条新闻添加sentiment和sentimentScore字段
3. 返回 enriched 后的新闻列表

**输出**: 带情感标签的新闻列表

**业务规则**:
- 单次批量最大100条
- 保留原始新闻所有字段

#### F-004: 股票情感得分

**功能描述**: 计算单只股票的综合情感得分，用于选股排名。

**输入**: stockCode: 股票代码

**处理逻辑**:
1. 获取该股票近期新闻
2. 调用新闻批量分析
3. 按置信度加权平均计算综合得分

**输出**: 情感得分 (float, -1到1)

**业务规则**:
- 无新闻时返回默认得分0
- 得分>0.3为偏正面，得分<-0.3为偏负面

## 7. 错误处理

### 7.1 错误码定义

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| SENT_001 | 模型加载失败 | FinBERT模型无法加载 |
| SENT_002 | 文本过长 | 超过512 tokens |
| SENT_003 | 文本为空 | 输入为空 |
| SENT_004 | 推理失败 | 模型推理异常 |

### 7.2 异常处理策略

| 异常类型 | 处理策略 |
|----------|----------|
| ModelLoadError | 返回默认值，记录告警 |
| TextTooLongError | 自动截断到512 tokens |
| EmptyTextError | 返回neutral |
| RuntimeError | 返回默认neutral，记录日志 |

### 7.3 默认返回值

{

  "confidence": 0.0,
