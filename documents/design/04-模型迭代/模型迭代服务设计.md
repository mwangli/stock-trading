## 2. 概述

### 2.1 模块定位

模型迭代模块是股票交易系统的自我优化组件，负责监控模型交易表现，当收益不理想时自动触发模型重新训练和优化，确保交易策略持续有效。

### 2.2 功能摘要

- 持续监控模型交易表现
- 评估模型收益指标
- 自动/手动触发模型重训练
- 模型版本管理与回滚
- A/B测试验证优化效果

### 2.3 架构演进

#### V1.0 (旧架构 - 已废弃)
```
Java后端 → HTTP调用 → Python训练API → 模型文件
```
- 问题: 服务间耦合、响应延迟、Python服务故障影响Java

#### V2.0 (新架构 - 当前)
```
Python定时任务 → 训练模型 → 写入MySQL → Java直接查询
```
- 优势: 零HTTP调用、Java无感知、稳定性高

### 2.4 上下游依赖

| 上游模块 | 数据来源 | 说明 |

## 3. 功能需求

### 3.1 核心功能列表

| 功能ID | 功能名称 | 功能描述 | 优先级 |
|--------|----------|----------|--------|
| F-001 | 收益监控 | 持续监控模型交易表现 | P0 |
| F-002 | 模型评估 | 评估模型是否需要重训练 | P0 |
| F-003 | 模型重训练 | 使用新数据重新训练模型 | P0 |
| F-004 | 版本管理 | 管理模型版本，支持回滚 | P0 |
| F-005 | A/B测试 | 对比新旧模型效果 | P1 |

### 3.2 功能详情

#### F-001: 收益监控

**功能描述**: 持续监控模型的交易表现，记录每日盈亏、胜率等指标。

**输入**: 交易记录(MySQL)、持仓数据(MySQL)

**处理逻辑**:
1. 从MySQL查询交易记录
2. 每日收盘后汇总当日交易数据
3. 计算各项收益指标
4. 存储到MySQL表现记录表

**输出**: 收益统计报告

**业务规则**:
- 记录每日收益率
- 统计胜率（盈利次数/总交易次数）
- 计算最大回撤

#### F-002: 模型评估

**功能描述**: 评估模型当前表现，判断是否需要重训练。

**输入**: 收益监控数据(MySQL)

**处理逻辑**:
1. 从MySQL查询最近N天收益数据
2. 设定评估周期（默认30天）
3. 计算评估周期内的综合得分
4. 与阈值比较，判断是否需要重训练

**输出**: 评估结果（需要/不需要重训练）

**业务规则**:
触发重训练条件（满足任一）：
- 连续N天（如10天）收益为负
- 胜率低于阈值（如40%）
- 综合得分低于阈值（如50分）

#### F-003: 模型重训练

**功能描述**: 当评估结果需要重训练时，使用最新数据重新训练模型。

**输入**: 历史K线数据(MySQL)、新闻情感数据(MySQL)

**处理逻辑**:
1. 从MySQL收集最新训练数据（最近2年）
2. 数据预处理和特征工程
3. 重新训练LSTM/FinBERT模型
4. 使用验证集验证效果
5. 保存新模型到指定目录
6. 记录模型版本到MySQL
7. 通过验证后更新活跃模型标识

**输出**: 新模型文件

**业务规则**:
- 保留原模型作为备份
- 新模型需通过验证才能上线
- 记录训练日志和参数

#### F-004: 版本管理

**功能描述**: 管理模型版本，支持回滚和切换。

**输入**: 模型文件

**处理逻辑**:
1. 存储新模型版本到MySQL
2. 记录版本元数据（训练参数、准确率等）
3. 支持指定版本切换
4. 更新活跃模型标识

**输出**: 版本列表

**业务规则**:
- 保留最近10个版本
- 支持一键回滚
- 记录版本切换日志

#### F-005: A/B测试

**功能描述**: 对比新旧模型表现，验证优化效果。

**输入**: 两版模型、测试数据(MySQL)

**处理逻辑**:
1. 从MySQL加载历史数据
2. 使用新旧模型分别回测
3. 对比收益指标
4. 确认优化效果

**输出**: 测试报告

**业务规则**:
- 新模型需优于旧模型才能上线
- 差异需达到统计显著（如5%）

### 6.3 服务设计

#### Python Service (stock-service)

```python
# app/services/model_iteration.py
class ModelIterationService:
    """模型迭代服务 - V2.0架构"""
    
    def __init__(self):
        self.mysql = MySQLDatabase()
        self.model_storage = ModelStorage()
    
    def collect_performance_data(self, days: int = 30):
        """收集收益数据"""
        # 从MySQL查询交易记录
        trades = self.mysql.get_trades(days)
        # 计算收益指标
        stats = self.calculate_stats(trades)
        # 写入performance_records表
        self.mysql.save_performance_record(stats)
        return stats
    
    def evaluate_model(self, model_type: str) -> EvaluationResult:
        """模型评估"""
        # 查询最近收益数据
        records = self.mysql.get_performance_records(model_type, days=30)
        # 计算评估得分
        score = self.calculate_score(records)
        # 判断是否需要重训练
        need_retrain = self.check_trigger_conditions(records)
        # 保存评估结果
        result = self.mysql.save_evaluation_result(model_type, score, need_retrain)
        return result
    
    def train_model(self, model_type: str, force: bool = False) -> TrainingTask:
        """模型训练"""
        # 检查是否需要训练
        if not force:
            eval_result = self.evaluate_model(model_type)
            if not eval_result.need_retrain:
                return None
        
        # 收集训练数据
        train_data = self.mysql.get_training_data(model_type, days=730)
        
        # 执行训练
        if model_type == "lstm":
            model = self.train_lstm(train_data)
        else:
            model = self.train_finbert(train_data)
        
        # 保存模型
        version = self.model_storage.save(model, model_type)
        
        # 记录版本
        self.mysql.save_model_version(version)
        
        return version
    
    def switch_version(self, model_type: str, version_id: int):
        """切换模型版本"""
        # 更新旧版本为非活跃
        self.mysql.deactivate_versions(model_type)
        # 激活新版本
        self.mysql.activate_version(version_id)
    
    def get_active_version(self, model_type: str) -> ModelVersion:
        """获取活跃模型版本"""
        return self.mysql.get_active_version(model_type)
```

## 7. 错误处理

### 7.1 错误码定义

| 错误码 | 错误信息 | 说明 |
|--------|----------|------|
| ITER_001 | 训练数据不足 | 数据量不够训练 |
| ITER_002 | 训练失败 | 模型训练异常 |
| ITER_003 | 验证未通过 | 新模型效果不佳 |
| ITER_004 | 版本不存在 | 指定版本不存在 |
| ITER_005 | 评估数据不足 | 收益数据不够评估 |

### 7.2 异常处理策略

| 异常类型 | 处理策略 |
|----------|----------|
| 数据不足 | 记录日志，跳过本次训练 |
| 训练失败 | 保留原模型，发送告警，更新任务状态为FAILED |
| 验证未通过 | 不切换，保留原模型，记录评估结果 |
| 版本不存在 | 返回错误提示 |
