## 2. 模块划分

| 模块 | 包路径 | 职责 | 关键技术 |
|------|--------|------|----------|
| 数据采集 | com.stock.databus | 股票数据、新闻采集 | Tushare API, ROME, Jsoup |
| AI 模型 | com.stock.models | 价格预测、情感分析 | DJL, LSTM, FinBERT |
| 策略模块 | com.stock.strategy | 选股决策、信号生成 | 双因子模型，排名算法 |
| 执行模块 | com.stock.executor | 风控检查、交易执行 | 风控规则，订单管理 |
| Web 层 | com.stock.web | REST API, 前端服务 | Spring Boot, Static Files |

## 3. 核心业务流程

### 3.1 数据采集流程

```
外部数据源 (Tushare/RSS/网页)
    ↓
采集客户端 (TushareClient/RomeClient/JsoupClient)
    ↓
数据清洗和验证
    ↓
存储 (MySQL/MongoDB)
    ↓
供 AI 模型模块使用
```

### 3.2 AI 预测流程

```
K 线数据 (MongoDB) → LSTM 预测 → 涨跌概率
新闻数据 (MySQL) → 实体识别 → 情感分析 → 情感得分
    ↓
输出到策略模块
```

### 3.3 策略决策流程

```
LSTM 预测结果 + 情感分析结果
    ↓
因子计算器 (标准化)
    ↓
综合得分 = LSTM×0.6 + 情感×0.4
    ↓
股票排名 (降序排序)
    ↓
决策引擎 (生成交易信号)
    ↓
输出到执行模块
```

### 3.4 交易执行流程

```
交易信号
    ↓
风控检查 (止损/仓位/熔断)
    ↓
通过？─否─→ 拒绝
    │
    是
    ↓
执行交易 (模拟/实盘)
    ↓
更新持仓和账户
    ↓
记录交易流水
```

## 5. 部署架构

### 5.1 开发环境

```
本地运行:
- 后端：mvn spring-boot:run (Port 8080)
- 前端：npm start (Port 8000, 开发模式)
- 数据库：Docker Compose
```

### 5.2 生产环境

```
Docker Compose 部署:
- backend:8080 (前后端集成)
```

## 7. 关键设计决策

### 7.1 前后端分离

- 前端独立开发部署
- 后端提供 RESTful API
- 通过 Nginx 统一入口

### 7.2 数据库选型

- MySQL: 结构化业务数据
- MongoDB: 时序数据 (K 线)
- Redis: 缓存和会话

### 7.3 模块化设计

- 各模块职责清晰
- 模块间松耦合
- 便于独立测试和维护
