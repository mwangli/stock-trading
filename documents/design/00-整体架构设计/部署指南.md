## 2. Docker 安装

### 2.1 一键安装

```bash
# 安装 Docker Engine
curl -fsSL https://get.docker.com | sh

# 启动并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo apt install docker-compose-plugin

# 验证安装
docker --version
docker compose version
```

### 2.2 配置镜像加速

```bash
# 创建配置目录
sudo mkdir -p /etc/docker

# 配置腾讯云镜像加速
sudo tee /etc/docker/daemon.json <<EOF
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.mirrors.ustc.edu.cn",
    "https://registry.docker-cn.com"
  ],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
EOF

# 重启 Docker
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 2.3 常用命令

| 命令 | 说明 |
|------|------|
| `docker compose up -d` | 后台启动服务 |
| `docker compose down` | 停止并删除服务 |
| `docker compose ps` | 查看服务状态 |
| `docker compose logs -f` | 查看日志 |
| `docker compose restart` | 重启服务 |
| `docker compose build` | 构建镜像 |
| `docker compose pull` | 拉取镜像 |

## 3. 项目部署

### 3.1 Docker Compose 部署

```bash
# 1. 克隆项目
cd /opt
git clone <repository-url> stock-trading
cd stock-trading

# 2. 启动所有服务
docker-compose up -d --build

# 3. 查看服务状态
docker-compose ps

# 4. 查看日志
docker-compose logs -f
```

### 3.2 分阶段部署

```bash
# 1. 先启动基础设施（MySQL、Redis、MongoDB）
docker-compose up -d mysql redis mongo

# 2. 验证基础设施
docker-compose ps

# 3. 启动应用服务
docker-compose up -d backend frontend stock-service
```

### 3.3 服务配置

## 4. 数据库初始化

### 4.1 导入SQL

```bash
# 进入 MySQL 容器
docker exec -it stock-mysql mysql -u root -pRoot.123456

# 创建数据库
CREATE DATABASE stock_trading CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 导入表结构
docker exec -i stock-mysql mysql -u root -pRoot.123456 stock_trading < stock-backend/src/main/resources/sql/found_trading.sql
```

## 6. 服务管理

### 6.1 启动/停止

```bash
# 启动所有服务
docker-compose up -d

# 停止所有服务
docker-compose down

# 重启特定服务
docker-compose restart backend

# 重启所有服务
docker-compose restart
```

### 6.2 更新服务

```bash
# 拉取最新镜像
docker-compose pull

# 重新构建并启动
docker-compose up -d --build

# 重启特定服务
docker-compose up -d --no-deps backend
```

## 8. 常见问题

### 8.1 容器启动失败

```bash
# 查看错误日志
docker-compose logs <service-name>

# 检查配置文件
docker-compose config
```

### 8.2 连接被拒绝

```bash
# 检查端口占用
netstat -tlnp | grep <port>

# 检查防火墙
sudo ufw status
```

### 8.3 磁盘空间不足

```bash
# 清理未使用的镜像
docker image prune -a

# 清理构建缓存
docker builder prune

# 清理日志
sudo truncate -s 0 /var/lib/docker/containers/*/*.log
```
