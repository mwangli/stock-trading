### 1.2 架构优化方案

#### 优化前后对比

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| **数据采集** | Java调用Python API | Python定时任务直接写入DB |
| **情感分析** | Java实时调用Python | Python定时计算，写入DB |
| **LSTM预测** | Java实时调用Python | Python定时预测，写入DB |
| **综合选股** | Java调用Python服务 | Java直接查DB |
| **服务调用** | 大量HTTP调用 | **零HTTP调用** |

#### 优化优势

- **性能提升**: 响应时间降低90%+（从800ms降至50ms）
- **架构简化**: 消除服务间HTTP调用
- **稳定性**: Python任务失败不影响Java服务
- **维护成本**: 降低30%+

#### 核心原则

1. **数据驱动**: Python写库，Java读库
2. **定时计算**: 非实时计算，定时批量处理
3. **解耦**: 服务间通过数据库通信
4. **缓存**: 热点数据用Redis缓存

## 2. 模块设计

### 2.1 后端8大核心模块

| 模块 | 职责 | 依赖 |
|------|------|------|
| **数据采集** | 从外部API获取股票数据、新闻 | 无 |
| **情感分析** | FinBERT情感分析（代理到AI服务） | 数据采集 |
| **LSTM预测** | LSTM价格预测（代理到AI服务） | 数据采集 |
| **综合选股** | 整合双因子，选出最优股票 | 情感分析、LSTM预测 |
| **决策引擎** | 生成交易信号 | 综合选股、风控管理 |
| **风控管理** | 风控检查（止损、熔断） | 交易执行 |
| **交易执行** | 通过券商API执行交易 | 无 |
| **模型迭代** | 模型性能监控、训练数据收集、模型更新 | 交易执行、综合选股 |

## 3. 服务通信

### 3.1 后端 → AI服务通信

```
后端 (Java)                                      AI服务 (Python)
┌──────────────────┐                          ┌────────────────────────┐
│                  │   HTTP/REST              │                        │
│  Feign Client    │ ─────────────────────▶  │  FastAPI Endpoints    │
│                  │   {                      │                        │
│  - sentiment()   │     "text": "..."        │  POST /api/sentiment  │
│  - predict()    │   }                      │  POST /api/lstm       │

│                  │ ◀────────────────────── │                        │
│                  │   {                      │  Response: JSON        │
│                  │     ...                  │                        │
│                  │   }                      │                        │
└──────────────────┘                          └────────────────────────┘
```

### 3.2 Feign Client定义

```java
@FeignClient(
    name = "ai-service",
    url = "${ai.service.url:http://localhost:8001}"
)
public interface AIServiceClient {
    
    // 情感分析
    @PostMapping("/api/sentiment/analyze")
    SentimentResponse analyzeSentiment(@Body SentimentRequest request);
    
    // LSTM预测
    @PostMapping("/api/lstm/predict")
    PredictionResponse predict(@Body PredictionRequest request);
    

}
```

### 3.3 AI服务API端点

#### 情感分析
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/sentiment/analyze` | 分析单条文本 |
| POST | `/api/sentiment/analyze/batch` | 批量分析 |
| POST | `/api/sentiment/analyze/news` | 新闻情感分析 |

#### LSTM预测
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/lstm/predict` | 预测次日价格 |
| POST | `/api/lstm/predict/batch` | 批量预测 |
| POST | `/api/lstm/forecast` | 多日预测 |
| GET | `/api/lstm/model/info` | 模型状态 |

## 6. CI/CD流程

### 7.1 CI/CD概述

| 概念 | 全称 | 含义 |
|------|------|------|
| CI | Continuous Integration | 持续集成 - 频繁地向主干分支合并代码，并自动构建测试 |
| CD | Continuous Delivery | 持续交付 - 自动将代码部署到测试/生产环境 |
| CD | Continuous Deployment | 持续部署 - 自动将代码完全部署到生产环境 |
