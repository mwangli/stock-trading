## 2. 背景与目标

### 2.1 当前架构问题

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           当前架构 (存在以下问题)                                │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  ┌─────────────┐    HTTP调用     ┌─────────────┐                               │
│  │ Java后端    │ ◄─────────────► │ Python AI    │                               │
│  │  (8080)    │    (阻塞/超时)   │  服务(8001)  │                               │
│  │             │                  │              │                               │
│  │ 数据采集    │                  │ 数据采集     │                               │
│  │ 情感分析    │                  │ 情感分析     │                               │
│  │ LSTM预测    │                  │ LSTM预测     │                               │
│  │ 综合选股    │                  │ 训练        │                               │
│  └─────────────┘                  └─────────────┘                               │
│                                                                                 │
│  问题：                                                                      │
│  1. 服务间HTTP调用增加延迟 (~200-500ms/次)                                    │
│  2. Python服务成为单点瓶颈                                                     │
│  3. 增加了系统复杂度（需维护两个服务）                                         │
│  4. 数据需要跨服务同步                                                         │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 优化目标

| 目标 | 说明 | 预期收益 |
|------|------|----------|
| **性能提升** | 减少服务间调用延迟 | 响应时间降低 50%+ |
| **架构简化** | 减少模块间依赖 | 维护成本降低 30% |
| **解耦** | 数据驱动而非服务调用 | 系统更稳定 |
| **效率** | Python负责计算密集任务 | 训练/推理效率更高 |

## 6. 定时任务设计

### 6.1 任务调度配置

```python
# stock-service/app/services/scheduler.py

from apscheduler.schedulers.background import BackgroundScheduler
from datetime import datetime, time

scheduler = BackgroundScheduler()

def setup_scheduled_tasks():
    """配置定时任务"""
    
    # ========== 每日任务 ==========
    
    # 1. 数据采集 (每个交易日下午4点后)
    scheduler.add_job(
        daily_data_collection,
        'cron',
        hour=16, minute=30,
        id='data_collection',
        name='每日数据采集'
    )
    
    # 2. 情感分析 (数据采集后)
    scheduler.add_job(
        daily_sentiment_analysis,
        'cron',
        hour=17, minute=0,
        id='sentiment_analysis',
        name='每日情感分析'
    )
    
    # 3. LSTM预测 (情感分析后)
    scheduler.add_job(
        daily_prediction,
        'cron',
        hour=17, minute=30,
        id='lstm_prediction',
        name='每日LSTM预测'
    )
    
    # 4. 综合选股 (预测后)
    scheduler.add_job(
        daily_stock_ranking,
        'cron',
        hour=18, minute=0,
        id='stock_ranking',
        name='每日综合选股'
    )
    
    # ========== 周任务 ==========
    
    # 5. 模型训练 (每周日凌晨)
    scheduler.add_job(
        weekly_model_training,
        'cron',
        day_of_week='sun', hour=2, minute=0,
        id='model_training',
        name='每周模型训练'
    )
```

```java
    private final StockRankingRepository rankingRepository;
    
    /**
     * 获取今日推荐的股票
     * 数据已由Python写入DB，Java直接查询
     */
    public List<StockRecommendation> getTodayRecommendations() {
        LocalDate today = LocalDate.now();
        
        return rankingRepository
            .findByRankingDateAndIsRecommendedTrueOrderByRecommendedRankAsc(today)
            .stream()
            .map(this::toDTO)
            .toList();
    }
    
    /**
     * 获取股票详情（含预测和情感）
     */
    public StockDetailDTO getStockDetail(String stockCode) {
        LocalDate today = LocalDate.now();
        
        // 从不同表查询
        StockRanking ranking = rankingRepository.findByStockCodeAndRankingDate(stockCode, today);
        StockPrediction prediction = predictionRepository.findByStockCodeAndPredictDate(stockCode, today);
        StockSentiment sentiment = sentimentRepository.findByStockCodeAndAnalyzeDate(stockCode, today);
        
        return StockDetailDTO.builder()
            .ranking(ranking)
            .prediction(prediction)
            .sentiment(sentiment)
            .build();
    }
}
```

## 10. 额外优化建议

### 10.1 缓存优化

```java
@Service
@RequiredArgsConstructor
public class CachedRecommendationService {
    
    private final RedisTemplate<String, Object> redisTemplate;
    
    private static final String CACHE_KEY = "stock:recommend:today";
    private static final Duration CACHE_TTL = Duration.ofHours(4);
    
    /**
     * 缓存推荐结果，减少DB查询
     */
    public List<StockRecommendation> getCachedRecommendations() {
        // 先查缓存
        List<StockRecommendation> cached = redisTemplate.opsForValue().get(CACHE_KEY);
        if (cached != null) {
            return cached;
        }
        
        // 缓存未命中，查DB
        List<StockRecommendation> recommendations = queryFromDB();
        
        // 写入缓存
        redisTemplate.opsForValue().set(CACHE_KEY, recommendations, CACHE_TTL);
        
        return recommendations;
    }
}
```
