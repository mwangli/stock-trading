# 模块八：交易执行 - 测试用例文档

## 文档信息
- **模块**: MOD-008 交易执行
- **版本**: 2.0
- **创建日期**: 2026-02-19
- **测试类型**: 真实数据库集成测试 + 数据流穿透测试

## 1. 测试范围

### 1.1 测试目标
验证交易执行模块的所有功能是否按需求正常工作，包括：
- 买入委托执行
- 卖出委托执行
- 订单状态管理
- **订单真实写入 MySQL**
- **持仓真实更新 MySQL**
- **资金变动真实记录**

### 1.2 测试环境
- **JDK**: 17
- **测试框架**: Spring Boot Test + JUnit 5
- **数据库**: MySQL 8.0 (直连远程Docker实例)
- **缓存**: Redis (Embedded)
- **数据管理**: Database Rider

### 1.3 测试原则
| 原则 | 说明 |
|------|------|
| 真实性 | 使用真实数据库，验证数据真实写入 |
| 隔离性 | @Transactional 自动回滚，不污染环境 |
| 可追溯 | 交易订单完整记录，可审计 |
| 全链路 | 从决策到交易执行完整穿透 |

## 2. 数据流测试用例

### 2.1 交易执行数据流测试

```
数据流: 决策信号 → TradeExecutionService → MySQL (订单) + MySQL (持仓)
验证点:     V1              V2                   V3              V4
```

| 用例ID | 用例名称 | 测试步骤 | 验证点 | 数据库验证 | 预期结果 | 优先级 |
|--------|---------|---------|--------|-----------|---------|--------|
| TC-008-F001 | 买入完整流程 | 1. 执行买入<br>2. 创建订单<br>3. 更新持仓 | V1-V4 | `SELECT * FROM trade_order WHERE trade_type = 'BUY'` | 1. 订单写入<br>2. 持仓更新 | P0 |
| TC-008-F002 | 卖出完整流程 | 1. 执行卖出<br>2. 创建订单<br>3. 更新持仓 | V1-V4 | `SELECT * FROM position WHERE quantity = 0` | 1. 订单写入<br>2. 持仓清空 | P0 |

### 2.2 数据库验证示例

```java
// 验证订单写入 MySQL
assertDataExists("trade_order", "stock_code = '000001' AND trade_type = 'BUY'");

// 验证订单字段
TradeOrder order = jdbcTemplate.queryForObject(
    "SELECT * FROM trade_order WHERE stock_code = ? ORDER BY id DESC LIMIT 1",
    new BeanPropertyRowMapper<>(TradeOrder.class),
    "000001"
);
assertThat(order.getStatus()).isIn("PENDING", "FILLED");
assertThat(order.getQuantity()).isGreaterThan(0);
assertThat(order.getPrice()).isGreaterThan(0);

// 验证持仓更新
Position position = jdbcTemplate.queryForObject(
    "SELECT * FROM position WHERE stock_code = ?",
    new BeanPropertyRowMapper<>(Position.class),
    "000001"
);
assertThat(position.getQuantity()).isGreaterThan(0);
assertThat(position.getCostPrice()).isGreaterThan(0);
```
