# 模块六：决策引擎 - 测试用例文档

## 文档信息
- **模块**: MOD-006 决策引擎
- **版本**: 2.0
- **创建日期**: 2026-02-19
- **测试类型**: 真实数据库集成测试 + 数据流穿透测试

---

## 1. 测试范围

### 1.1 测试目标
验证决策引擎模块的所有功能是否按需求正常工作，包括：
- 交易信号生成（BUY/HOLD/REJECT）
- 信号置信度计算
- 决策规则执行（优先级、熔断、止损）
- **决策日志真实写入 MySQL**
- **信号真实写入 Redis 缓存**

### 1.2 测试环境
- **JDK**: 17
- **测试框架**: Spring Boot Test + JUnit 5 + Testcontainers
- **数据库**: MySQL 8.0 (Testcontainers)
- **缓存**: Redis (Embedded)
- **数据管理**: Database Rider

### 1.3 测试原则
| 原则 | 说明 |
|------|------|
| 真实性 | 使用真实数据库，验证数据真实写入 |
| 隔离性 | @Transactional 自动回滚，不污染环境 |
| 可追溯 | 决策日志完整记录，可审计 |
| 全链路 | 从选股结果到决策信号完整穿透 |

## 2. 数据流测试用例

### 2.1 决策引擎数据流测试

```
数据流: 选股结果 → DecisionEngine → MySQL (决策日志) + Redis (信号)
验证点:      V1            V2               V3                   V4
```

| 用例ID | 用例名称 | 测试步骤 | 验证点 | 数据库验证 | 预期结果 | 优先级 |
|--------|---------|---------|--------|-----------|---------|--------|
| TC-006-F001 | 决策完整数据流 | 1. 输入选股结果<br>2. 执行决策<br>3. 写入日志 | V1-V4 | `SELECT * FROM decision_log WHERE stock_code = ?` | 1. 决策生成<br>2. 日志写入MySQL | P0 |
| TC-006-F002 | 信号缓存流程 | 1. 生成信号<br>2. 写入Redis<br>3. 验证有效期 | V3-V4 | `GET decision:signal:latest` | 1. 信号缓存<br>2. TTL=当日有效 | P0 |

### 2.2 数据库验证示例

```java
// 验证决策日志写入 MySQL
assertDataExists("decision_log", "stock_code = '000001' AND signal = 'BUY'");

// 验证日志字段完整性
DecisionLog log = jdbcTemplate.queryForObject(
    "SELECT * FROM decision_log WHERE stock_code = ? ORDER BY id DESC LIMIT 1",
    new BeanPropertyRowMapper<>(DecisionLog.class),
    "000001"
);
assertThat(log.getSignal()).isNotNull();
assertThat(log.getConfidence()).isBetween(0.0, 1.0);
assertThat(log.getReason()).isNotBlank();
assertThat(log.getCreateTime()).isNotNull();

// 验证信号缓存
String signal = redisTemplate.opsForValue().get("decision:signal:latest");
assertThat(signal).isNotNull();
```
