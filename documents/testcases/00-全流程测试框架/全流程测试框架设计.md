## 1. 设计原则

### 1.1 核心理念
**真实验证，拒绝 Mock** - 所有测试必须验证数据真实写入数据库，确保从数据源到数据终点的完整链路。

### 1.2 测试原则
| 原则 | 说明 |
|------|------|
| **真实性** | 使用真实数据库，验证数据真实写入 |
| **隔离性** | 测试数据自动回滚，不污染生产环境 |
| **可追溯** | 每个测试步骤都可验证数据状态 |
| **全链路** | 从 API 到数据库完整穿透 |
| **可重复** | 测试可多次执行，结果一致 |

## 4. 测试基类设计

### 4.1 集成测试基类

```java
/**
 * 集成测试基类 - 真实数据库测试
 * 
 * 特性:
 * 1. 使用 Testcontainers 启动真实数据库容器
 * 2. @Transactional 自动回滚，不污染数据库
 * 3. 提供数据库验证工具方法
 */
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@Testcontainers
@Transactional
@ActiveProfiles("test")
@TestMethodOrder(MethodOrderer.OrderAnnotation.class)
public abstract class BaseIntegrationTest {

    // MySQL 容器
    @Container
    protected static final MySQLContainer<?> mysqlContainer = new MySQLContainer<>("mysql:8.0")
            .withDatabaseName("stock_trading_test")
            .withUsername("test")
            .withPassword("test");

    // MongoDB 容器
    @Container
    protected static final MongoDBContainer mongoContainer = new MongoDBContainer("mongo:5.0");

    // 动态配置数据源
    @DynamicPropertySource
    static void configureDatabase(DynamicPropertyRegistry registry) {
        registry.add("spring.datasource.url", mysqlContainer::getJdbcUrl);
        registry.add("spring.datasource.username", mysqlContainer::getUsername);
        registry.add("spring.datasource.password", mysqlContainer::getPassword);
        registry.add("spring.data.mongodb.uri", mongoContainer::getReplicaSetUrl);
    }

    @Autowired
    protected JdbcTemplate jdbcTemplate;

    @Autowired
    protected TestRestTemplate restTemplate;

    @Autowired
    protected MongoTemplate mongoTemplate;

    // ==================== 数据库验证工具方法 ====================

    /**
     * 验证 MySQL 表中数据存在
     */
    protected void assertDataExists(String table, String whereClause) {
        Integer count = jdbcTemplate.queryForObject(
            "SELECT COUNT(*) FROM " + table + " WHERE " + whereClause,
            Integer.class
        );
        assertThat(count).as("数据应该存在: " + whereClause).isGreaterThan(0);
    }

    /**
     * 验证 MySQL 表中数据不存在
     */
    protected void assertDataNotExists(String table, String whereClause) {
        Integer count = jdbcTemplate.queryForObject(
            "SELECT COUNT(*) FROM " + table + " WHERE " + whereClause,
            Integer.class
        );
        assertThat(count).as("数据不应该存在: " + whereClause).isEqualTo(0);
    }

    /**
     * 验证 MySQL 表记录数
     */
    protected void assertRowCount(String table, int expectedCount) {
        Integer count = jdbcTemplate.queryForObject(
            "SELECT COUNT(*) FROM " + table,
            Integer.class
        );
        assertThat(count).isEqualTo(expectedCount);
    }

    /**
     * 查询单个字段值
     */
    protected <T> T queryValue(String sql, Class<T> type, Object... args) {
        return jdbcTemplate.queryForObject(sql, type, args);
    }

    /**
     * 验证 MongoDB 集合中数据存在
     */
    protected void assertMongoDataExists(String collection, String field, Object value) {
        Query query = Query.query(Criteria.where(field).is(value));
        long count = mongoTemplate.count(query, collection);
        assertThat(count).as("MongoDB数据应该存在").isGreaterThan(0);
    }

    /**
     * 验证 MongoDB 集合记录数
     */
    protected void assertMongoRowCount(String collection, long expectedCount) {
        long count = mongoTemplate.getCollection(collection).countDocuments();
        assertThat(count).isEqualTo(expectedCount);
    }

    /**
     * 插入测试数据到 MySQL
     */
    protected void insertTestData(String table, Map<String, Object> data) {
        String columns = String.join(", ", data.keySet());
        String placeholders = data.keySet().stream().map(k -> "?").collect(Collectors.joining(", "));
        String sql = "INSERT INTO " + table + " (" + columns + ") VALUES (" + placeholders + ")";
        jdbcTemplate.update(sql, data.values().toArray());
    }

    /**
     * 等待异步操作完成
     */
    protected void awaitFor(int milliseconds) {
        try {
            Thread.sleep(milliseconds);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
        }
    }
}
```

### 4.2 端到端测试基类

```java
/**
 * 端到端测试基类 - 全链路穿透测试
 */
@SpringBootTest(webEnvironment = SpringBootTest.WebEnvironment.RANDOM_PORT)
@Testcontainers
@ActiveProfiles("test")
public abstract class BaseE2ETest {

    @Container
    protected static final MySQLContainer<?> mysqlContainer = new MySQLContainer<>("mysql:8.0")
            .withDatabaseName("stock_trading_test")
            .withInitScript("sql/init-schema.sql");

    @Container
    protected static final MongoDBContainer mongoContainer = new MongoDBContainer("mongo:5.0");

    @LocalServerPort
    protected int port;

    protected RequestSpecification requestSpec;

    @BeforeEach
    void setUp() {
        RestAssured.port = port;
        requestSpec = new RequestSpecBuilder()
                .setContentType(ContentType.JSON)
                .setAccept(ContentType.JSON)
                .build();
    }

    /**
     * 发送 GET 请求并验证响应
     */
    protected Response getAndVerify(String path, int expectedStatus) {
        return given()
                .spec(requestSpec)
                .when()
                .get(path)
                .then()
                .statusCode(expectedStatus)
                .extract()
                .response();
    }

    /**
     * 发送 POST 请求并验证响应
     */
    protected Response postAndVerify(String path, Object body, int expectedStatus) {
        return given()
                .spec(requestSpec)
                .body(body)
                .when()
                .post(path)
                .then()
                .statusCode(expectedStatus)
                .extract()
                .response();
    }

    /**
     * 验证数据库最终一致性
     */
    protected void assertEventually(Runnable assertion, Duration timeout) {
        Awaitility.await()
                .atMost(timeout)
                .untilAsserted(() -> assertion.run());
    }
}
```

## 5. 数据流测试模式

### 5.1 单模块数据流测试

```java
/**
 * 数据采集模块 - 真实数据库流测试
 */
@SpringBootTest
@Testcontainers
@Transactional
@ActiveProfiles("test")
class DataCollectionFlowTest extends BaseIntegrationTest {

    @Autowired
    private StockDataService stockDataService;

    @Autowired
    private StockInfoMapper stockInfoMapper;

    @Test
    @Order(1)
    void testFullFlow_FetchStockListAndSaveToMySQL() {
        // ========== 阶段1: 数据源获取 ==========
        log.info("=== 阶段1: 从外部API获取数据 ===");
        List<StockInfo> stockList = stockDataService.fetchStockList();
        assertThat(stockList).isNotEmpty();
        
        // ========== 阶段2: 数据处理 ==========
        log.info("=== 阶段2: 处理数据 ===");
        StockInfo testStock = stockList.get(0);
        testStock.setCreateTime(new Date());
        testStock.setDeleted("0");
        
        // ========== 阶段3: 数据写入 ==========
        log.info("=== 阶段3: 写入MySQL ===");
        int result = stockInfoMapper.insert(testStock);
        assertThat(result).isEqualTo(1);
        
        // ========== 阶段4: 数据库验证 (关键!) ==========
        log.info("=== 阶段4: 验证数据真实写入数据库 ===");
        
        // 4.1 验证记录数
        assertRowCount("stock_info", 1);
        
        // 4.2 验证数据存在
        assertDataExists("stock_info", "code = '" + testStock.getCode() + "'");
        
        // 4.3 验证字段值
        String nameInDb = queryValue(
            "SELECT name FROM stock_info WHERE code = ?", 
            String.class, 
            testStock.getCode()
        );
        assertThat(nameInDb).isEqualTo(testStock.getName());
        
        // 4.4 验证通过 Mapper 查询
        StockInfo savedStock = stockInfoMapper.getByCode(testStock.getCode());
        assertThat(savedStock).isNotNull();
        assertThat(savedStock.getId()).isNotNull();
        
        log.info("✅ 数据流测试通过: API → Service → Mapper → MySQL");
    }
}
```

## 7. 测试执行策略

### 7.1 执行顺序

```
1. 数据访问层测试 (Data Access Layer)
   └── Mapper/Repository 测试，验证 SQL 正确性

2. 模块集成测试 (Module Integration)
   └── 单模块内完整数据流测试

3. 跨模块集成测试 (Cross-Module Integration)
   └── 多模块协作测试

4. 端到端测试 (End-to-End)
   └── 完整业务流程穿透测试
```

### 7.2 测试命令

```bash
# 运行所有测试
mvn test

# 运行指定测试类
mvn test -Dtest=DataCollectionFlowTest

# 运行集成测试
mvn test -Dtest="*IntegrationTest"

# 运行端到端测试
mvn test -Dtest="*E2ETest"

# 跳过慢速测试
mvn test -DexcludedGroups=slow
```

## 8. 测试验证清单

- [ ] 数据真实写入数据库 (SELECT 验证)
- [ ] 字段值正确性验证

- [ ] 异常场景处理验证

### 8.2 全链路测试必须验证

- [ ] 数据起点：外部 API/输入参数
- [ ] 数据处理：业务逻辑转换
- [ ] 数据写入：数据库持久化
- [ ] 数据终点：查询验证结果
- [ ] 数据流转：各阶段数据一致性

## 9. 与 Mockito 的区别

| 对比项 | Mockito (Mock测试) | SpringBootTest (真实测试) |
|--------|-------------------|--------------------------|
| 数据库 | ❌ 不连接 | ✅ 真实连接 |
| SQL执行 | ❌ 不执行 | ✅ 真实执行 |
| 数据验证 | ❌ 模拟验证 | ✅ 数据库查询验证 |
| 事务测试 | ❌ 无法测试 | ✅ 真实事务 |
| 性能 | ⚡ 快 | 🐢 较慢 |
| 可靠性 | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
