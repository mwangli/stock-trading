# 模块七：风控管理 - 测试用例文档

## 文档信息
- **模块**: MOD-007 风控管理
- **版本**: 2.0
- **创建日期**: 2026-02-19
- **测试类型**: 真实数据库集成测试 + 数据流穿透测试

## 1. 测试范围

### 1.1 测试目标
验证风控管理模块的所有功能是否按需求正常工作，包括：
- 日止损检查（单日亏损≥3%）
- 月熔断检查（单月亏损≥10%）
- 卖出独立性保障
- **风控日志真实写入 MySQL**
- **风控状态真实写入 Redis**

### 1.2 测试环境
- **JDK**: 17
- **测试框架**: Spring Boot Test + JUnit 5 + Testcontainers
- **数据库**: MySQL 8.0 (Testcontainers)
- **缓存**: Redis (Embedded)
- **数据管理**: Database Rider

### 1.3 测试原则
| 原则 | 说明 |
|------|------|
| 真实性 | 使用真实数据库，验证数据真实写入 |
| 隔离性 | @Transactional 自动回滚，不污染环境 |
| 可追溯 | 风控日志永久保留，可审计 |
| 全链路 | 从交易到风控检查完整穿透 |

## 2. 数据流测试用例

### 2.1 风控检查数据流测试

```
数据流: 交易请求 → RiskControlService → MySQL (风控日志) + Redis (风控状态)
验证点:     V1              V2                    V3                   V4
```

| 用例ID | 用例名称 | 测试步骤 | 验证点 | 数据库验证 | 预期结果 | 优先级 |
|--------|---------|---------|--------|-----------|---------|--------|
| TC-007-F001 | 日止损触发流程 | 1. 模拟亏损3%<br>2. 风控检查<br>3. 验证日志 | V1-V4 | `SELECT * FROM risk_event_log WHERE event_type = 'stop_loss'` | 1. 触发止损<br>2. 日志写入MySQL | P0 |
| TC-007-F002 | 熔断触发流程 | 1. 模拟月亏损10%<br>2. 风控检查<br>3. 验证状态 | V1-V4 | `GET risk:status:circuit_breaker` | 1. 触发熔断<br>2. Redis状态更新 | P0 |

### 2.2 数据库验证示例

```java
// 验证风控日志写入 MySQL
assertDataExists("risk_event_log", "event_type = 'circuit_breaker'");

// 验证日志字段
RiskEventLog log = jdbcTemplate.queryForObject(
    "SELECT * FROM risk_event_log WHERE event_type = ? ORDER BY id DESC LIMIT 1",
    new BeanPropertyRowMapper<>(RiskEventLog.class),
    "stop_loss"
);
assertThat(log.getTriggerTime()).isNotNull();
assertThat(log.getLossPercent()).isGreaterThanOrEqualTo(3.0);

// 验证 Redis 风控状态
String status = redisTemplate.opsForValue().get("risk:status:current");
assertThat(status).contains("canBuy", "canSell");
```
