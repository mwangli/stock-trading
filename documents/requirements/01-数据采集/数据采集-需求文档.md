## 1. 引言

### 1.1 目的

本文档定义了股票交易系统数据采集模块的软件需求，为开发、测试和验收提供依据。

### 1.2 范围

数据采集模块负责从外部数据源采集A股行情数据、财经新闻数据，为情感分析、LSTM预测、Dexter分析等下游模块提供数据支撑。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| AKShare | Python开源金融数据接口库 |
| AKTools | 开源金融数据接口库（已弃用，改用AKShare直接调用） |
| OHLCV | 开盘价、最高价、最低价、收盘价、成交量 |
| K线 | 蜡烛图，展示股票价格走势 |
| ST股票 | 特别处理股票，风险较高 |

---

## 2. 业务背景

### 2.1 业务场景

股票自动交易系统需要实时获取市场行情和财经新闻，作为AI模型分析和交易决策的基础数据来源。数据质量直接影响后续分析的准确性和交易决策的可靠性。

### 2.2 业务流程

```
外部数据源 ──▶ Python数据采集服务 ──▶ 数据存储 ──▶ 下游模块
     │                │              │              │
   AKShare        定时任务        MySQL        情感分析
   新浪财经       直接写入       MongoDB       LSTM预测
   东方财富       异常处理       Redis         Dexter分析
                  增量同步
```

### 2.3 模块依赖关系

```
无前置依赖
     ↓
数据采集模块
     ↓
┌────┼────┐
↓    ↓    ↓
情感  LSTM  Dexter
分析  预测  分析
```

## 3. 功能需求

### 3.1 股票列表获取 (FR-001)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-001 |
| 需求名称 | 股票列表获取 |
| 优先级 | P0 |

**功能描述**：从 AKTools API 获取沪深两市全部股票列表，过滤 ST、退市、停牌等不可交易股票。

**输入**：无

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码（6位数字） |
| stockName | String | 股票名称 |
| market | String | 市场（SH/SZ） |
| industry | String | 所属行业 |
| isST | Boolean | 是否ST |
| isTradable | Boolean | 是否可交易 |

**业务规则**：
1. 股票代码必须为6位数字
2. ST股票标记 isST=true，不参与交易
3. 停牌股票标记 isTradable=false

---

### 3.2 历史K线数据获取 (FR-002)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-002 |
| 需求名称 | 历史K线数据获取 |
| 优先级 | P0 |

**功能描述**：获取个股历史K线数据，用于LSTM模型训练和预测。

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCode | String | 是 | 股票代码 |
| startDate | Date | 是 | 开始日期 |
| endDate | Date | 是 | 结束日期 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| tradeDate | Date | 交易日期 |
| open | Decimal | 开盘价 |
| high | Decimal | 最高价 |
| low | Decimal | 最低价 |
| close | Decimal | 收盘价 |
| volume | Decimal | 成交量 |
| change | Decimal | 涨跌幅 |

**业务规则**：
1. 日期范围不能超过3年
2. 返回数据按日期升序排列
3. 自动跳过非交易日

### 3.3 实时行情获取 (FR-003)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-003 |
| 需求名称 | 实时行情获取 |
| 优先级 | P0 |

**功能描述**：获取股票实时行情数据，支持单只和批量查询，数据缓存1分钟。

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码 |

| change | Decimal | 涨跌额 |

**业务规则**：

2. 数据延迟不超过1分钟
3. 非交易时段标记 isRealTime=false

## 7. 外部依赖

### 7.1 数据源依赖

| 接口名称 | 提供方 | 用途 | 可用性要求 |
|----------|--------|------|------------|
| AKShare | 开源Python库 | 行情数据（直接调用） | 99% |
| 新浪财经 | 新浪 | 财经新闻 | 95% |
| 东方财富 | 东财 | 财经新闻 | 95% |

### 7.2 架构变更说明

**变更前**: Java后端 → HTTP调用 → Python服务 → 返回数据 → Java写入数据库

**变更后**: Python服务 → 直接采集 → 直接写入 → MySQL/MongoDB/Redis

**优势**:
1. 减少HTTP调用链路，降低延迟
2. Python直接操作数据库，数据一致性更好
3. 避免Java层中转，减少内存占用
4. 定时任务在Python层执行，职责更清晰
