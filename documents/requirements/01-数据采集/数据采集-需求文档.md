## 1. 引言

### 1.1 目的

本文档定义了股票交易系统数据采集模块的软件需求，为开发、测试和验收提供依据。

### 1.2 范围

数据采集模块负责从外部数据源采集A股行情数据、财经新闻数据，为情感分析、LSTM预测等下游模块提供数据支撑。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| AKShare | Python开源金融数据接口库 |
| OHLCV | 开盘价、最高价、最低价、收盘价、成交量 |
| K线 | 蜡烛图，展示股票价格走势 |
| ST股票 | 特别处理股票，风险较高 |

---

## 2. 业务背景

### 2.1 业务场景

股票自动交易系统需要实时获取市场行情和财经新闻，作为AI模型分析和交易决策的基础数据来源。数据质量直接影响后续分析的准确性和交易决策的可靠性。

### 2.2 业务流程

数据采集采用Python定时任务架构。定时任务调用外部数据源获取数据后，直接写入数据库供Java后端查询使用。

### 2.3 模块依赖关系

数据采集模块无前置依赖，直接为下游模块提供数据服务。

## 3. 功能需求

### 3.1 股票列表获取 (FR-001)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-001 |
| 需求名称 | 股票列表获取 |
| 优先级 | P0 |

**功能描述**：从 AKTools API 获取沪深两市全部股票列表，过滤 ST、退市、停牌等不可交易股票。

**输入**：无

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码（6位数字） |
| stockName | String | 股票名称 |
| market | String | 市场（SH/SZ） |
| industry | String | 所属行业 |
| isST | Boolean | 是否ST |
| isTradable | Boolean | 是否可交易 |

**业务规则**：
1. 股票代码必须为6位数字
2. ST股票标记 isST=true，不参与交易
3. 停牌股票标记 isTradable=false

---

### 3.2 历史K线数据获取 (FR-002)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-002 |
| 需求名称 | 历史K线数据获取 |
| 优先级 | P0 |

**功能描述**：获取个股历史K线数据，用于LSTM模型训练和预测。

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCode | String | 是 | 股票代码 |
| startDate | Date | 是 | 开始日期 |
| endDate | Date | 是 | 结束日期 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| tradeDate | Date | 交易日期 |
| open | Decimal | 开盘价 |
| high | Decimal | 最高价 |
| low | Decimal | 最低价 |
| close | Decimal | 收盘价 |
| volume | Decimal | 成交量 |
| change | Decimal | 涨跌幅 |

**业务规则**：
1. 日期范围不能超过3年（约1095天）
2. 返回数据按日期升序排列
3. 自动跳过非交易日
4. **初始化采集**：系统首次部署时，需要对所有可交易股票进行全量历史数据采集，每只股票采集近3年数据
5. **增量同步**：日常运行每天同步最新1个交易日的数据

---

### 3.3 实时行情获取 (FR-003)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-003 |
| 需求名称 | 实时行情获取 |
| 优先级 | P0 |

**功能描述**：获取股票实时行情数据，支持单只和批量查询，数据缓存1分钟。

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCodes | List | 是 | 股票代码列表 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码 |
| currentPrice | Decimal | 当前价格 |
| change | Decimal | 涨跌额 |
| changePercent | Decimal | 涨跌幅 |
| isRealTime | Boolean | 是否实时数据 |

**业务规则**：
1. 交易时段：9:30-11:30, 13:00-15:00
2. 数据延迟不超过1分钟
3. 非交易时段标记 isRealTime=false

---

### 3.4 财经新闻采集 (FR-004)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-004 |
| 需求名称 | 财经新闻采集 |
| 优先级 | P0 |

**功能描述**：通过爬虫采集主流财经新闻网站的新闻，支持新闻去重。

**数据源**（按优先级排序）：
1. 新浪财经 (finance.sina.com.cn) - 权威财经门户，新闻量大
2. 东方财富 (eastmoney.com) - 证券财经龙头，内容丰富
3. 凤凰网财经 (finance.ifeng.com) - 综合财经媒体
4. 网易财经 (money.163.com) - 主流财经平台
5. 同花顺 (10jqka.com.cn) - 证券资讯专业

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCode | String | 否 | 股票代码（可选，不填则采集全市场新闻） |
| limit | Integer | 否 | 返回数量，默认50 |
| days | Integer | 否 | 回溯天数，默认7天 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| newsId | String | 新闻ID |
| title | String | 新闻标题 |
| content | String | 新闻内容 |
| summary | String | 新闻摘要 |
| source | String | 来源网站 |
| publishTime | DateTime | 发布时间 |
| url | String | 原文链接 |
| stockCode | String | 关联股票代码 |

**业务规则**：
1. **初始化采集**：系统首次部署时，采集近30天的全市场财经新闻
2. **增量同步**：日常运行每小时同步最新财经新闻
3. **新闻去重**：基于URL或news_id去重，避免重复采集
4. **反爬策略**：增加请求间隔、使用随机User-Agent、处理验证码
5. **数据清洗**：过滤广告、无效内容，提取正文

**采集频率**：
- 初始化：首次启动后执行一次（采集30天数据）
- 增量：每小时执行（采集最近1小时新闻）

## 6. 接口定义

### 6.1 Python服务接口（HTTP API - 数据源）

Python服务提供HTTP API接口，Java通过Feign Client调用获取数据。

| 接口路径 | 方法 | 说明 | 返回数据 |
|----------|------|------|---------|
| `/api/data/stock/list` | GET | 获取A股股票列表 | 股票基本信息列表 |
| `/api/data/stock/{stockCode}/prices` | GET | 获取历史K线数据 | OHLCV数据列表 |
| `/api/data/stock/{stockCode}/quote` | GET | 获取单只股票实时行情 | 行情数据 |
| `/api/data/quotes` | GET | 批量获取实时行情 | 行情列表 |

### 6.2 Java Feign Client接口

Java通过声明式HTTP客户端调用Python服务获取数据。接口包括：获取股票列表、获取历史K线数据、获取单只股票实时行情、批量获取实时行情。

### 6.3 Java后端服务接口

Java后端提供服务接口供其他模块调用，包括：获取并保存股票列表、获取并保存历史K线数据、获取并保存实时行情、全量同步所有股票数据。
