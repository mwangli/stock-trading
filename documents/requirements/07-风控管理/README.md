# 风控管理模块

## 模块职责

执行交易前的风控检查，包括止损、熔断等规则，保护资金安全。

## 需求列表

### 1. 风控规则需求

| 需求ID | 需求描述 | 优先级 |
|--------|----------|--------|
| RC-001 | 单日止损线：-3% | P0 |
| RC-002 | 单月熔断线：-10% | P0 |
| RC-003 | 单只股票仓位：100%（满仓） | P0 |
| RC-004 | 风控日志记录 | P1 |
| RC-005 | 熔断后自动暂停交易 | P0 |
| RC-006 | 卖出操作独立于风控（止损必须执行） | P0 |
| RC-007 | 资金不足时次日继续执行买入 | P1 |

### 2. 止损机制

| 条件 | 动作 |
|------|------|
| 单日亏损≥3% | 强制禁止新买入，但允许卖出持仓 |
| 持仓股票跌幅≥3% | 立即卖出（上午时段，优先执行） |

### 3. 熔断机制

| 条件 | 动作 |
|------|------|
| 单月亏损≥10% | 触发月熔断，暂停新买入，但允许卖出持仓 |
| 熔断恢复 | 需人工确认后恢复 |

### 4. 策略有效性

```
目标：月收益5% + 手续费2%（20次×0.1%）≈ 7%每月

盈亏平衡分析（按月）：
- 假设每月20次交易，胜率60%
- 12次盈利 × X% - 8次亏损 × X% = 7%
- X = 0.35%

结论：每次交易平均盈利0.35%即可达成月目标
```

### 5. 资金阻塞处理

```
场景：因风控导致当日无法买入

处理流程：
1. 当日执行卖出（释放资金）
2. 风控只阻止买入，不阻止卖出
3. 次日09:30继续执行卖出（如有持仓）
4. 次日13:00重新尝试买入（重新检查风控）
5. 循环直到买入成功
```

## 输入接口

```java
public interface RiskControlService {
    RiskCheckResult checkBeforeBuy();
    RiskCheckResult checkBeforeSell(Position position);
    RiskStatus getStatus();
}
```

## 输出数据格式

```json
{
    "canBuy": true,
    "canSell": true,
    "dailyLoss": -1.5,
    "monthlyLoss": -5.2,
    "riskLevel": "normal",
    "message": "风控检查通过"
}
```
