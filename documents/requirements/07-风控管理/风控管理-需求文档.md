## 1. 引言

### 1.1 目的

本文档定义了股票交易系统风控管理模块的软件需求，为开发、测试和验收提供依据。

### 1.2 范围

风控管理模块负责执行交易前的风控检查，包括止损、熔断等规则，保护资金安全。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 止损 | 当亏损达到预设阈值时，强制平仓止损 |
| 熔断 | 当亏损达到严重程度时，暂停所有买入操作 |
| 风控状态 | 系统当前的风险等级，包括正常、警告、暂停 |
| 仓位控制 | 单只股票的最大持仓比例限制 |

---

## 2. 业务背景

### 2.1 业务场景

风控管理是交易系统的安全防线，通过设置止损线、熔断线等规则，在市场异常或策略失效时保护资金安全。风控模块独立于交易决策，具有最高优先级。

### 2.2 模块依赖关系

```
MOD-001 数据采集 ──▶ MOD-007 风控管理 ──▶ MOD-006 决策引擎
                              │
                              └──▶ MOD-008 交易执行
```

## 3. 功能需求

### 3.1 日止损检查 (FR-001)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-001 |
| 需求名称 | 日止损检查 |
| 优先级 | P0 |

**功能描述**：检查当日累计亏损是否达到止损线。

**业务规则**：

| 条件 | 动作 | 说明 |
|------|------|------|
| 单日亏损 >= 3% | 禁止新买入 | 触发日止损 |
| 单日亏损 < 3% | 正常交易 | 风控通过 |

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| date | Date | 是 | 检查日期 |
| accountInfo | Object | 是 | 账户信息 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| triggered | Boolean | 是否触发止损 |
| dailyLoss | Float | 当日亏损百分比 |
| remainingQuota | Float | 剩余可亏损额度 |

---

### 3.2 月熔断检查 (FR-002)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-002 |
| 需求名称 | 月熔断检查 |
| 优先级 | P0 |

**功能描述**：检查当月累计亏损是否达到熔断线。

**业务规则**：

| 条件 | 动作 | 说明 |
|------|------|------|
| 单月亏损 >= 10% | 触发熔断 | 暂停所有买入 |
| 熔断恢复 | 人工确认 | 需管理员手动恢复 |

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| month | String | 是 | 检查月份(YYYY-MM) |
| accountInfo | Object | 是 | 账户信息 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| triggered | Boolean | 是否触发熔断 |
| monthlyLoss | Float | 当月亏损百分比 |
| canResume | Boolean | 是否可恢复交易 |

---

### 3.3 卖出独立性保障 (FR-003)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-003 |
| 需求名称 | 卖出独立性保障 |
| 优先级 | P0 |

**功能描述**：确保卖出操作不受风控限制，止损卖出必须执行。

**业务规则**：
1. 风控只限制买入，不限制卖出
2. 止损卖出优先于所有买入操作
3. 上午时段(09:30-11:30)优先执行止损卖出

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| position | Object | 是 | 持仓信息 |
| currentPrice | Float | 是 | 当前价格 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| canSell | Boolean | 是否可以卖出(始终为true) |
| needStopLoss | Boolean | 是否需要止损卖出 |
| lossPercent | Float | 当前亏损百分比 |

---

### 3.4 仓位控制 (FR-004)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-004 |

| 优先级 | P0 |

---

### 3.5 资金阻塞处理 (FR-005)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-005 |
| 需求名称 | 资金阻塞处理 |
| 优先级 | P1 |

**功能描述**：处理因风控导致的资金阻塞场景。

**处理流程**：

```
场景：因风控导致当日无法买入

处理流程：
1. 当日执行卖出（释放资金）
2. 风控只阻止买入，不阻止卖出
3. 次日09:30继续执行卖出（如有持仓）
4. 次日13:00重新尝试买入（重新检查风控）
5. 循环直到买入成功
```

## 6. 接口定义

```java
public interface RiskControlService {
    /**
     * 买入前风控检查
     * @return 风控检查结果
     */
    RiskCheckResult checkBeforeBuy();
    
    /**
     * 卖出前风控检查
     * @param position 持仓信息
     * @return 风控检查结果
     */
    RiskCheckResult checkBeforeSell(Position position);
    
    /**
     * 获取当前风控状态
     * @return 风控状态
     */
    RiskStatus getStatus();
    
    /**
     * 恢复交易（熔断后）
     * @param operator 操作人
     * @return 是否成功
     */
    boolean resumeTrading(String operator);
}

public interface StopLossService {
    /**
     * 检查止损条件
     * @param position 持仓
     * @return 是否触发止损
     */
    boolean checkStopLoss(Position position);
    
    /**
     * 执行止损卖出
     * @param stockCode 股票代码
     * @return 执行结果
     */
    StopLossResult executeStopLoss(String stockCode);
}
```

## 9. 策略有效性分析

### 9.1 收益目标

```
目标：月收益5% + 手续费2%（20次×0.1%）≈ 7%每月

盈亏平衡分析（按月）：
- 假设每月20次交易，胜率60%
- 12次盈利 × X% - 8次亏损 × X% = 7%
- X = 0.35%

结论：每次交易平均盈利0.35%即可达成月目标
```

### 9.2 风控阈值设计

| 阈值 | 设置值 | 设计依据 |
|------|--------|----------|
| 日止损 | -3% | 约束单日最大回撤 |
| 月熔断 | -10% | 保护本金安全 |
| 单只仓位 | 100% | T+1策略专注操作 |
