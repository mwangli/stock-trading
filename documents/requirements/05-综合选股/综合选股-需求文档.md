## 1. 引言

### 1.1 目的

本文档定义了股票交易系统综合选股模块的软件需求，为开发、测试和验收提供依据。

### 1.2 范围

综合选股模块负责整合情感分析、LSTM预测双因子，计算综合得分，选出最优股票。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 双因子模型 | 情感因子 + 技术因子(LSTM预测) |
| 综合得分 | 加权计算后的股票评分 |
| Top-N选股 | 选择综合得分最高的N只股票 |

---

## 2. 业务背景

### 2.1 业务场景

单一因子选股存在局限性，综合多因子可以平衡风险，提高选股准确性。系统将情感分析、技术分析两个维度融合，形成更可靠的投资决策。

### 2.2 模块依赖关系

```
MOD-002 情感分析 ──┐
                   ├──▶ MOD-004 综合选股 ──▶ MOD-005 决策引擎
MOD-003 LSTM预测 ──┘
```

## 3. 功能需求

### 3.1 双因子综合评分 (FR-001)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-001 |
| 需求名称 | 双因子综合评分 |
| 优先级 | P0 |

**功能描述**：整合情感、技术两个因子，计算综合得分。

**评分公式**：

```
综合得分 = LSTM因子得分 × 0.6 + 情感因子得分 × 0.4

各因子得分转换：
- LSTM因子：预测涨跌幅归一化到0-100分
- 情感因子：情感得分(-1~1)归一化到0-100分
```

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码 |
| overallScore | Float | 综合得分 (0-100) |
| lstmScore | Float | LSTM因子得分 |
| sentimentScore | Float | 情感因子得分 |


---

### 3.2 选股执行 (FR-002)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-002 |
| 需求名称 | 选股执行 |
| 优先级 | P0 |

**功能描述**：根据综合得分选出最优股票。

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockPool | List | 是 | 候选股票池 |
| topN | Integer | 否 | 选出数量，默认1 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| selectedStock | Object | 选出的主推股票 |
| alternatives | List | 备选股票列表(Top3) |
| analyzedAt | DateTime | 分析时间 |

**业务规则**：
1. 主推股票选择得分最高者
2. 备选股票选择得分次高的3只
3. 过滤ST、停牌股票

---

### 3.3 并行因子获取 (FR-003)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-003 |
| 需求名称 | 并行因子获取 |
| 优先级 | P0 |

**功能描述**：并行调用两个分析模块获取因子数据。

**业务规则**：
1. 使用 CompletableFuture 并行调用
2. 单个因子超时不影响整体流程
3. 设置30秒总超时时间
