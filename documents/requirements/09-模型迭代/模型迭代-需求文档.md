## 2. 概述

### 2.1 模块定位

模型迭代模块是股票交易系统的自我优化组件，负责监控模型表现，当收益不理想时自动触发模型重新训练和优化，确保交易策略持续有效。

### 2.2 功能摘要

- 监控模型交易表现
- 评估模型收益指标
- 自动触发模型重训练
- 模型版本管理
- A/B测试验证

## 3. 业务需求

### 3.1 功能需求

#### FR-001: 收益监控

**功能描述**: 持续监控模型的交易表现，记录每日盈亏、胜率等指标。

**输入**: 交易记录、持仓数据

**处理逻辑**:
1. 每日收盘后汇总当日交易数据
2. 计算各项收益指标
3. 存储历史表现数据

**输出**: 收益统计报告

**业务规则**:
- 记录每日收益率
- 统计胜率（盈利次数/总交易次数）
- 计算最大回撤

#### FR-002: 模型评估

**功能描述**: 评估模型当前表现，判断是否需要重训练。

**输入**: 收益监控数据

**处理逻辑**:
1. 设定评估周期（如30天）
2. 计算评估周期内的综合得分
3. 与阈值比较，判断是否需要重训练

**输出**: 评估结果（需要/不需要重训练）

**业务规则**:
- 连续N天收益为负
- 胜率低于阈值（如40%）
- 综合得分低于阈值（如50分）

#### FR-003: 模型重训练

**功能描述**: 当评估结果需要重训练时，使用最新数据重新训练模型。

**输入**: 历史K线数据、新闻数据

**处理逻辑**:
1. 收集最新训练数据
2. 预处理数据
3. 重新训练LSTM模型
4. 验证新模型效果

**输出**: 新模型文件

**业务规则**:
- 保留原模型作为备份
- 新模型需通过验证才能上线
- 记录训练日志和参数

#### FR-004: 模型版本管理

**功能描述**: 管理模型版本，支持回滚和切换。

**输入**: 模型文件

**处理逻辑**:
1. 存储新模型版本
2. 记录版本元数据
3. 支持指定版本切换

**输出**: 版本列表

**业务规则**:
- 保留最近N个版本
- 支持一键回滚
- 记录版本切换日志

#### FR-005: A/B测试

**功能描述**: 对比新旧模型表现，验证优化效果。

**输入**: 两版模型、测试数据

**处理逻辑**:
1. 使用历史数据回测
2. 对比收益指标
3. 确认优化效果

**输出**: 测试报告

**业务规则**:
- 新模型需优于旧模型才能上线
- 差异需达到统计显著

### 3.2 数据需求

#### DR-001: 收益记录

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 主键 |
| trade_date | Date | 交易日期 |
| daily_return | Decimal | 日收益率 |
| cumulative_return | Decimal | 累计收益率 |
| win_count | Integer | 盈利次数 |
| loss_count | Integer | 亏损次数 |
| total_trades | Integer | 总交易次数 |
| max_drawdown | Decimal | 最大回撤 |
| created_at | DateTime | 创建时间 |

#### DR-002: 模型版本

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | Long | 主键 |
| model_type | String | 模型类型(LSTM/FinBERT) |
| version | String | 版本号 |
| file_path | String | 模型文件路径 |
| train_date | Date | 训练日期 |
| accuracy | Decimal | 准确率 |
| is_active | Boolean | 是否当前活跃 |

## 4. 接口需求

### 4.1 接口列表

| 接口路径 | 方法 | 说明 |
|----------|------|------|
| /api/model/iteration/stats | GET | 获取收益统计 |
| /api/model/iteration/evaluate | POST | 触发模型评估 |
| /api/model/iteration/train | POST | 触发模型重训练 |
| /api/model/iteration/versions | GET | 获取模型版本列表 |
| /api/model/iteration/switch | POST | 切换模型版本 |
| /api/model/iteration/abtest | POST | 触发A/B测试 |

### 4.2 接口详情

**GET /api/model/iteration/stats**

响应:
```json
{
  "period": "30d",
  "total_return": 0.05,
  "win_rate": 0.55,
  "max_drawdown": 0.08,
  "total_trades": 45,
  "avg_return": 0.001
}
```

**POST /api/model/iteration/train**

请求:
```json
{
  "model_type": "lstm",
  "force": false
}
```

响应:
```json
{
  "task_id": "train_001",
  "status": "started",
  "estimated_time": "30min"
}
```
