# AI 模型模块需求

## 1. 总体需求

AI 模型模块是整个系统的核心，负责基于历史数据和市场信息，对股票未来走势进行预测，并为交易策略提供决策依据。

### 1.1. 功能范围

- **LSTM 价格预测**：利用历史 K 线数据（OHLCV），训练 LSTM 模型，预测未来 N 天的股票收盘价。
- **新闻情感分析**：对采集到的财经新闻进行情感倾向分析，输出情感评分（正面/负面/中性）。
- **模型管理**：提供模型的保存、加载、版本管理和监控功能。
- **推理服务**：提供统一的 API 接口，供策略模块调用，获取模型预测结果。

### 1.2. 性能要求

- **训练效率**：单个股票模型的完整训练时间应在 10 分钟以内。
- **推理延迟**：单次价格预测的 API 响应时间应低于 200ms。
- **情感分析吞吐量**：每秒至少能处理 10 条新闻文本。
- **系统资源**：模型训练和推理过程中的 CPU 和内存占用应在合理范围内，不能导致系统崩溃。

## 2. LSTM 价格预测需求

### 2.2. 模型输入与特征工程

#### 2.2.1. 基础数据

- **OHLCV**: 开盘价 (Open), 最高价 (High), 最低价 (Low), 收盘价 (Close), 成交量 (Volume)。
- **时间窗口**: 使用过去 60 天的日 K 线数据作为输入序列。

#### 2.2.2. 衍生特征 (Feature Engineering) - **P1 优先级**

为了提升模型性能，需要基于现有的 OHLCV 数据计算衍生技术指标，并将其作为新增特征一同输入模型。所有这些指标都可以通过现有数据计算得出。

- **动量指标**:
  - **RSI (相对强弱指数)**: 衡量股价的超买超卖状态。
  - **MACD (平滑异同移动平均线)**: 显示股价动量的变化。
- **趋势指标**:
  - **MA (移动平均线)**: 如 5日、10日、20日均线，反映短期和中期趋势。
- **波动率指标**:
  - **Bollinger Bands (布林带)**: 衡量股价的波动范围和趋势通道。
- **成交量相关指标**:
  - **OBV (能量潮)**: 结合成交量和价格变动，推断资金流向。

**实现要求**:
- **P1**: 在数据预处理阶段，增加一个特征计算步骤。
- **P1**: 将计算出的衍生特征与原始 OHLCV 数据拼接，形成一个更丰富的特征矩阵。
- **P1**: 更新模型的输入层 `inputSize` 以匹配新的特征数量。

### 2.3. 模型输出
### 2.2. 模型输出

- **预测目标**：预测未来 1 天的收盘价。

### 2.3. 模型训练

- **训练频率**：支持手动触发和每日定时（收盘后）自动触发训练。
- **训练模式**：
  - **独立模型**：为每支股票单独训练和保存一个 LSTM 模型，以捕捉其独特的波动特性。
  - **增量学习**：支持在现有模型基础上，使用最新的数据进行增量训练，而不是每次都从头开始。
- **超参数可调**：支持通过 API 或配置文件动态调整模型的关键超参数，如 `epochs`, `batch_size`, `learning_rate`。
- **早停机制** (Early Stopping)：训练过程中，若验证集损失在连续多个周期内不再显著下降，则自动停止训练，防止过拟合，并保存当前最佳模型。

### 2.4. 模型持久化

- **存储方案**：采用 MongoDB 二进制存储。
  - **MongoDB 序列化**：模型权重、网络结构和归一化参数在内存中序列化为字节数组 (`byte[] params`)，并封装为 ZIP 压缩流，直接存储于 MongoDB 的 `lstm_models` 集合。全程不涉及任何本地文件 I/O 或临时磁盘文件。
  - **MongoDB 序列化**：模型权重、网络结构和归一化参数统一序列化为二进制字节流（ZIP），存储于 MongoDB 的 `lstm_models` 集合。不使用本地磁盘进行中间转换或备份。
  - **内存缓存**：加载后的模型以字节码形式驻留在内存中，通过 `ModelCacheManager` 提供高频读取支持。
- **版本控制**：

  #BY|  - **内存快照**：支持在训练过程中捕获内存中的模型快照，确保早停机制能恢复至最佳状态，无需磁盘写入。

## 3. 新闻情感分析需求

### 3.1. 模型选择

- **预训练模型**：采用社区开源的、针对中文金融领域优化的情感分析预训练模型（如 BERT-based FinBERT）。

### 3.2. 分析逻辑

- **输入**：新闻标题和正文。
- **输出**：
  - **情感倾向**：`POSITIVE`, `NEGATIVE`, `NEUTRAL`。
  - **置信度评分**：一个 `[0, 1]` 范围内的浮点数，表示分类结果的置信度。

### 3.3. API 接口

- 提供一个 RESTful API，接收新闻文本，返回情感分析结果。

## 4. 模型管理与推理

### 4.1. 模型加载

- **动态加载**：系统能够根据请求（如预测某支股票），从 MongoDB 中动态加载对应的模型到内存中。
- **模型缓存**：对热点股票（近期频繁预测的股票）的模型进行缓存，减少重复从数据库加载的开销。

### 4.2. 推理接口

- **价格预测 API** (`/api/lstm/predict`):
  - **输入**：股票代码 (`stockCode`)，需要用于预测的最新历史数据（或由系统自动获取）。
  - **输出**：预测的次日收盘价。
- **状态监控 API** (`/api/lstm/train/status`):
  - **输入**：训练任务 ID (`trainingId`)。
  - **输出**：返回指定训练任务的实时状态，包括进度、当前 `epoch`、损失值等。
