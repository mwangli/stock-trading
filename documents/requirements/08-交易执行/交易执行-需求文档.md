## 1. 引言

### 1.1 目的

本文档定义了股票交易系统交易执行模块的软件需求，为开发、测试和验收提供依据。

### 1.2 范围

交易执行模块负责对接证券公司API，执行股票买卖操作，管理订单和持仓。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 委托单 | 向证券公司提交的买入或卖出请求 |
| 成交 | 委托单成功匹配并完成交易 |
| 持仓 | 当前账户持有的股票资产 |
| T+1 | 当日买入的股票，次日才能卖出 |

---

## 2. 业务背景

### 2.1 业务场景

交易执行模块是系统与证券市场的连接桥梁，负责将决策引擎的交易信号转化为实际的股票买卖操作。模块需要保证交易的准确性和及时性，同时管理好订单状态和持仓信息。

### 2.2 模块依赖关系

```
MOD-006 决策引擎 ──▶ MOD-008 交易执行 ──▶ 中信证券API
       │
MOD-007 风控管理 ────────┘
```

### 2.3 模块信息

| 属性 | 值 |
|------|-----|
| 模块ID | MOD-008 |
| 优先级 | P0 |
| 依赖模块 | MOD-006, MOD-007 |
| 前置条件 | 交易信号已生成，风控检查通过 |

## 3. 功能需求

### 3.1 买入执行 (FR-001)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-001 |
| 需求名称 | 买入执行 |
| 优先级 | P0 |

**功能描述**：在收盘前执行买入操作。

**业务规则**：

| 规则 | 配置 | 说明 |
|------|------|------|
| 买入时间 | 14:50-14:55 | 收盘前5分钟 |
| 买入数量 | 100股的整数倍 | A股交易规则 |
| 价格类型 | 限价委托 | 以当前卖一价委托 |

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCode | String | 是 | 股票代码 |
| quantity | Integer | 是 | 买入数量 |
| price | Float | 否 | 委托价格（默认卖一价） |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| orderId | String | 订单ID |
| status | String | 订单状态(pending/filled/cancelled) |
| message | String | 执行结果说明 |

---

### 3.2 卖出执行 (FR-002)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-002 |
| 需求名称 | 卖出执行 |
| 优先级 | P0 |

**功能描述**：执行卖出操作，包括次日卖出和止损卖出。

**业务规则**：

| 规则 | 配置 | 说明 |
|------|------|------|
| 次日卖出时间 | 14:50-14:55 | T+1策略 |
| 止损卖出时间 | 09:30-11:30 | 上午优先执行 |
| 价格类型 | 限价委托 | 以当前买一价委托 |

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCode | String | 是 | 股票代码 |
| quantity | Integer | 是 | 卖出数量 |
| reason | String | 否 | 卖出原因(normal/stop_loss) |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| orderId | String | 订单ID |
| status | String | 订单状态 |
| message | String | 执行结果说明 |

---

### 3.6 止损自动卖出 (FR-006)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-006 |
| 需求名称 | 止损自动卖出 |
| 优先级 | P0 |

**功能描述**：当持仓亏损达到止损线时，自动执行卖出。

**业务规则**：

| 条件 | 动作 |
|------|------|
| 持仓跌幅 >= 3% | 立即执行卖出 |
| 执行时间 | 09:30-14:55（上午优先） |

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| position | Object | 是 | 持仓信息 |
| triggerPrice | Float | 是 | 触发止损的价格 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| executed | Boolean | 是否执行 |
| orderId | String | 订单ID |
| reason | String | 止损原因 |

## 4. 非功能需求

### 4.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 下单响应时间 | < 2秒 | 委托单提交 |
| 查询响应时间 | < 1秒 | 持仓和资金查询 |
| 状态更新延迟 | < 3秒 | 成交状态更新 |

### 4.2 可靠性需求

| 指标 | 要求 |
|------|------|
| 交易准确性 | 100%正确执行 |
| 网络故障处理 | 自动重试3次 |
| 异常告警 | 交易失败即时告警 |

### 4.3 安全性需求

| 指标 | 要求 |
|------|------|
| API密钥保护 | 加密存储，不明文传输 |
| 交易确认 | 重要操作记录日志 |
| 异常检测 | 异常交易自动拦截 |

## 6. 接口定义

```java
public interface TradeExecutionService {
    /**
     * 执行买入
     * @param stockCode 股票代码
     * @param quantity 数量
     * @return 订单结果
     */
    OrderResult executeBuy(String stockCode, int quantity);
    
    /**
     * 执行卖出
     * @param stockCode 股票代码
     * @param quantity 数量
     * @return 订单结果
     */
    OrderResult executeSell(String stockCode, int quantity);
    
    /**
     * 获取持仓列表
     * @return 持仓列表
     */
    List<Position> getHoldingPositions();
    
    /**
     * 获取账户信息
     * @return 账户信息
     */
    AccountInfo getAccountInfo();
    
    /**
     * 查询订单状态
     * @param orderId 订单ID
     * @return 订单状态
     */
    OrderStatus queryOrderStatus(String orderId);
}

public interface BrokerApiAdapter {
    /**
     * 提交买入委托
     * @param request 委托请求
     * @return 委托结果
     */
    OrderResult submitBuyOrder(OrderRequest request);
    
    /**
     * 提交卖出委托
     * @param request 委托请求
     * @return 委托结果
     */
    OrderResult submitSellOrder(OrderRequest request);
    
    /**
     * 查询持仓
     * @return 持仓列表
     */
    List<Position> queryPositions();
    
    /**
     * 查询账户
     * @return 账户信息
     */
    AccountInfo queryAccount();
}
```
