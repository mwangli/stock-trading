## 1. 引言

### 1.1 目的

本文档定义了股票交易系统情感分析模块的软件需求，为开发、测试和验收提供依据。

### 1.2 范围

情感分析模块使用 FinBERT 模型对财经新闻进行情感分析，计算股票情感得分，为综合选股提供情绪因子。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| FinBERT | 金融领域预训练情感分析模型 |
| 情感得分 | -1到1的数值，表示新闻情感倾向 |
| 多模态 | 融合多种数据源的AI模型 |
| BERT | Bidirectional Encoder Representations from Transformers |

### 1.4 架构变更说明 (V2.0)

**优化后架构**: Python定时任务计算 → 直接写入MySQL → Java直接查库

| 对比项 | 优化前 | 优化后 |
|--------|--------|--------|
| 执行方式 | Java实时调用Python API | Python定时任务批量计算 |
| 数据存储 | Python返回JSON | 直接写入MySQL |
| Java调用 | HTTP请求 | 直接DB查询 |
| 响应时间 | ~500ms | ~10ms |

### 1.4 模型介绍

#### 1.4.1 什么是FinBERT

FinBERT是由ProsusAI团队开发的专门用于金融文本情感分析的预训练模型。它基于BERT架构，在大规模金融语料库上进行预训练，能够准确识别金融文本中的情感倾向。

FinBERT的本质是一个**文本分类模型**，将输入文本分类为三种情感标签：
- **Positive（正面）**：文本表达积极、乐观的情感
- **Negative（负面）**：文本表达消极、悲观的情感
- **Neutral（中性）**：文本不表达明显情感倾向

#### 1.4.2 为什么要用FinBERT而不是通用BERT

金融文本具有独特的词汇和表达方式，与通用文本存在显著差异：

| 对比维度 | 通用BERT | FinBERT |
|----------|---------|---------|
| 训练数据 | 通用文本 | 金融财经语料 |
| 金融术语理解 | 一般 | 深入 |
| 金融情感判断 | 准确率较低 | 准确率较高 |
| 预训练任务 | MLM | MLM + 情感分类 |

FinBERT的优势：
1. **领域适配**：在金融数据上预训练，理解"涨停"、"跌停"、"财报"、"业绩"等术语
2. **更高准确率**：金融情感分析任务上比通用BERT准确率提升10-15%
3. **即插即用**：提供预训练模型，无需额外训练

#### 1.4.3 FinBERT模型输入输出

**输入格式**：
```
输入文本: "贵州茅台发布最新财报，营收同比增长15%，超出市场预期"
最大长度: 512 tokens
```

**输出格式**：
```json
{
  "label": "positive",
  "score": 0.85,
  "confidence": 0.92,
  "probabilities": {
    "positive": 0.85,
    "neutral": 0.10,
    "negative": 0.05
  }
}
```

#### 1.4.4 本系统的FinBERT应用

本系统使用 `ProsusAI/finbert` 预训练模型，该模型特点：

| 特性 | 值 |
|------|-----|
| 模型名称 | ProsusAI/finbert |
| 基础架构 | BERT-base |
| 参数量 | ~110M |
| 最大输入长度 | 512 tokens |
| 输出类别 | positive, negative, neutral |

**得分转换**：FinBERT原生输出0-1的范围，本系统将其转换为-1到1：
- positive: score → 0到1
- negative: -score → -1到0
- neutral: 0

## 2. 业务背景

### 2.1 业务场景

财经新闻情感是影响股票价格的重要因素。正面新闻往往推动股价上涨，负面新闻则可能导致下跌。通过AI自动分析新闻情感，可以量化市场情绪，辅助交易决策。

### 2.2 业务流程

```
财经新闻 ──▶ 预处理 ──▶ FinBERT模型 ──▶ 情感得分 ──▶ 聚合计算 ──▶ 选股因子
    │            │              │              │            │
  标题/正文    分词/截断     positive/      -1~1得分    加权平均
              文本清洗     negative/                   多新闻聚合
                          neutral
```

### 2.3 模块依赖关系

```
MOD-001 数据采集
      ↓
MOD-002 情感分析
      ↓
MOD-003 LSTM预测
      ↓
MOD-004 模型迭代
      ↓
MOD-005 综合选股
```

## 3. 功能需求

### 3.1 单条文本情感分析 (FR-001)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-001 |
| 需求名称 | 单条文本情感分析 |
| 优先级 | P0 |

**功能描述**：使用 FinBERT 模型对单条财经文本进行情感分析。

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| text | String | 是 | 待分析文本（标题+正文） |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| label | String | 情感标签（positive/negative/neutral） |
| score | Float | 情感得分（0-1） |
| confidence | Float | 置信度（0-1） |
| probabilities | Map | 各分类概率分布 |

**业务规则**：
1. 文本最大长度 512 tokens，超出自动截断
2. 空文本返回 neutral，置信度 0
3. 模型加载失败返回默认值

---

### 3.2 批量新闻情感分析 (FR-002)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-002 |
| 需求名称 | 批量新闻情感分析 |
| 优先级 | P0 |

**功能描述**：批量分析多只股票的新闻情感，提高处理效率。

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| newsItems | List | 是 | 新闻列表（stockCode, title, content） |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码 |
| sentimentScore | Float | 情感得分（-1到1） |
| newsCount | Integer | 分析新闻数量 |

**业务规则**：
1. 单次批量最大 100 条新闻
2. 返回每只股票的聚合情感得分
3. 聚合方式：加权平均（权重=置信度）

---

### 3.3 股票情感得分计算 (FR-003)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-003 |
| 需求名称 | 股票情感得分计算 |
| 优先级 | P0 |

**功能描述**：计算单只股票的综合情感得分，用于选股排名。

**得分计算公式**：
```
情感得分 = Σ(news_i.sentiment × confidence_i) / Σ(confidence_i)

其中：
- sentiment: positive=1, neutral=0, negative=-1
- confidence: 模型输出置信度
```

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCode | String | 是 | 股票代码 |
| days | Integer | 否 | 分析天数，默认7天 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码 |
| sentimentScore | Float | 情感得分（-1到1） |
| positiveCount | Integer | 正面新闻数 |
| neutralCount | Integer | 中性新闻数 |
| negativeCount | Integer | 负面新闻数 |

**业务规则**：
1. 无新闻时返回默认得分 0
2. 仅统计指定天数内的新闻
3. 相同标题新闻去重

| 属性 | 描述 |
|------|------|
| 需求ID | FR-004 |
| 需求名称 | 市场整体情绪计算 |
| 优先级 | P1 |

**功能描述**：计算全市场整体情绪，用于风控判断。

**输入**：无（自动获取当日所有新闻）

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| overall | String | 整体情绪（bullish/bearish/neutral） |
| score | Float | 整体得分（-1到1） |
| positiveRatio | Float | 正面新闻占比 |
| newsCount | Integer | 分析新闻总数 |

**业务规则**：
1. 得分 > 0.3 为 bullish
2. 得分 < -0.3 为 bearish
3. 其他为 neutral

| 属性 | 描述 |

| 需求名称 | 情感分析结果写入MySQL |
| 优先级 | P0 |
