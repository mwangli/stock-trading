## 1. 引言

### 1.1 目的

本文档定义了股票交易系统决策引擎模块的软件需求，为开发、测试和验收提供依据。

### 1.2 范围

决策引擎模块负责根据综合选股结果和风控检查结果，生成最终交易信号，控制交易执行时机。

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| 交易信号 | 系统生成的买入/卖出/观望指令 |
| 信号置信度 | 决策的可信程度，范围0-1 |
| 信号有效期 | 交易信号的有效时间窗口，仅当日有效 |
| 决策因子 | 影响决策的输入因素，包括综合评分和风控状态 |

---

## 2. 业务背景

### 2.1 业务场景

决策引擎是交易系统的核心决策模块，负责将分析结果转化为可执行的交易指令。系统需要综合考虑选股评分和风控状态，在确保资金安全的前提下，最大化投资收益。

### 2.2 模块依赖关系

```

                   ├──▶ MOD-006 决策引擎 ──▶ MOD-008 交易执行
MOD-007 风控管理 ──┘
```

## 3. 功能需求

### 3.1 交易信号生成 (FR-001)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-001 |
| 需求名称 | 交易信号生成 |
| 优先级 | P0 |

**功能描述**：根据综合选股结果和风控检查结果，生成最终交易信号。

**信号类型**：

| 信号类型 | 触发条件 | 说明 |
|----------|----------|------|
| BUY | 风控正常 AND 综合评分>=阈值 | 执行买入 |
| HOLD | 风控正常 AND 综合评分<阈值 | 观望不操作 |
| REJECT | 风控异常 | 禁止买入 |

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| analysisResult | Object | 是 | 综合选股结果 |
| riskStatus | Object | 是 | 风控状态 |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| signal | String | 信号类型(buy/hold/reject) |
| stockCode | String | 目标股票代码 |
| quantity | Integer | 建议买入数量 |
| confidence | Float | 信号置信度(0-1) |
| validUntil | DateTime | 信号有效期截止时间 |
| reason | String | 决策原因说明 |

---

### 3.2 信号置信度计算 (FR-002)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-002 |
| 需求名称 | 信号置信度计算 |
| 优先级 | P1 |

**功能描述**：计算交易信号的置信度，反映决策的可靠程度。

**置信度公式**：

```
置信度 = 综合评分权重 × (综合评分/100) + 风控权重 × 风控健康度

权重配置：
- 综合评分权重：60%
- 风控权重：40%

风控健康度计算：
- 正常：1.0
- 警告：0.7
- 异常：0.0
```

---

### 3.3 决策规则执行 (FR-003)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-003 |
| 需求名称 | 决策规则执行 |
| 优先级 | P0 |

**功能描述**：执行决策规则，生成最终决策结果。

**决策规则**：

```
规则优先级（从高到低）：
1. 熔断规则：月亏损>=10% → REJECT
2. 日止损规则：日亏损>=3% → REJECT  
3. 综合评分规则：评分>=70 → BUY
4. 默认规则：其他情况 → HOLD

决策流程：
IF 风控状态 = 熔断 THEN
    生成 REJECT 信号
    reason = "触发月熔断"
ELSE IF 风控状态 = 日止损 THEN
    生成 REJECT 信号
    reason = "触发日止损"
ELSE IF 风控状态 = 正常 AND 综合评分 >= 70 THEN
    生成 BUY 信号
    reason = "综合评分达标，风控正常"
ELSE
    生成 HOLD 信号
    reason = "综合评分不足或风控警告"
```

### 3.4 信号有效期管理 (FR-004)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-004 |
| 需求名称 | 信号有效期管理 |
| 优先级 | P0 |

**功能描述**：管理交易信号的有效期，确保信号仅在当日有效。

**业务规则**：
1. 买入信号有效期：生成时间至当日14:55
2. 超过有效期的信号自动失效
3. 次日需重新生成信号

## 6. 接口定义

```java
public interface DecisionEngine {
    /**
     * 根据分析结果生成交易信号
     * @param analysis 综合选股分析结果
     * @return 交易信号
     */
    TradingSignal generateSignal(AnalysisResult analysis);
    
    /**
     * 根据选股结果做出决策
     * @param selection 选股结果
     * @return 决策结果
     */
    Decision makeDecision(StockSelectionResult selection);
    
    /**
     * 检查信号是否有效
     * @param signalId 信号ID
     * @return 是否有效
     */
    boolean isSignalValid(String signalId);
}

public interface DecisionRuleEngine {
    /**
     * 执行决策规则
     * @param context 决策上下文
     * @return 决策结果
     */
    DecisionResult executeRules(DecisionContext context);
}
```
