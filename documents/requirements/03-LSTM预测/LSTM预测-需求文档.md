## 1. 引言

### 1.1 目的

本文档定义LSTM预测模块的详细需求，使用 LSTM 神经网络预测股票价格走势，为选股系统提供技术因子。

### 1.2 范围

LSTM预测模块负责基于历史K线数据预测**下一个交易日**的收盘价格，为综合选股提供预测因子。

**设计理念**：只预测下一个交易日的价格，时间跨度最短，预测准确率最高。

### 1.3 架构变更说明 (V2.0)

**优化后架构**: Python定时任务预测 → 直接写入MySQL → Java直接查库

| 对比项 | 优化前 | 优化后 |
|--------|--------|--------|
| 执行方式 | Java实时调用Python API | Python定时任务批量预测 |
| 数据存储 | Python返回JSON | 直接写入MySQL |
| Java调用 | HTTP请求 | 直接DB查询 |
| 响应时间 | ~1000ms | ~10ms |

### 1.4 术语定义

| 术语 | 定义 |
|------|------|
| LSTM | 长短期记忆网络，适合处理时间序列数据 |
| OHLCV | 开盘价、最高价、最低价、收盘价、成交量 |
| 下一个交易日 | T+1制度下，次日可卖出的日期 |

### 1.4 模型介绍

#### 1.4.1 什么是LSTM

LSTM（Long Short-Term Memory，长短期记忆网络）是一种特殊的循环神经网络（RNN），由Hochreiter和Schmidhuber于1997年提出。LSTM通过引入"门控"机制，能够有效解决传统RNN在处理长序列时的梯度消失和梯度爆炸问题。

LSTM的核心组件：

| 组件 | 作用 |
|------|------|
| 遗忘门 (Forget Gate) | 决定哪些信息从细胞状态中丢弃 |
| 输入门 (Input Gate) | 决定哪些新信息存储到细胞状态中 |
| 输出门 (Output Gate) | 决定输出什么信息 |

#### 1.4.2 为什么要用LSTM预测股票

股票价格数据是典型的时间序列数据，具有以下特点：

1. **时序依赖性**：当前价格与历史价格存在关联
2. **非线性特征**：价格波动受多种因素影响，呈非线性变化
3. **长期记忆需求**：需要学习长期趋势和短期波动

相比传统方法，LSTM的优势：

| 对比项 | 传统统计方法 | LSTM |
|--------|-------------|------|
| 特征提取 | 人工设计 | 自动学习 |
| 非线性建模 | 能力有限 | 能力强 |
| 长序列处理 | 效果差 | 效果好 |
| 多变量融合 | 困难 | 容易 |

#### 1.4.3 本系统LSTM模型设计

本系统采用的LSTM模型设计：

```
输入层 → LSTM层 → LSTM层 → Dense层 → 输出层
(60,5)   50单元   50单元    25单元    1单元
```

| 参数 | 设计 | 理由 |
|------|------|------|
| 序列长度 | 60天 | 涵盖约3个月交易日，平衡历史信息量与噪声 |
| 特征维度 | 5维 | OHLCV（开盘、最高、最低、收盘、成交量） |
| LSTM单元数 | 50 | 足够学习复杂模式，避免过拟合 |
| Dropout | 0.2 | 防止过拟合 |
| 预测目标 | 次日收盘价 | T+1交易策略只需要次日价格 |

**为什么只预测次日价格**：
- 预测误差会随时间累积，多日预测准确率急剧下降
- 短期预测更能捕捉市场微观结构信息
- T+1制度下，次日价格是交易决策的关键信息

## 2. 业务背景

### 2.1 业务场景

股票价格存在一定的时间序列规律，通过 LSTM 神经网络学习历史价格走势，可以预测**下一个交易日**的收盘价变动趋势，辅助交易决策。

**为什么只预测下一个交易日**：
1. **准确率最高**：预测时间跨度越短，误差累积越小
2. **符合T+1制度**：A股T+1制度下，当日买入次日才能卖出
3. **实用性强**：交易决策只需要知道次日走势即可

### 2.2 模块依赖关系

```
MOD-001 数据采集
         ↓
MOD-002 情感分析
         ↓
MOD-003 LSTM预测
         ↓
MOD-004 模型迭代
         ↓
MOD-005 综合选股
```

### 2.3 模块信息

| 属性 | 值 |
|------|-----|
| 模块ID | MOD-003 |
| 优先级 | P0 |
| 依赖模块 | MOD-001 数据采集, MOD-002 情感分析 |
| 前置条件 | 历史K线数据充足(≥60天)，AI服务已启动 |
| 后置条件 | 预测结果可供选股模块使用 |

## 3. 功能需求

### 3.1 单股票价格预测 (FR-001)

| 属性 | 描述 |
|------|------|
| 需求ID | FR-001 |
| 需求名称 | 单股票价格预测 |
| 优先级 | P0 |

**功能描述**：使用 LSTM 模型预测单只股票次日价格。

**输入**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| stockCode | String | 是 | 股票代码 |
| data | List | 是 | 历史K线数据（至少60条） |

**输出**：

| 字段 | 类型 | 说明 |
|------|------|------|
| stockCode | String | 股票代码 |
| predictedPrice | Float | 预测价格 |
| currentPrice | Float | 当前价格 |
| change | Float | 预测涨跌额 |
| changePercent | Float | 预测涨跌幅 |
| direction | String | 方向 (up/down/neutral) |
| confidence | Float | 置信度 (0-1) |

**业务规则**：
1. 输入数据至少需要 60 天历史数据
2. 数据不足时返回错误信息
3. 模型未训练时使用 fallback 模式

## 6. 接口定义

### 6.1 设计原则

**只预测下一个交易日**：LSTM模型专注于预测下一个交易日的收盘价，这是准确率最高的预测方式。预测时间跨度越长，误差累积越大，因此设计只做短期预测。

### 6.2 AI 服务接口 (Python)

```
POST /api/lstm/predict          # 预测下一交易日价格
POST /api/lstm/predict/batch    # 批量预测
GET  /api/lstm/model/info       # 模型信息
```

### 6.3 后端调用接口 (Java)

```java
public interface PredictionService {
    /** 预测下一交易日价格 */
    PredictionResult predict(String stockCode);
    
    /** 批量预测 */
    List<PredictionResult> predictBatch(List<String> stockCodes);
}
```
