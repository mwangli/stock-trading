## 2. 概述

### 2.1 模块定位

Databus模块是整个股票交易系统的基础数据层，负责从外部数据源采集、清洗、存储和分发股票相关数据。

### 2.2 功能摘要

通过HTTP客户端从东方财富、Baostock等外部API获取股票列表、历史K线、实时行情和财经新闻数据，存储到MySQL和MongoDB，并提供缓存管理。

---

## 3. 业务需求

### 3.1 功能需求

#### FR-001: 股票列表采集

**功能描述**: 从东方财富API获取A股全部可交易股票列表

**输入**: 无（定时触发或手动触发）

**处理逻辑**:
1. 调用东方财富API获取股票列表
2. 过滤ST/*ST股票
3. 根据代码判断市场（6开头为上海，其他为深圳）
4. 批量保存或更新到MySQL

**输出**: 采集数量

**业务规则**:
- 过滤名称包含"ST"或"*ST"的股票
- 股票代码6开头对应市场"sh"，其他对应"sz"
- 已存在的股票执行更新操作

---

#### FR-002: 历史K线采集

**功能描述**: 获取指定股票的历史K线数据

**输入**: 
- stockCode: 股票代码
- days: 获取天数

**处理逻辑**:
1. 调用东方财富K线API
2. 解析K线数据（日期、开盘、最高、最低、收盘、成交量）
3. 保存到MongoDB

**输出**: 采集的K线数量

**业务规则**:
- 每只股票最多保留500条历史数据
- 支持日K线数据采集

---

#### FR-003: 实时行情采集

**功能描述**: 定时采集所有股票的实时行情

**输入**: 无（定时任务触发）

**处理逻辑**:
1. 获取全部可交易股票列表
2. 调用实时行情API
3. 更新MySQL中的价格和涨跌幅
4. 缓存到Redis

**输出**: 更新的股票数量

**业务规则**:
- 每5分钟执行一次
- 优先从Redis读取，Redis无数据则查询MySQL

---

#### FR-004: 财经新闻采集

**功能描述**: 采集财经新闻并关联到相关股票

**输入**: 无（定时任务触发）

**处理逻辑**:
1. 调用财经新闻API
2. 解析新闻内容（标题、来源、时间）
3. 根据新闻内容关联股票代码
4. 保存到MongoDB

**输出**: 采集的新闻数量

**业务规则**:
- 只保留最近7天的新闻
- 新闻需关联到具体股票

---

#### FR-005: 数据存储

**功能描述**: 多数据源存储管理

**输入**: 各类业务数据

**处理逻辑**:
- 股票信息 -> MySQL (stock_info表)
- K线数据 -> MongoDB (stock_prices集合)
- 新闻数据 -> MongoDB (stock_news集合)
- 热点数据 -> Redis

**输出**: 存储结果

**业务规则**:
- MySQL使用逻辑删除
- MongoDB使用日期作为分区键

## 4. 接口需求

### 4.1 接口列表

| 接口路径 | 方法 | 说明 |
|----------|------|------|
| `/api/data/stock/list` | GET | 获取股票列表 |
| `/api/data/stock/info/{code}` | GET | 获取股票详情 |
| `/api/data/stock/prices` | GET | 获取历史K线 |
| `/api/data/quote/realtime` | GET | 获取实时行情 |
| `/api/data/news` | GET | 获取财经新闻 |
| `/api/data/sync/stock-list` | POST | 手动同步股票列表 |
| `/api/data/sync/historical` | POST | 手动同步历史数据 |

| 历史数据采集 | < 10秒/股票 | 60天数据 |
| 实时行情更新 | < 5分钟 | 全量更新 |
| 接口响应时间 | < 200ms | 查询接口 |

## 6. 可用性需求

- **SLA**: 每日数据采集成功率 > 99%
- **故障处理**: 采集失败自动重试3次
- **降级策略**: API不可用时使用缓存数据

---

## 7. 安全性需求

- 请求频率限制：同一IP每分钟100次
- 数据来源验证：验证API签名
- 敏感数据脱敏：手机号、身份证号等
