# 01-数据采集需求文档

## 文档信息

| 项目 | 内容 |
|------|------|
| 模块 | 数据采集模块 (Databus) |
| 包路径 | com.stock.databus |
| 版本 | 1.0.0 |
| 创建日期 | 2024-03-01 |

## 1. 模块概述

数据采集模块是整个系统的基础数据层，负责股票数据、财经新闻的采集、存储和分发。

## 2. 功能需求

### 2.1 股票数据采集

| 需求 ID | 需求描述 | 优先级 | 验收标准 |
|--------|----------|--------|----------|
| DATABUS-REQ-001 | 从 Tushare API 获取 A 股股票列表 | P0 | 能获取所有 A 股股票基本信息 |
| DATABUS-REQ-002 | 采集股票历史 K 线数据 | P0 | 包含开高低收成交量字段 |
| DATABUS-REQ-003 | 实时获取股票价格信息 | P0 | 延迟不超过 5 分钟 |
| DATABUS-REQ-004 | 股票数据去重 | P0 | 同一股票同一交易日只保留一条记录 |
| DATABUS-REQ-005 | 数据验证和清洗 | P0 | 异常数据被过滤或标记 |

### 2.2 新闻数据采集

| 需求 ID | 需求描述 | 优先级 | 验收标准 |
|--------|----------|--------|----------|
| DATABUS-REQ-006 | 通过 RSS 源采集财经新闻 | P0 | 支持至少 3 个 RSS 源 |
| DATABUS-REQ-007 | 通过网页抓取补充新闻 | P1 | RSS 失效时能自动切换 |
| DATABUS-REQ-008 | 新闻去重 | P0 | 相同标题/内容的新闻不重复存储 |
| DATABUS-REQ-009 | 提取新闻标题、内容、发布时间 | P0 | 字段完整准确 |
| DATABUS-REQ-010 | 识别新闻中的股票代码 | P1 | 能关联到相关股票 |

### 2.3 定时任务

| 需求 ID | 需求描述 | 优先级 | 调度时间 |
|--------|----------|--------|----------|
| DATABUS-REQ-011 | 每日凌晨同步股票列表 | P0 | 01:00 |
| DATABUS-REQ-012 | 交易日收盘后同步 K 线 | P0 | 18:30 (工作日) |
| DATABUS-REQ-013 | 交易时间采集新闻 | P0 | 每 30 分钟 (9:00-15:00) |

## 3. 数据存储需求

### 3.1 MySQL 存储

| 表名 | 存储内容 | 主要字段 |
|------|----------|----------|
| stock_info | 股票基本信息 | stock_code, stock_name, list_date, industry |
| news | 新闻数据 | id, title, content, source, url, pub_time |

### 3.2 MongoDB 存储

| 集合名 | 存储内容 | 主要字段 |
|--------|----------|----------|
| stock_prices | K 线价格数据 | stock_code, trade_date, open, high, low, close, volume |

## 4. 接口需求

### 4.1 对外提供接口

```java
// 股票数据查询
StockInfo findByStockCode(String code);
List<StockInfo> findAll();

// K 线数据查询
List<StockPrice> findByStockCodeAndDateRange(String code, LocalDate start, LocalDate end);

// 新闻数据查询
List<StockNews> findByStockCode(String code);
List<StockNews> findByDateRange(LocalDate start, LocalDate end);
```

### 4.2 服务消费者

- AI 模型模块：获取新闻数据进行情感分析
- 策略模块：获取股票列表和 K 线数据用于选股

## 5. 性能需求

| 指标 | 要求 |
|------|------|
| Tushare API 调用频率 | 遵守频控 (50 次/分钟) |
| 新闻采集覆盖率 | 每 30 分钟至少采集 100 条新闻 |
| 数据存储延迟 | 采集后 5 分钟内完成存储 |
| 数据查询响应 | 95% 查询 < 200ms |

## 6. 容错需求

- 网络失败自动重试 3 次
- 单个数据源失败不影响其他采集任务
- 数据采集失败要有日志记录和告警

## 7. 依赖关系

### 7.1 外部依赖

- Tushare Pro API (需要积分)
- 新浪财经 RSS 源
- 东方财富 RSS 源

### 7.2 内部依赖

- 无 (作为基础数据层，不依赖其他业务模块)
