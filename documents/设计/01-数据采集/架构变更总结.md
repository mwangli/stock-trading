# 数据采集模块架构变更总结

## 变更概述

将数据采集模块从**Java调用Python HTTP接口**架构，改为**Python服务直接采集并写入数据库**架构。

## 架构对比

### 旧架构（已弃用）
```
外部数据源 ──▶ Java后端 ──▶ HTTP调用Python API ──▶ 返回数据 ──▶ Java写入数据库
     │                                                            │
   AKTools                                                    MySQL/MongoDB
```

**问题**：
- 调用链路长，延迟高
- Java层中转，内存占用大
- 代码复杂，维护困难

### 新架构（当前）
```
外部数据源 ──▶ Python数据采集服务 ──▶ 直接写入 ──▶ MySQL/MongoDB/Redis
     │                                                              │
   AKShare                                                    Java查询(只读)
```

**优势**：
- 减少HTTP调用链路，降低延迟
- Python直接操作数据库，数据一致性更好
- 避免Java层中转，减少内存占用
- 定时任务在Python层执行，职责更清晰
- 代码更简洁，维护更容易

## 核心功能实现

### Python层功能

1. **股票列表采集**
   - 从AKShare获取全市场股票列表
   - 过滤ST股票和不可交易股票
   - 直接写入MySQL `stock_info` 表

2. **历史K线数据采集**
   - 获取个股历史K线数据
   - 支持增量同步（检查已有数据）
   - 直接写入MongoDB `stock_prices` 集合

3. **实时行情采集**
   - 获取全市场实时行情
   - 更新Redis缓存（TTL=60秒）
   - 更新MySQL `stock_info` 表

4. **定时任务**
   - 股票列表同步：每天09:00
   - 历史数据同步：每天15:30
   - 实时行情同步：交易时段每分钟

### Java层功能（变更为只读）

1. **股票列表查询**
   - 从MySQL查询 `stock_info` 表

2. **历史数据查询**
   - 从MongoDB查询 `stock_prices` 集合

3. **实时行情查询**
   - 优先从Redis缓存读取
   - 回退到MySQL查询

## API变更

### Python新增API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/sync/stocks` | POST | 手动触发股票列表同步 |
| `/api/sync/stocks/{code}/history` | POST | 手动触发个股历史数据同步 |
| `/api/sync/quotes` | POST | 手动触发实时行情同步 |
| `/api/sync/all` | POST | 手动触发全量同步 |
| `/api/sync/status` | GET | 获取同步状态统计 |
| `/api/sync/batch/history` | POST | 批量同步历史数据 |

### Python保留API（查询）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/data/stock/list` | GET | 从MySQL查询股票列表 |
| `/api/data/stock/{code}` | GET | 从MySQL查询单只股票详情 |
| `/api/data/stock/{code}/prices` | GET | 从MongoDB查询历史价格 |
| `/api/data/stock/{code}/quote` | GET | 从Redis/MySQL查询实时行情 |
| `/api/data/quotes` | GET | 查询多只股票实时行情 |

### Java保留API（查询）

| 接口 | 说明 |
|------|------|
| `queryStockList()` | 从MySQL查询股票列表 |
| `queryHistoricalData()` | 从MongoDB查询历史数据 |
| `queryRealTimePrice()` | 从Redis/MySQL查询实时价格 |
| `triggerDataSync()` | 触发Python服务同步（可选） |

## 配置变更

### Python服务配置（app/core/config.py）

```python
# 数据库配置
MYSQL_HOST = "localhost"
MYSQL_PORT = 3306
MYSQL_DATABASE = "stock_trading"
MONGODB_URL = "mongodb://localhost:27017"
REDIS_HOST = "localhost"

# 采集配置
DATA_COLLECTION_ENABLED = True
DATA_COLLECTION_CRON_STOCK_LIST = "0 0 9 * * *"      # 每天09:00
DATA_COLLECTION_CRON_HISTORICAL = "0 30 15 * * *"    # 每天15:30
DATA_COLLECTION_CRON_REALTIME = "0 0/1 9-15 * * *"   # 每分钟
DATA_COLLECTION_HISTORY_DAYS = 60
DATA_COLLECTION_BATCH_SIZE = 100
DATA_COLLECTION_THREAD_POOL_SIZE = 10
```

## 迁移指南

### 1. 部署步骤

```bash
# 1. 安装Python依赖
cd stock-service
pip install -r requirements.txt

# 2. 配置数据库连接
# 编辑 app/core/config.py 或设置环境变量

# 3. 启动Python服务
uvicorn app.main:app --reload --host 0.0.0.0 --port 8001

# 4. 验证数据采集
# 访问 http://localhost:8001/health
# 访问 http://localhost:8001/api/sync/status
```

### 2. Java后端适配

- 无需修改业务代码
- 查询接口保持不变
- 可以逐步替换 `fetchXxx()` 为 `queryXxx()`

### 3. 数据库迁移

- 无需迁移数据
- Python服务会创建必要的表和索引
- 可以逐步删除Java端废弃的数据表

## 注意事项

1. **Python服务必须先启动**，确保数据采集正常进行
2. **数据库连接配置**必须正确，确保Python能写入数据
3. **定时任务**由Python APScheduler管理，Java Quartz定时任务已弃用
4. **监控**重点从Java层转移到Python层

## 回滚方案

如需回滚到旧架构：

1. 恢复Java端的AKToolsClient调用
2. 禁用Python定时任务
3. 启用Java Quartz定时任务
4. 恢复旧的数据采集流程
