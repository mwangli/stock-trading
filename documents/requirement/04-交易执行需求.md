# 04-交易执行需求文档

## 文档信息

| 项目 | 内容 |
|------|------|
| 模块 | 交易执行模块 (Executor) |

## 1. 模块概述

执行模块是系统的交易执行层，负责风控检查、订单执行和持仓管理。

## 3. 风控规则

### 3.1 止损规则

| 规则名 | 阈值 | 说明 |
|--------|------|------|
| 日亏损限制 | 3% | 当日累计亏损超过 3% 禁止买入 |
| 月亏损限制 | 10% | 当月累计亏损超过 10% 禁止买入 |
| 个股止损 | 8% | 单只股票亏损超过 8% 触发卖出 |
| 总止损 | 15% | 总账户亏损超过 15% 清仓 |

### 3.2 仓位规则

| 规则名 | 阈值 | 说明 |
|--------|------|------|
| 单股仓位上限 | 30% | 单只股票市值不超过总资产 30% |
| 总仓位上限 | 80% | 股票总市值不超过总资产 80% |
| 最低仓位 | 20% | 保持至少 20% 现金 |
| 行业集中度 | 50% | 同一行业股票不超过 50% |

### 3.3 交易限制

| 规则名 | 限制 | 说明 |
|--------|------|------|
| 单笔最小金额 | 1000 元 | 低于此金额不执行 |
| 单日交易次数 | 10 次 | 超过次数禁止交易 |
| T+1 规则 | 强制 | 当日买入次日才能卖出 |
| 涨跌停限制 | 强制 | 涨停不买入，跌停不卖出 |

## 4. 接口需求

```java
// 风控检查接口
RiskCheckResult checkRisk(TradeRequest request);

// 交易执行接口
TradeResult executeBuy(TradeRequest request);
TradeResult executeSell(TradeRequest request);

// 持仓查询接口
Position getPosition(String stockCode);
List<Position> getAllPositions();

// 账户查询接口
Account getAccount();

// 订单查询接口
Order getOrder(String orderId);
List<Order> getOrdersByStatus(OrderStatus status);
```

## 5. 性能需求

| 指标 | 要求 |
|------|------|
| 风控检查响应 | < 50ms |
| 订单执行响应 | < 200ms |
| 持仓更新延迟 | < 1 秒 |
| 支持并发 | 支持 10 个并发请求 |
