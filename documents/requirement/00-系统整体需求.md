## 1. 项目概述

### 1.1 项目背景

AI 股票自动交易系统是基于 LSTM 神经网络和财经新闻情感分析的智能交易决策系统，旨在通过 AI 技术辅助股票投资决策，实现自动化交易。

### 1.2 系统目标

- **智能化**: 利用 AI 模型预测股票价格走势和分析市场情绪
- **自动化**: 自动采集数据、生成交易信号、执行交易
- **可视化**: 提供直观的 Dashboard 展示数据和分析结果
- **安全可控**: 完善的风控机制确保交易安全

### 1.3 用户群体

- 个人投资者
- 量化交易爱好者
- 金融分析师

## 3. 非功能需求

### 3.1 性能要求

| 指标 | 要求 |
|------|------|
| 数据采集延迟 | K 线数据收盘后 1 小时内完成 |
| 新闻采集频率 | 交易时间每 30 分钟采集一次 |
| AI 预测完成时间 | 每日收盘后 2 小时内完成所有股票预测 |
| 策略计算时间 | 选股计算在 10 分钟内完成 |
| API 响应时间 | 95% 请求响应时间 < 500ms |

### 3.2 可用性要求

- 系统可用性 ≥ 99%(工作日交易时间)
- 数据准确性 ≥ 99.9%
- 支持失败重试机制

### 3.3 安全要求

- 敏感数据加密存储
- API 密钥安全存储，不提交到代码库
- 交易操作需要风控检查
- 模拟/实盘环境隔离

### 3.4 可维护性要求

- 代码符合规范，有完整的单元测试
- 有详细的日志记录和错误追踪
- 支持热配置更新
- 模块化设计，便于扩展

## 4. 系统约束

### 4.1 技术约束

- 后端：Java 17 + Spring Boot 3.2
- 前端：React 18 + TypeScript 5
- 数据库：MySQL 8.0 + MongoDB 6.0 + Redis 7.x
- AI 框架：Deep Java Library (DJL) + PyTorch

### 4.2 业务约束

- 仅支持 A 股市场
- 遵守 T+1 交易规则
- 遵守涨跌停限制
- 遵守交易时间 (工作日 9:30-11:30, 13:00-15:00)

### 4.3 数据源约束

- Tushare API 有调用频率限制 (根据积分)
- RSS 源可能失效，需要备用方案
- 新闻数据存在时效性
