# 02-AI 模型模块 - 设计文档

## 模块概述

AI 模型模块（`com.stock.modelService`）是系统的智能分析层，负责股票价格预测和财经新闻情感分析，为交易决策提供 AI 支持。

## 核心职责

1. **LSTM 价格预测** - 基于历史 K 线数据预测次日股票价格走势
2. **情感分析推理** - 分析财经新闻对股票的情感倾向 (利好/利空)
3. **股票实体识别** - 识别新闻中提及的股票 (代码/简称/全称)
4. **模型管理** - 模型加载、版本管理、热切换
5. **预测服务** - 为策略模块提供预测结果和置信度

## 核心组件

### 1. StockLSTM (LSTM 网络模型)

- **职责**: 定义 LSTM 神经网络结构
- **输入**: 过去 N 日的 K 线数据 (开高低收量)
- **输出**: 次日涨跌概率
- **特性**: 纯 Java 实现，DJL 框架加载

### 2. LstmInference (LSTM 推理服务)

- **职责**: 加载 LSTM 模型并执行价格预测推理
- **输入**: 股票代码
- **输出**: PredictionResult(涨跌概率，置信度)
- **流程**: 数据获取 → 特征工程 → 模型推理 → 结果输出

### 3. SentimentInference (情感分析服务)

- **职责**: 分析财经新闻的情感倾向
- **模型**: FinBERT (中文金融情感分析模型)
- **输入**: 新闻文本 + 股票代码
- **输出**: SentimentResult(情感得分，利好/利空)

### 4. EntityRecognizer (股票实体识别器)

- **职责**: 识别新闻文本中提及的股票
- **策略**:
  - 股票代码匹配 (6 位数字)
  - 股票全称匹配
  - 股票简称匹配
  - 曾用名匹配
- **输出**: 股票 ID 列表

### 5. NewsProcessor (新闻处理流水线)

- **职责**: 处理新闻的完整流水线
- **流程**:
  1. 新闻文本清洗
  2. 句子分割
  3. 股票实体识别
  4. 句子级情感分析
  5. 股票级情感聚合
  6. 输出结果

## 数据流程

### LSTM 价格预测流程

```
MongoDB (K 线) → 数据预处理 → 特征标准化 → LSTM 模型 → 结果后处理 → 预测结果
                      ↓
               过去 60 日 K 线              涨跌概率
               开高低收量                 置信度
                                         预测价格
```

### 新闻情感分析流程

```
新闻文本 → 文本清洗 → 句子分割 → 实体识别 → 句子级情感 → 聚合 → 股票级情感
                                        ↓
                                 FinBERT 模型
                                 利好/利空/中性
```

### 多股票新闻处理

```
单条新闻 (涉及多股票)
    ↓
句子分割 (拆分为独立语义单元)
    ↓
每句实体识别 (识别提及的股票)
    ↓
句子级情感分析 (针对特定股票)
    ↓
股票级情感聚合 (加权平均)
    ↓
输出: 每只股票的情感得分
```

## 模型设计

### LSTM 网络结构

```
输入层 (60 时间步 × 6 特征)
    ↓
LSTM 层 (128 隐藏单元)
    ↓
Dropout (0.3)
    ↓
全连接层 (64 单元)
    ↓
ReLU 激活
    ↓
输出层 (2 单元：涨/跌概率)
    ↓
Softmax
```

### 特征工程

**输入特征 (6 维):**
1. 开盘价 (归一化)
2. 最高价 (归一化)
3. 最低价 (归一化)
4. 收盘价 (归一化)
5. 成交量 (归一化)
6. 涨跌幅 (%)

**归一化方法:**
- 价格特征：除以基准价 (首日收盘价)
- 成交量：对数变换后归一化
- 涨跌幅：直接使用百分比

### 情感分析模型

**模型选择**: FinBERT (中文金融领域 BERT)

**输入格式:**
```
[CLS] 新闻文本 [SEP]
```

**输出:**
- 利好概率 (0-1)
- 利空概率 (0-1)
- 中性概率 (0-1)

**情感得分计算:**
```
score = 利好概率 - 利空概率
score > 0.1: 利好
score < -0.1: 利空
其他：中性
```

- **数值计算**: ND4J
- **工具类**: Hutool, Commons Math

## 关键设计决策

### 1. 纯 Java 实现

- **选择**: 使用 DJL 而非 Python 服务
- **原因**: 
  - 简化部署 (无需 Python 环境)
  - 降低延迟 (无进程间通信)
  - 统一技术栈 (全 Java)

### 2. 句子级情感分析

- **问题**: 一条新闻可能涉及多只股票，情感可能相反
- **方案**: 先句子级分析，再股票级聚合
- **优势**: 准确捕捉每只股票的情感倾向

### 3. 模型与代码分离

- **设计**: 模型文件独立于代码
- **优势**:
  - 支持模型热更新
  - 版本管理方便
  - 可以 A/B 测试不同模型

### 4. 置信度输出

- **设计**: 除了预测结果，还输出置信度
- **用途**: 策略模块可以根据置信度调整权重
- **计算**: 基于概率分布的熵或 margin

## FinBERT2 中文金融情感分析模型

### 模型介绍

**FinBERT2**是一个专门针对中文金融领域优化的预训练语言模型，具有以下特点：

- **模型类型**: 中文金融领域专用预训练BERT
- **训练语料**: 320亿Token的中文金融语料，包括研报、新闻、公告等
- **任务性能**: 在金融情感分析任务上F1-score达到0.895，显著优于普通BERT模型
- **应用场景**: 金融新闻情绪分析、财务报告情感判断、公告语调分析等

### 模型优势

相比之前的hfl/chinese-bert-wwm-ext模型，FinBERT2具备以下显著优势：

1. **领域专业化**: 在320亿Tokens的金融语料上专门预训练，更好地理解金融术语和语境
2. **更高性能**: 在金融情感分析任务上的F1-score高达0.895，相比之前的模型有明显提升
3. **准确性更强**: 金融领域的专业化使模型能够更精准地识别情感偏向

### 配置参数

在`application.yml`中已更新配置以最优支持FinBERT2：
```yaml
models:
  sentiment:
    pretrained-model: "valuesimplex-ai-lab/FinBERT2-base"  # 采用FinBERT2-base
    model-path: "models/sentiment-analysis"
    max-sequence-length: 256        # 支持更长的文本序列
    epochs: 3                     # FinBERT2预训练充分，微调 epochs 减少
    batch-size: 16
    learning-rate: 5e-6           # 使用更小学习率进行微调
    accuracy-threshold: 0.85      # 设定较高精度阈值
    use-early-stopping: true
    early-stopping-patience: 3
    warmup-ratio: 0.1
    max-checkpoints: 5
    labels: ["负面", "中性", "正面"]
```

### 1. 配置管理 (`SentimentTrainingConfig`)
- 位置：`backend/src/main/java/com/stock/models/config/SentimentTrainingConfig.java`
- 功能：集中管理情感分析训练的所有超参数
- 配置项：

### 自动标注规则
基于情感关键词匹配：

**正面关键词**（+1 分）：
- 增长、盈利、上涨、突破、利好、推荐、买入、收益、业绩、向好、创新高、大涨、飙升、激增、超预期、优秀、强劲、复苏、回暖

**负面关键词**（-1 分）：
- 下跌、亏损、风险、减持、利空、卖出、警告、业绩下滑、暴跌、缩水、下滑、衰退、恶化、承压、下调、警惕、低迷、疲软、危机

**标注规则**：
- 正面分 > 负面分 + 1 → 正面 (label=2)
- 负面分 > 正面分 + 1 → 负面 (label=0)
- 其他 → 中性 (label=1)

## LSTM 模型训练功能

本次实现为 AI 股票交易系统添加了完整的 LSTM 模型训练功能，支持使用历史股票数据训练价格预测模型。

### 1. 配置管理 (`LstmTrainingConfig`)
- 位置：`backend/src/main/java/com/stock/models/config/LstmTrainingConfig.java`
- 功能：集中管理 LSTM 训练的所有超参数
- 配置项：
  - `model-path`: 模型保存路径
  - `sequence-length`: 时间序列长度（默认 60 天）
  - `hidden-size`: 隐藏层大小（默认 50）
  - `num-layers`: LSTM 层数（默认 2）
  - `epochs`: 训练轮次（默认 100）
  - `batch-size`: 批次大小（默认 32）
  - `learning-rate`: 学习率（默认 0.001）
  - `train-ratio`: 训练集比例（默认 0.8）

### 2. 数据预处理 (`LstmDataPreprocessor`)
- 位置：`backend/src/main/java/com/stock/models/service/LstmDataPreprocessor.java`
- 功能：
  - 从 MongoDB 获取股票历史价格数据
  - 特征提取：开盘价、最高价、最低价、收盘价、成交量（5 个特征）
  - 时间序列构建：滑动窗口方式创建训练样本
  - Min-Max 归一化：将数据缩放到 [0, 1] 范围
  - 支持多只股票联合训练

### 3. 模型训练服务 (`LstmTrainerService`)
- 位置：`backend/src/main/java/com/stock/models/service/LstmTrainerService.java`
- 功能：
  - 训练流程编排
  - 训练进度追踪
  - 模型保存与管理
  - 支持参数动态调整（epochs, batch size, learning rate）
  - 输出：训练损失、验证损失、模型路径、训练样本数等

### 4. 推理组件增强 (`LstmInference`)
- 位置：`backend/src/main/java/com/stock/models/inference/LstmInference.java`
- 新增功能：
  - 动态模型加载/重新加载
  - 模型状态查询
  - 上次加载时间追踪

### 5. REST API 控制器 (`LstmTrainingController`)
- 位置：`backend/src/main/java/com/stock/web/controller/LstmTrainingController.java`
- 接口：
  - `POST /api/models/lstm/train` - 启动训练
