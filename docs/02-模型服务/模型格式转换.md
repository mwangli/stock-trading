# 情感分析模型格式转换指南（djl-convert）

## 1. 背景与问题

情感分析模块使用 DJL 加载 Hugging Face 的 FinBERT 模型 (`yiyanghkust/finbert-tone-chinese`)。Hugging Face 默认提供 safetensors 格式，DJL 需 TorchScript（`.pt`）格式。使用官方 **djl-convert** 工具可一键完成转换。

---

## 2. DJL 模型格式要求

| 文件 | 说明 |
|------|------|
| `sentiment.pt` | TorchScript 模型权重 |
| `tokenizer.json` | 分词器配置 |
| `config.json` | 模型配置 |
| `serving.properties` | DJL 服务配置 |

目录结构（项目根目录 `models/sentiment/`）：

```
models/sentiment/
├── sentiment.pt
├── tokenizer.json
├── config.json
└── serving.properties
```

---

## 3. 使用 djl-convert 转换

### 3.1 安装

```bash
pip install https://publish.djl.ai/djl_converter
```

需 Python 3.8+ 及 PyTorch / transformers 环境。若未安装，可先执行：

```bash
pip install torch transformers
```

### 3.2 转换命令

在项目根目录执行：

```bash
djl-convert -m yiyanghkust/finbert-tone-chinese -o models/sentiment -t text-classification
```

| 参数 | 说明 |
|------|------|
| `-m` | Hugging Face 模型 ID |
| `-o` | 输出目录 |
| `-t` | 任务类型，情感分析用 `text-classification` |

工具会从 Hugging Face 下载模型并转换为 TorchScript，输出到指定目录。

### 3.3 验证与调整

转换完成后检查：

```bash
ls models/sentiment/
# 应包含: *.pt, tokenizer.json, config.json, serving.properties
```

若输出的 `.pt` 文件名不是 `sentiment.pt`，可二选一：

- 重命名：`mv models/sentiment/*.pt models/sentiment/sentiment.pt`
- 或编辑 `serving.properties`，将 `option.modelName` 改为实际文件名（不含 `.pt`）

确保 `serving.properties` 内容为：

```properties
engine=PyTorch
option.modelName=sentiment
```

---

## 4. 网络与离线

- **需联网**：djl-convert 会从 Hugging Face 下载模型，首次运行需可访问 `huggingface.co`
- **超时**：大模型下载可能较慢，可设置 `HF_ENDPOINT` 或使用代理
- **离线**：若已有本地模型目录，可尝试 `-m /path/to/local/model`（视 djl-convert 版本支持情况）

---

## 5. 常见问题

### 5.1 转换失败或模型不可用

部分 Hugging Face 模型与 djl-convert 存在兼容性问题。若转换后 DJL 加载失败，可参考 [djl-convert 已知问题](https://github.com/deepjavalibrary/djl/issues/3518)，或使用手动 Python 脚本转换（见项目历史文档）。

### 5.2 DJL 找不到 .pt 文件

- 确认 `serving.properties` 中 `option.modelName` 与 `.pt` 文件名一致（不含扩展名）
- 确认 `models.sentiment.model-path` 指向 `models/sentiment`

### 5.3 推理结果异常

- 检查标签映射：`neutral=0, positive=1, negative=2`
- 确认 tokenizer 与模型训练时一致

## 6. 配置与验证

### application.yml

```yaml
models:
  sentiment:
    model-path: "models/sentiment"
    model-source: "local"
    download-pretrained: false
```

### 接口验证

- 健康检查：`GET /api/models/sentiment/health`
- 重新加载：`POST /api/models/sentiment/reload`

`modelLoaded=true` 表示模型已成功加载。
