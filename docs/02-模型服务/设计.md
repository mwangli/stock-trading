## 4. 模型准确率定量评估体系

### 4.1 评估指标

| 指标 | 计算方式 | 合格阈值 | 触发微调阈值 |
|------|----------|----------|--------------|
| 准确率 (Accuracy) | (TP+TN) / (TP+TN+FP+FN) | ≥ 70% | < 70% |
| F1 分数 | 2×Precision×Recall / (P+R) | ≥ 60% | < 60% |
| 方向准确率 | 预测方向与实际方向一致率 | ≥ 65% | < 65% |
| ROC-AUC | 曲线下面积 | ≥ 0.70 | < 0.70 |
| 夏普比率 | (Rp - Rf) / σp | ≥ 0.5 | < 0.5 |

### 4.2 测试数据集划分

| 数据集 | 比例 | 用途 |
|--------|------|------|
| 训练集 | 70% | 模型微调 |
| 验证集 | 15% | 超参调优 |
| 测试集 | 15% | 最终评估 |

划分原则：时序划分（避免未来数据泄露）

### 4.3 评估触发机制

```
触发条件（满足任一）：
├── 定时触发（每日收盘后）
├── 手动触发（API调用）
├── 准确率 < 70%
├── F1 < 60%
├── 方向准确率 < 65%
├── 连续3周期准确率下降
└── 夏普比率 < 0.5
```

## 5. 收益率定量验证方案

### 5.1 收益率计算

```
收益率 = (卖出价 - 买入价) / 买入价 × 100%

说明：
- 不扣除交易费用（佣金、印花税）
- 仅反映价格波动收益
- T日买入信号执行价格 vs T+1卖出执行价格
```

### 5.2 统计周期

| 周期 | 指标 |
|------|------|
| 日收益 | 单日收益率 |
| 周收益 | 周收益率、周夏普比率 |
| 月收益 | 月收益率、月夏普比率 |
| 累计收益 | 累计收益率、年化收益率 |

### 5.3 风险调整指标

| 指标 | 计算公式 | 评判标准 |
|------|----------|----------|
| 夏普比率 | (Rp - Rf) / σp | < 0.5 较差，≥ 1.0 良好 |
| 最大回撤 | max(Peak - Trough) / Peak | > 20% 高风险 |

## 6. 自动标注管道

### 6.1 标注规则

| T+1 收益率 | 标注标签 |
|------------|----------|
| > 1% | 利好 (Positive) |
| -1% ~ 1% | 中性 (Neutral) |
| < -1% | 利空 (Negative) |

### 6.2 标注数据质量

- 置信度过滤：仅保留置信度 ≥ 0.6 的标注
- 来源验证：仅使用 T+1 实际成交的反馈
- 去重处理：相同文本仅保留一条标注

### 6.3 数据存储

```json
{
  "text": "原文内容",
  "label": "positive|neutral|negative",
  "source": "trading_feedback",
  "confidence": 0.85,
  "stockCode": "600519",
  "return": 2.5,
  "createdAt": "2026-03-29T16:00:00Z",
  "status": "pending|validated|used|discarded"
}
```

## 7. 模型微调闭环

### 7.1 闭环流程

```
交易执行 → 反馈数据 → 自动标注 → 训练数据存储 → 微调训练 → 模型上线
    ↑                                                              │
    └──────────────────── 评估触发 ◀──────────────────────────────┘
```

### 7.2 触发条件

满足以下任一条件时触发微调：
- 准确率 < 70%
- F1 < 60%
- 方向准确率 < 65%
- 连续 3 个评估周期准确率下降
- 夏普比率 < 0.5

触发微调还需满足：
- 标注样本数 ≥ 500 条
- 模型版本已稳定 7 天以上

### 7.3 回滚机制

触发回滚条件：
- 微调后准确率下降 > 5%
- 微调后夏普比率下降 > 20%
- 连续 3 天模型效果恶化

## 10. 关键设计决策

### 10.1 收益率计算简化

- 不扣除交易费用，仅反映价格波动收益
- 便于与 LSTM 预测结果直接对比分析

### 10.2 自动标注为主

- 利用交易反馈自动生成标注，无需人工标注成本
- 标注数据经置信度过滤，保证质量

### 10.3 复用现有架构

- 利用现有 JobSchedulerService + MongoDB
- 无新增外部服务依赖
