## 2. 当前阻塞问题：手机验证页滑块不弹出

### 2.1 问题现象

调用 `POST /api/browser/phone/send-code`（点击「获取验证码」按钮）后：
- `GET /api/browser/phone/slider-status` 返回 `hasSlider=false`
- 滑块验证码未被检测到

### 2.2 代码层面分析

查看当前 `CaptchaService.detectSliderType()` 实现：

```java
// 当前实现 —— 仅在当前 frame 上下文中查找
public SliderType detectSliderType(WebDriver driver) {
    if (!driver.findElements(By.cssSelector("[class*='yidun']")).isEmpty()) {
        return SliderType.YIDUN;
    }
    return SliderType.NONE;
}
```

**问题**：
1. 点击「获取验证码」时，代码在 **iframe 内** 操作
2. 滑块弹出后，`.yidun` 元素可能挂载在 **顶层 document** 上
3. `findElements` 只在当前 frame 中搜索，**无法跨 frame 检测**

查看 `BrowserSessionManager.ensureLoginFrame()` 实现：

```java
// 当前实现 —— 通过查找 password 输入框定位 iframe
if (driver.findElements(By.xpath("//input[@type='password']")).size() > 0) {
    return true;
}
```

**问题**：手机验证页（activePhone.html）**没有 password 输入框**，所以 `ensureLoginFrame()` 可能无法正确切入手机验证页的 iframe。

### 2.3 修复方案

#### 修复点 1：`ensureLoginFrame()` 增加手机验证页支持

```java
/**
 * 切换到表单所在 iframe（支持两种页面类型）
 * - login.html: 查找包含 password 输入框的 iframe
 * - activePhone.html: 查找包含"获取验证码"按钮的 iframe
 */
public boolean ensureLoginFrame() {
    driver.switchTo().defaultContent();

    // 检查顶层是否包含目标元素
    if (hasLoginFormElements(driver)) {
        log.info("登录表单在顶层 document");
        return true;
    }

    // 递归遍历所有 iframe
    return findLoginFrame(driver, 0);
}

/**
 * 判断当前 frame 是否包含登录/验证表单元素
 */
private boolean hasLoginFormElements(WebDriver driver) {
    // 标准登录页：有 password 输入框
    if (!driver.findElements(By.xpath("//input[@type='password']")).isEmpty()) {
        return true;
    }
    // 手机验证页：有"获取验证码"按钮或"手机号"输入框
    if (!driver.findElements(By.xpath(
        "//*[contains(text(), '获取验证码')] | " +
        "//input[contains(@placeholder, '手机号')]"
    )).isEmpty()) {
        return true;
    }
    return false;
}
```

#### 修复点 2：`detectSliderType()` 增加跨 frame 检测

```java
/**
 * 检测滑块验证码类型（跨 frame 搜索）
 *
 * 策略：
 * 1. 先在当前 frame 查找
 * 2. 切到顶层 document 查找
 * 3. 遍历顶层所有 iframe 查找
 */
public SliderType detectSliderType(WebDriver driver) {
    // 记录当前 frame 上下文
    String currentUrl = driver.getCurrentUrl();

    // 策略 1: 当前 frame 查找
    if (hasYidunElement(driver)) {
        log.info("在当前 frame 中检测到网易云盾滑块");
        return SliderType.YIDUN;
    }

    // 策略 2: 切到顶层查找
    driver.switchTo().defaultContent();
    if (hasYidunElement(driver)) {
        log.info("在顶层 document 中检测到网易云盾滑块");
        return SliderType.YIDUN;
    }

    // 策略 3: 遍历顶层 iframe 查找
    List<WebElement> iframes = driver.findElements(By.tagName("iframe"));
    for (int i = 0; i < iframes.size(); i++) {
        try {
            driver.switchTo().defaultContent();
            driver.switchTo().frame(i);
            if (hasYidunElement(driver)) {
                log.info("在第 {} 个 iframe 中检测到网易云盾滑块", i);
                return SliderType.YIDUN;
            }
        } catch (Exception e) {
            log.debug("切换到第 {} 个 iframe 失败", i);
        }
    }

    log.debug("未检测到滑块验证码");
    return SliderType.NONE;
}

private boolean hasYidunElement(WebDriver driver) {
    return !driver.findElements(By.cssSelector("[class*='yidun']")).isEmpty();
}
```

#### 修复点 3：滑块操作前确保在正确的 frame 上下文

```java
/**
 * 切换到滑块所在的 frame 上下文
 * 返回 true 表示成功找到并切换
 */
public boolean switchToSliderFrame(WebDriver driver) {
    // 先检查当前 frame
    if (hasYidunElement(driver)) return true;

    // 切到顶层
    driver.switchTo().defaultContent();
    if (hasYidunElement(driver)) return true;

    // 遍历 iframe
    List<WebElement> iframes = driver.findElements(By.tagName("iframe"));
    for (int i = 0; i < iframes.size(); i++) {
        try {
            driver.switchTo().defaultContent();
            driver.switchTo().frame(i);
            if (hasYidunElement(driver)) return true;
        } catch (Exception ignored) {}
    }

    return false;
}
```

## 3. 诊断接口详细设计

### 3.1 DOM 诊断接口

```java
/**
 * 在顶层和所有 iframe 中搜索 .yidun 元素
 * 返回每个 frame 中的搜索结果
 */
@GetMapping("/debug/dom-inspect")
public ResponseDTO<List<FrameInspectResult>> domInspect() {
    WebDriver driver = browserSessionManager.getDriver();
    List<FrameInspectResult> results = new ArrayList<>();

    // 检查顶层
    driver.switchTo().defaultContent();
    results.add(inspectFrame(driver, "顶层 document"));

    // 检查所有 iframe
    List<WebElement> iframes = driver.findElements(By.tagName("iframe"));
    for (int i = 0; i < iframes.size(); i++) {
        try {
            driver.switchTo().defaultContent();
            driver.switchTo().frame(i);
            results.add(inspectFrame(driver, "iframe[" + i + "]"));
        } catch (Exception e) {
            results.add(new FrameInspectResult("iframe[" + i + "]", false, e.getMessage()));
        }
    }

    return ResponseDTO.success(results);
}

private FrameInspectResult inspectFrame(WebDriver driver, String frameName) {
    List<WebElement> yidunElements = driver.findElements(By.cssSelector("[class*='yidun']"));
    boolean found = !yidunElements.isEmpty();

    String details = "";
    if (found) {
        // 记录找到的元素信息
        details = yidunElements.stream()
            .map(e -> e.getTagName() + "." + e.getAttribute("class"))
            .collect(Collectors.joining(", "));
    }

    return new FrameInspectResult(frameName, found, details);
}
```

### 3.2 滑块轮询接口

```java
/**
 * 持续轮询检测滑块出现
 * 每秒检测一次，在顶层和所有 iframe 中查找
 */
@GetMapping("/debug/slider-poll")
public ResponseDTO<List<PollResult>> sliderPoll(
        @RequestParam(defaultValue = "10") int seconds) {
    WebDriver driver = browserSessionManager.getDriver();
    List<PollResult> results = new ArrayList<>();

    for (int i = 0; i < seconds; i++) {
        // 在所有 frame 中查找
        String foundIn = findYidunInAllFrames(driver);
        results.add(new PollResult(i + 1, foundIn != null, foundIn));

        if (foundIn != null) {
            log.info("第 {}秒 在 {} 中检测到滑块", i + 1, foundIn);
            break;
        }

        Thread.sleep(1000);
    }

    return ResponseDTO.success(results);
}
```

@GetMapping("/debug/fingerprint")
public ResponseDTO<Map<String, Object>> fingerprint() {
    WebDriver driver = browserSessionManager.getDriver();
    JavascriptExecutor js = (JavascriptExecutor) driver;

    Map<String, Object> result = new HashMap<>();
    result.put("webdriver", js.executeScript("return navigator.webdriver"));
    result.put("languages", js.executeScript("return navigator.languages"));
    result.put("platform", js.executeScript("return navigator.platform"));
    result.put("hardwareConcurrency", js.executeScript("return navigator.hardwareConcurrency"));
    result.put("userAgent", js.executeScript("return navigator.userAgent"));
    result.put("chromeRuntime", js.executeScript("return typeof window.chrome"));

    return ResponseDTO.success(result);
}

### 3.5 表单状态接口

```java
/**
 * 返回当前页面所有表单元素的状态
 */
@GetMapping("/debug/form-state")
public ResponseDTO<Map<String, Object>> formState() {
    WebDriver driver = browserSessionManager.getDriver();
    Map<String, Object> state = new HashMap<>();

    // 所有 input 元素
    List<WebElement> inputs = driver.findElements(By.tagName("input"));
    List<Map<String, String>> inputStates = inputs.stream().map(input -> {
        Map<String, String> m = new HashMap<>();
        m.put("type", input.getAttribute("type"));
        m.put("placeholder", input.getAttribute("placeholder"));
        m.put("value", input.getAttribute("value"));
        m.put("checked", String.valueOf(input.isSelected()));
        m.put("displayed", String.valueOf(input.isDisplayed()));
        return m;
    }).toList();
    state.put("inputs", inputStates);

    // 所有 button 元素
    List<WebElement> buttons = driver.findElements(By.tagName("button"));
    List<Map<String, String>> buttonStates = buttons.stream().map(btn -> {
        Map<String, String> m = new HashMap<>();
        m.put("text", btn.getText());
        m.put("enabled", String.valueOf(btn.isEnabled()));
        m.put("displayed", String.valueOf(btn.isDisplayed()));
        return m;
    }).toList();
    state.put("buttons", buttonStates);

    return ResponseDTO.success(state);
}
```

### 4.3 执行滑块验证

```java
@PostMapping("/phone/solve-slider")
public ResponseDTO<Boolean> solveSlider() {
    WebDriver driver = browserSessionManager.getDriver();

    // 1. 切换到滑块所在 frame
    if (!captchaService.switchToSliderFrame(driver)) {
        return ResponseDTO.failure("未找到滑块元素");
    }

    // 2. 提取图片 URL
    CaptchaService.ImageUrls urls = captchaService.extractYidunImageUrls(driver);
    if (urls.getBgUrl() == null || urls.getSliderUrl() == null) {
        return ResponseDTO.failure("无法提取滑块图片 URL");
    }

    // 3. 下载图片 + 计算距离
    byte[] bgImage = captchaService.downloadImage(urls.getBgUrl());
    byte[] sliderImage = captchaService.downloadImage(urls.getSliderUrl());
    int distance = captchaService.calculateSliderDistance(bgImage, sliderImage);
    log.info("滑块距离: {}px", distance);

    // 4. 执行拖动
    boolean dragSuccess = captchaService.executeSliderDrag(driver, distance);
    if (!dragSuccess) {
        return ResponseDTO.success(false, "滑块拖动失败");
    }

    // 5. 等待验证结果
    boolean verified = captchaService.waitForVerificationResult(driver, 3);
    return ResponseDTO.success(verified, verified ? "滑块验证成功" : "滑块验证失败");
}
```

## 9. Chrome Cookie 持久化设计

> **问题**：默认 Docker Chrome 每次启动都是全新 session，导致每次都被视为"首次登录"需要手机验证。
> **方案**：通过 Docker volume 挂载 Chrome 用户数据目录，保持 Cookie 和 session 跨容器重启持久化。

### 9.1 CDP 直连模式配置

通过 CDP 直连模式连接 Chrome，实现会话持久化。

#### docker-compose.yml 配置

```yaml
chrome:
  image: selenium/chrome:latest
  container_name: stock-chrome
  ports:
    - "9222:9222"    # Chrome DevTools Protocol 端口
    - "7900:7900"    # VNC 可视化端口（可选）
  mem_limit: 1500m
  shm_size: '2gb'
  volumes:
    - chrome-userdata:/home/seluser/chrome-data
  environment:
    - CHROME_PORT=9222
  networks:
    - stock-network
  restart: unless-stopped
  command: >
    bash -c "
      rm -f /tmp/.X*-lock /home/seluser/chrome-data/SingletonLock 2>/dev/null;
      mkdir -p /home/seluser/chrome-data && chown -R seluser:seluser /home/seluser/chrome-data &&
      su - seluser -c 'google-chrome --headless=new --remote-debugging-port=9222 --user-data-dir=/home/seluser/chrome-data --disable-blink-features=AutomationControlled --no-sandbox --disable-dev-shm-usage --disable-gpu'
    "

volumes:
  chrome-userdata:
```

#### 架构设计

| 组件 | 配置 | 说明 |
|------|------|------|
| CDP 端口 | 9222:9222 | 直连 Chrome DevTools Protocol |
| 容器内存限制 | mem_limit: 1500m | 适配 4G 服务器 |
| 会话持久化 | chrome-userdata 卷 | 保存 Cookie/LocalStorage |
| VNC 端口 | 7900:7900 | 可选，用于调试 |

#### 工作原理

1. Chrome 以 headless 模式运行，启用 `--remote-debugging-port=9222`
2. 后端通过 CDP 直连端口 9222
3. `--user-data-dir` 持久化到 `chrome-userdata` 卷
4. 容器重启后会话自动恢复

#### 4G 内存可行性评估

| 组件 | 内存占用 | 说明 |
|------|---------|------|
| Chrome Headless | 200-300MB | 基础开销 |
| 渲染进程 | 50-150MB | 取决于页面复杂度 |
| 系统其他 | 500-800MB | SSH、systemd 等 |
| **总计** | **~1.5GB** | 分配 mem_limit: 1500m |

**结论：4G 内存完全可行，推荐使用 headless 模式。**

#### 效果

- 首次登录完成手机验证后，Cookie 保存在 volume 中
- 下次重启容器/重新连接时，直接进入登录页（跳过手机验证）
- `AutoLoginService` 根据页面内容自动选择对应流程
- 容器重启后会话自动恢复，无需重新登录
