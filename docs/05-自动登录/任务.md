# 05-自动登录任务清单

> 目标：实现中信证券自动化登录全链路，支持两种独立流程：
> 1. **手机验证流程**（首次登录时出现）→ 滑块验证 → 短信验证码 → 跳转登录页
> 2. **登录页流程**（常规登录）→ 账号密码 → 数学验证码 → 滑块验证 → Token 获取
>
> 原则：每个步骤可执行、可验证，必须验收通过才进入下一步。

---

## 核心技术要点（已验证）

### 页面检测方式

> ⚠️ **不使用 URL 判断页面类型**：URL 跳转可能延迟导致误判。
> 通过**页面标题和 DOM 内容**检测当前页面：

| 页面 | 检测方式 |
|------|---------|
| 手机验证页 | 页面包含"手机验证"标题 + "获取验证码"按钮 + 手机号输入框 |
| 登录页 | 页面包含 password 输入框 + "登录"按钮 + 资金账号输入框 |

### jQuery 事件触发（核心）

> ⚠️ **该网站使用 jQuery 2.1.1 绑定事件**，原生 `element.click()` 无法触发 jQuery handler。
> 所有按钮点击必须使用 `jQuery(el).trigger('click')`。

| 操作 | 正确方式 | 错误方式（已删除）|
|------|---------|---------------|
| 点击按钮 | `jQuery(el).trigger('click')` | `element.click()` / `Selenium.click()` |
| 输入文本 | `jQuery(el).val(text).trigger('input').trigger('change')` | 仅设置 `element.value` |
| 勾选协议 | `jQuery(span).trigger('click')` 点击 span.icon_check | 原生 `span.click()` |

### 弹窗处理

> 该网站使用 **xubox layer 弹窗插件**，弹窗 ID 为 `xubox_layer{N}`，确定按钮 ID 为 `pop_tip_alert_btn`，
> 按钮文字中含多余空格（"确  定"而非"确定"）。

### 滑块检测

> 滑块弹出后挂载在**顶层 document**，不在 iframe 中。
> 检测必须使用**可见性过滤**（`.yidun_panel`, `.yidun_popup` 等面板元素且 `isDisplayed()=true`），
> 排除隐藏的 `input.yidun_input`（否则会误报）。

## 流程二：登录页（常规登录） ⬜ 待开发

> 该流程独立于手机验证流程。首次登录完成后，后续可直接进入登录页。

### Task 4：登录页分步接口开发

#### 4.1 页面类型检测

- **操作**：实现基于页面内容的检测方法（非 URL）
- **检测逻辑**：
  - 检查页面是否包含 `password` 输入框 → 登录页
  - 检查页面是否包含"手机验证"标题 + 手机号输入框 → 手机验证页
- **后置验收**：`GET /api/browser/debug/page-type` 返回 `PHONE_VERIFY` 或 `LOGIN`

#### 4.2 登录表单填写

- **操作**：实现账号、密码输入（使用 jQuery val + trigger）
- **接口**：
  - `POST /api/browser/login/input-account?account=XXX`（复用已有）
  - `POST /api/browser/login/input-password?password=XXX`（复用已有）
- **后置验收**：截图确认账号密码已填入

#### 4.3 数学验证码自动计算

- **操作**：提取 DOM 中的算式文本，自动计算结果并填入
- **接口**：
  - `GET /api/browser/login/capture-captcha` → 返回算式和计算结果
  - `POST /api/browser/login/input-captcha?captcha=结果`
- **后置验收**：验证码正确填入

#### 4.4 登录提交 + 滑块验证

- **操作**：点击登录按钮（jQuery trigger），检测并处理滑块
- **接口**：
  - `POST /api/browser/login/submit` → 点击登录
  - `GET /api/browser/phone/slider-status` → 复用滑块检测
  - `POST /api/browser/phone/solve-slider` → 复用滑块验证
- **后置验收**：滑块验证通过，页面跳转到交易主页

#### 4.5 Token 提取与同步

- **操作**：从 Cookie/localStorage/sessionStorage 中提取 Token
- **接口**：`GET /api/browser/login/check` → 返回 hasToken、tokenValue
- **后置验收**：Token 长度 > 0，同步到 `ZXRequestUtils`

### Task 5：登录页端到端测试

**完整测试流程**：
```
POST /api/browser/start                              → 启动浏览器
POST /api/browser/navigate/login                     → 导航到登录页
GET  /api/browser/debug/page-type                    → 检测页面类型
  ↓ 如果是 PHONE_VERIFY → 走流程一
  ↓ 如果是 LOGIN → 继续下面步骤
POST /api/browser/frame/switch                       → 切入表单区域
POST /api/browser/login/input-account?account=XXX    → 输入资金账号
POST /api/browser/login/input-password?password=XXX  → 输入交易密码
GET  /api/browser/login/capture-captcha              → 获取数学验证码
POST /api/browser/login/input-captcha?captcha=结果    → 填入验证码
POST /api/browser/login/check-agreements             → 勾选协议
POST /api/browser/login/submit                       → 点击登录
  ↓ 滑块弹窗弹出
GET  /api/browser/phone/slider-status                → hasSlider=true
POST /api/browser/phone/solve-slider                 → 滑块验证成功
GET  /api/browser/login/check                        → success=true, hasToken=true
```
