### 6. CDP 物理点击方案（已完成核心代码）

**关键发现**：该网站使用 AngularJS 框架，JS `.click()` 和 `angular.element().triggerHandler()` 都不能正确触发表单提交。**只有 CDP `Input.dispatchMouseEvent` 物理点击才能正确触发 Angular 事件处理器。**

已实现：
- [x] `cdpClick()` 通用方法：通过 CDP `Input.dispatchMouseEvent`（mousePressed + mouseReleased）执行浏览器级别点击
- [x] `jsCheckAllAgreements()`：JS 获取 checkbox 坐标 → CDP 物理点击（兼容 checkbox 被 CSS 隐藏的情况，回退到父元素坐标）
- [x] `clickLoginButton()`：JS 获取按钮坐标 → CDP 物理点击（不使用任何 JS click）
- [x] `clickSendCodeButton()`：JS 获取按钮坐标 → CDP 物理点击
- [x] `inputAccount()`：CDP 点击输入框聚焦 → 三击全选 → Backspace 清空 → `Input.insertText` 输入

### 7. 浏览器管理改进（已完成）

- [x] `startBrowser()` 启动前自动关闭残留的旧 driver 会话
- [x] `quitBrowser()` 增强：先关闭所有窗口再 quit，重置 driver/wait/actions 引用
- [x] 移除各方法中不必要的 `ensureOnLoginPage()` 调用，避免在手机验证页时被导航走

## 三、待完成事项

### 优先级 P0：修复"获取验证码"按钮 CDP 点击不生效

**现象**：CDP 点击返回成功，checkbox 状态保持（说明没触发 Angular 重渲染），但滑块未弹出。

**可能原因**：
1. CDP 点击坐标偏差——JS `getBoundingClientRect()` 获取的坐标可能与 CDP 视口坐标有偏移
2. 按钮可能有 `disabled` 状态或前端校验未通过
3. 需要确认手动点击在同一状态下是否能弹出滑块

**排查方向**：
- 在代码已输入手机号+勾选协议的状态下，手动在 Selenium 浏览器上点击"获取验证码"按钮，确认是否能弹出滑块
- 如果手动能弹出，说明是坐标问题——可以通过日志中打印的坐标值 vs 实际按钮位置来调试
- 如果手动也不能弹出，说明 `Input.insertText` 输入的手机号虽然显示正确但 Angular model 未正确绑定

**修复建议**：
- 尝试在 CDP `insertText` 后额外派发 `Input.dispatchKeyEvent`（Enter 或 Tab），确保 Angular input 事件完整
- 或改用 CDP 逐字符 `Input.dispatchKeyEvent` 输入（`char` 类型事件），而不是 `insertText`

## 四、关键技术决策记录

| 决策 | 原因 | 日期 |
|------|------|------|
| 使用 Selenium 方案（非 API 逆向） | 网易云盾 fp/cb 参数依赖真实浏览器 API，混淆 JS 逆向成本极高 | 2026-03-22 |
| CDP 物理点击替代 JS click | AngularJS 框架不响应 JS `.click()` 和 `triggerHandler()`，只响应浏览器原生事件 | 2026-03-24 |
| CDP `Input.insertText` 替代逐字符 sendKeys | Selenium `sendKeys` 在跨 iframe 场景下报 `element not interactable` | 2026-03-24 |
| 移除 `ensureOnLoginPage()` 强制导航 | 在手机验证页调用会导致 SPA 路由跳转重置表单状态 | 2026-03-24 |
