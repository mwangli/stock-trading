## Task 3：手机验证页分步接口

### 3.2 实现 `/api/browser/phone/slider-status`（跨 frame 检测）

- **前置条件**：Task 3.1 验收通过
- **操作**：
  1. 修改 `CaptchaService.detectSliderType()` 增加跨 frame 搜索
  2. 新增 `CaptchaService.switchToSliderFrame()` 方法
  3. 在 `BrowserApiController` 中新增接口
- **后置验收**：
  - `mvn compile` 成功
  - 点击「获取验证码」后调用 `GET /api/browser/phone/slider-status`，返回 `hasSlider=true, sliderType=YIDUN`
  - **这是核心验收点 — 通过说明滑块检测问题已解决**

### 3.3 实现 `/api/browser/phone/solve-slider`（CDP 网络拦截方案）

- **前置条件**：Task 3.2 验收通过（滑块能被检测到）
- **背景**：Yidun NECaptcha 使用 Canvas 渲染 + 跨域 iframe 隔离。DOM 提取图片 URL 失败，全页面截图抓不到 iframe 内的滑块弹窗。改用 CDP 网络拦截方案。
- **方案演进**：DOM 提取 → 截图+Sobel 边缘检测 → **CDP 网络拦截**（最终方案）
- **操作**：
  1. `CaptchaService` 新增 CDP 拦截方法：`enableNetworkInterception()` 监听图片请求，`interceptSliderImages()` 获取背景图/拼图块
  2. `BrowserApiController` 修改 `/phone/solve-slider` 为 CDP/DOM 双策略级联 + 3 次自动重试
  3. CDP 拦截原始图片 → SAD 模板匹配计算距离 → S 曲线拖动
- **后置验收**：
  - `mvn compile` 成功
  - 调用后日志出现：CDP 拦截到背景图/拼图块 → 距离计算（20~250px）→ 拖动完成 → 验证通过/失败
  - 通过率 > 70%（3 次重试内）

### 3.4 实现 `/api/browser/phone/confirm-slider`

- **操作**：在 `BrowserApiController` 中新增点击「确定」按钮接口
