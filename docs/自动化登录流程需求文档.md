## 1. 功能需求

### 1.1 核心功能

- 实现中信证券交易系统的自动化登录流程
- 支持账号密码输入
- 支持数学运算验证码自动计算（如 `3+5=?`）
- 支持图片验证码人工介入（截图保存 → 人工写入文件 → 系统读取）
- 支持**网易云盾（NECaptcha）滑块验证码**自动识别和验证
- 支持手机短信验证码处理（新设备首次登录场景）
- 支持登录状态保持和 Token 管理
- 支持常驻浏览器模式（登录后不关闭浏览器，维持会话）

### 1.2 验证码类型说明

中信证券登录页面存在以下验证码场景：

| 验证码类型 | 出现时机 | 处理方式 |
|-----------|---------|---------|
| 数学运算验证码（图片） | 登录页面表单中（图片形式，如 `9÷3=?`） | 截图保存到 `.tmp/` → 人工识别后写入 `captcha_code.txt`；或自动 DOM 提取算式文本并计算 |
| 网易云盾滑块验证码 | 点击登录按钮后弹出 / 手机验证页点击"获取验证码"后弹出 | Selenium 模拟拖动（图像识别计算距离 + 人类轨迹模拟） |
| 手机短信验证码 | 新设备首次登录（`activePhone.html`） | 人工写入 `sms_code.txt` |

### 1.3 业务流程

```
1. 启动 Chrome 浏览器（本地或 Remote Docker）
2. 访问登录页面 https://weixin.citicsinfo.com/tztweb/deal/index.html#!/account/login.html
3. 检测当前页面类型：
   ├── activePhone.html → 手机验证流程（短信验证码 + 可能的滑块验证）
   └── login.html → 标准登录流程
4. [标准登录流程]
   a. 切换到登录表单所在的 iframe
   b. 定位并输入账号（模拟人类打字，80-140ms/字符）
   c. 定位并输入密码
   d. 处理验证码：
      ├── 数学运算验证码 → 自动提取算式并计算
      └── 图片验证码 → 截图保存，等待人工写入结果文件
   e. 勾选《隐私保护条款》和《授权书》复选框
   f. 点击登录按钮
5. 等待网易云盾滑块验证码弹出
6. [网易云盾滑块处理]
   a. 检测 .yidun 容器元素出现
   b. 通过 JS 提取背景图和拼图图片的 URL
   c. 下载图片 → 模板匹配计算拼图缺口位置 → 得到滑动距离
   d. 根据距离生成 S 曲线缓动轨迹（加速→匀速→减速 + 随机抖动）
   e. 使用 Selenium Actions 执行 clickAndHold → moveByOffset → release
   f. 等待验证结果（最多重试 3 次）
7. 登录成功后从 Cookie / localStorage / sessionStorage 提取 Token
8. 同步 Token 到 ZXRequestUtils 供后续交易接口使用
```

## 2. 技术方案

### 2.1 方案选择：Selenium 浏览器模拟（已确认）

**选择原因**：

中信证券使用**网易云盾（NECaptcha）**作为滑块验证码服务。网易云盾的核心防护参数（`fp` 指纹、`cb` 回调）由一段 ~2KB 的混淆压缩 JS 动态生成，涉及浏览器指纹采集（Canvas、WebGL、Fonts、AudioContext 等数十个维度）。

曾评估两种方案：

| 方案 | 优点 | 缺点 | 结论 |
|------|------|------|------|
| **纯 API 逆向** | 资源消耗低、速度快 | fp/cb 参数依赖真实浏览器 API（Canvas、WebGL 等），Java 无法原生提供；混淆 JS 逆向成本极高且生命周期短（云盾定期更新） | **不采用** |
| **Selenium 模拟** | 真实浏览器环境自动生成所有指纹参数；现有代码已高度成熟；维护成本低 | 需额外 Chrome 容器（~500MB 内存）；执行稍慢 | **采用** |

### 2.2 技术栈

| 组件 | 技术 | 版本 |
|------|------|------|
| 浏览器驱动 | Selenium WebDriver | 4.17.0 |
| 驱动管理 | WebDriverManager | 5.6.3 |
| 运行时浏览器 | Chrome / Chrome Headless | latest |
| 容器化部署 | selenium/standalone-chrome | latest |
| 图像处理 | Java BufferedImage + 模板匹配 | JDK 17 内置 |
| HTTP 工具 | OkHttp / Fastjson2 | 4.12 / 2.x |

### 2.3 核心组件

| 组件 | 类名 | 职责 |
|------|------|------|
| 浏览器会话管理 | `BrowserSessionManager` | WebDriver 生命周期、指纹注入、iframe 切换、元素交互、人类行为模拟 |
| 登录页面处理 | `LoginPageHandler` | 表单元素定位、账号/密码/验证码输入、协议勾选、登录按钮点击 |
| 验证码服务 | `CaptchaService` | 滑块距离计算（模板匹配 + 边缘检测）、滑动轨迹生成（S 曲线缓动） |
| 登录编排服务 | `AutoLoginService` | 完整登录流程编排、重试逻辑、异常处理、Token 同步 |
| Token 管理 | `CookieManager` | 多策略 Token 提取（Cookie → localStorage → sessionStorage → 启发式匹配） |
| HTTP 请求工具 | `ZXRequestUtils` | 携带 Token 发送交易请求、Token 缓存（30 分钟 TTL） |
| 登录维护任务 | `AutoLoginMaintenanceJob` | 每 30 分钟检查登录状态，过期自动重新登录 |

## 3. 登录页面结构分析

### 3.1 页面层级

登录页面使用 iframe 嵌套结构，登录表单不在顶层 document 中：

```
顶层 document
└── iframe (可能多层嵌套)
    └── 登录表单（账号、密码、验证码、协议、登录按钮）
```

**关键处理**：`BrowserSessionManager.ensureLoginFrame()` 递归遍历所有 iframe，找到包含密码输入框的 frame 并切入。

### 3.2 页面元素定位（基于实际页面截图确认）

**登录页（`login.html` - 普通交易登录）**

|------|-------------------|---------|------|

## 4. 滑块验证核心算法

### 4.1 滑块距离计算

采用**灰度模板匹配**算法：

1. 将背景图和拼图块转为灰度矩阵
2. 在背景图上滑动拼图块模板，逐像素计算差异值（`Sum of Absolute Differences`）
3. 差异值最小的 X 坐标即为缺口位置
4. 备用方案：**边缘检测**，分析背景图垂直边缘变化最大的位置

**距离修正**：页面渲染尺寸与图片原始尺寸可能不一致，需按比例修正：
```
实际距离 = 计算距离 × (页面渲染宽度 / 图片原始宽度)
```

### 4.2 滑动轨迹生成

使用 **S 曲线缓动函数** 模拟人类拖动行为：

| 阶段 | 进度区间 | 行为特征 |
|------|---------|---------|
| 快速加速 | 0% ~ 30% | 二次缓动 `2t²` |
| 匀速过渡 | 30% ~ 70% | 三次曲线 |
| 缓慢减速 | 70% ~ 100% | 二次缓出 |

附加拟人特征：
- X 轴随机抖动 ±2px
- Y 轴正弦波动 ±3px（`sin(i * 0.3) * 3`）
- 轨迹点数 = `30 + distance / 10`
- 每步间隔 15-25ms

## 5. 异常处理与重试

### 5.1 重试策略

| 操作 | 最大重试次数 | 等待时间 |
|------|------------|---------|
| 完整登录流程 | 3 次 | 3 秒/次 |
| 图片验证码识别 | 5 次 | 首次 300 秒，后续 120 秒 |
| 滑块验证 | 3 次 | 立即重试 |
| Token 同步 | 8 次 | 600ms/次 |

### 5.2 异常类型与处理

| 异常类型 | 原因 | 处理策略 |
|---------|------|---------|
| 浏览器启动失败 | Chrome 版本不兼容 | 检查 WebDriverManager 配置 |
| iframe 切换失败 | 页面结构变更 | 递归遍历所有 frame |
| 元素定位失败 | DOM 结构变化 | 多策略定位 + 超时重试 |
| 数学验证码提取失败 | 页面使用图片验证码 | 降级为人工介入模式 |
| 滑块图片下载失败 | 网络异常或 URL 变化 | 重新提取 URL 并重试 |
| 滑块验证被拒绝 | 轨迹被识别为机器人 | 重新生成轨迹 + 调整参数 |
| Token 获取失败 | Cookie/Storage 中无 Token | 多策略依次尝试 + 启发式匹配 |
| 登录会话过期 | 30 分钟 Token TTL | AutoLoginMaintenanceJob 自动重登 |

## 6. 安全考虑

### 6.1 反爬虫检测规避

1. **浏览器指纹注入**：覆写 `navigator.webdriver`、`languages`、`platform`、`hardwareConcurrency`、`deviceMemory`
2. **人类行为模拟**：逐字符输入（80-140ms 延迟）、随机等待、鼠标移动事件派发
3. **Automation 特征移除**：`--disable-blink-features=AutomationControlled`、移除自动化扩展

### 6.2 敏感信息保护

- 密码不在日志中输出
- Token 日志仅显示前 10 位 + `***`
- 账号密码通过 API 参数传入或环境变量配置

## 7. 部署方案

### 7.1 本地开发

```bash
# 后端自动通过 WebDriverManager 下载 ChromeDriver
cd backend && mvn spring-boot:run
```

### 7.2 Docker 部署（生产环境）

```yaml
services:
  chrome:
    image: selenium/standalone-chrome:latest
    container_name: stock-chrome
    ports:
      - "4444:4444"    # Selenium Grid
      - "7900:7900"    # VNC 可视化（调试用）
    shm_size: '2gb'
    environment:
      - SE_NODE_MAX_SESSIONS=1
      - SE_SESSION_REQUEST_TIMEOUT=300
    restart: unless-stopped

  stock-backend:
    environment:
      - CHROME_REMOTE_URL=http://chrome:4444/wd/hub
      - CHROME_HEADLESS=true
    depends_on:
      - chrome
```

### 10.4 关键日志检查点

执行登录后，在后端日志中依次确认以下关键节点：

| 日志关键字 | 说明 | 预期 |
|-----------|------|------|
| `检测到滑块验证码类型: yidun` | 滑块类型识别 | 点击登录后出现 |
| `网易云盾图片提取成功` | 背景图/拼图 URL 获取 | bgUrl 和 sliderUrl 均为"已获取" |
| `滑块图片下载成功` | 图片已下载到内存 | bg 和 slider 字节数 > 0 |
| `网易云盾距离修正` | 渲染尺寸 vs 原始尺寸修正 | 仅在两者不一致时出现 |
| `滑块距离: Xpx → 拖动距离: Ypx` | 最终拖动距离 | 距离应在合理范围内（20~250px） |
| `网易云盾滑块验证通过` 或 `滑块已消失，视为通过` | 验证结果 | 3 次重试内出现 |
| `Token 已同步到 ZXRequestUtils` | Token 获取成功 | 长度 > 0 |

### 10.5 验证标准

| 指标 | 目标 |
|------|------|
| 后端启动成功 | 无报错，端口 8080 正常监听 |
| 浏览器启动 | `/api/browser/start` 返回成功 |
| 登录页面访问 | URL 包含 `login.html` 或 `activePhone.html` |
| 网易云盾滑块检测 | 日志中出现 `yidun` 类型 |
| 滑块验证通过率 | > 70%（3 次重试内） |
| Token 获取成功率 | > 95% |
| 单次完整登录耗时 | < 60 秒 |
