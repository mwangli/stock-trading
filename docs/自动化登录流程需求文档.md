## 1. 功能需求

### 1.1 核心功能

- 实现中信证券交易系统的自动化登录流程
- 支持**两阶段登录流程**：手机验证页（Phase 1）→ 登录页（Phase 2）
- 两个阶段均需支持**网易云盾（NECaptcha）滑块验证码**自动识别和验证
- 支持账号密码输入、数学运算验证码自动计算
- 支持手机短信验证码处理（人工写入文件）
- 支持登录状态保持和 Token 管理
- 支持常驻浏览器模式（登录后不关闭浏览器，维持会话）

### 1.2 完整登录流程（两阶段）

```
┌────────────────────────────────────────────────────────────┐
│  Phase 1: 手机验证页 (activePhone.html)                    │
│  ─────────────────────────────────────                     │
│  1. 启动浏览器，访问登录 URL                                │
│  2. 页面自动跳转到 activePhone.html（新设备首次登录）        │
│  3. 切换到表单所在 iframe                                   │
│  4. 输入手机号码                                            │
│  5. 勾选《隐私保护条款》和《授权书》                         │
│  6. 点击「获取验证码」按钮                                   │
│  7. 【关键】等待网易云盾滑块弹出 → 拖动滑块验证              │
│  8. 滑块验证通过后，点击「确定」按钮                         │
│  9. 等待用户收到短信 → 输入短信验证码                        │
│  10. 点击「下一步」/「登录」按钮                             │
│  11. 验证通过 → 自动跳转到登录页                             │
└────────────────────────────────────────────────────────────┘
                          ↓ 页面跳转
┌────────────────────────────────────────────────────────────┐
│  Phase 2: 登录页 (login.html)                              │
│  ─────────────────────────────────                         │
│  1. 切换到登录表单所在 iframe                                │
│  2. 输入手机号码/资金账号                                    │
│  3. 输入交易密码                                            │
│  4. 处理验证码（数学运算自动计算 / 图片人工介入）             │
│  5. 勾选《隐私保护条款》和《授权书》                         │
│  6. 点击「登录」按钮                                        │
│  7. 【关键】等待网易云盾滑块弹出 → 拖动滑块验证              │
│  8. 滑块验证通过后，点击「登录」按钮完成登录                  │
│  9. 登录成功 → 提取 Token → 同步到 ZXRequestUtils            │
└────────────────────────────────────────────────────────────┘
```

**核心共性**：两个阶段都在特定按钮点击后触发网易云盾滑块验证，滑块处理逻辑完全一致。**解决其中一个页面的滑块问题，另一个页面自然解决。**

### 1.3 当前开发重点：手机验证页滑块问题

> **当前阻塞问题**：点击「获取验证码」按钮后，网易云盾滑块**没有正确弹出**（或弹出了但未被检测到）。

**聚焦目标**：先在手机验证页实现完整的「点击按钮 → 滑块弹出 → 滑块检测 → 滑块拖动 → 验证通过」链路。

## 2. 验证码类型说明

| 验证码类型       | 出现时机                          | 处理方式                                                         |
| ----------- | ----------------------------- | ------------------------------------------------------------ |
| 网易云盾滑块验证码   | Phase 1: 点击「获取验证码」后弹出<br>Phase 2: 点击「登录」按钮后弹出 | Selenium 模拟拖动（图像识别计算距离 + 人类轨迹模拟） |
| 数学运算验证码（图片） | Phase 2: 登录页表单中（如 `9÷3=?`）       | 自动 DOM 提取算式文本并计算；失败则截图人工介入 |
| 手机短信验证码     | Phase 1: `activePhone.html`   | 人工写入 `.tmp/sms_code.txt`                                          |

## 3. 技术方案

### 3.1 方案选择：Selenium 浏览器模拟（已确认）

| 方案              | 优点                                | 缺点                                                                         | 结论      |
| --------------- | --------------------------------- | -------------------------------------------------------------------------- | ------- |
| **纯 API 逆向**    | 资源消耗低、速度快                         | fp/cb 参数依赖真实浏览器 API；混淆 JS 逆向成本极高且生命周期短 | **不采用** |
| **Selenium 模拟** | 真实浏览器环境自动生成所有指纹参数；维护成本低 | 需额外 Chrome 容器（~500MB 内存）；执行稍慢                                             | **采用**  |

### 3.2 技术栈

| 组件      | 技术                         | 版本         |
| ------- | -------------------------- | ---------- |
| 浏览器驱动   | Selenium WebDriver         | 4.17.0     |
| 驱动管理    | WebDriverManager           | 5.6.3      |
| 运行时浏览器  | Chrome / Chrome Headless   | latest     |
| 容器化部署   | selenium/standalone-chrome | latest     |
| 图像处理    | Java BufferedImage + 模板匹配  | JDK 17 内置  |
| HTTP 工具 | OkHttp / Fastjson2         | 4.12 / 2.x |

### 3.3 核心组件

| 组件        | 类名                        | 职责                                                           |
| --------- | ------------------------- | ------------------------------------------------------------ |
| 浏览器会话管理   | `BrowserSessionManager`   | WebDriver 生命周期、指纹注入、iframe 切换、元素交互、人类行为模拟 |
| 登录页面处理    | `LoginPageHandler`        | 两个页面的元素定位、表单输入、按钮点击 |
| 验证码服务     | `CaptchaService`          | 滑块距离计算（模板匹配 + 边缘检测）、滑动轨迹生成（S 曲线缓动）|
| 登录编排服务    | `AutoLoginService`        | 完整登录流程编排（Phase 1 → Phase 2）、重试、异常处理 |
| Token 管理  | `CookieManager`           | 多策略 Token 提取（Cookie → localStorage → sessionStorage → 启发式匹配） |
| 登录维护任务    | `AutoLoginMaintenanceJob` | 每 30 分钟检查登录状态，过期自动重新登录 |

## 4. 页面结构分析

### 4.1 页面层级

两个页面均使用 iframe 嵌套结构：

```
顶层 document
└── iframe (可能多层嵌套)
    └── 表单元素（输入框、按钮、复选框等）
```

**关键处理**：`BrowserSessionManager.ensureLoginFrame()` 递归遍历所有 iframe，找到包含目标输入框的 frame 并切入。

> **注意**：手机验证页没有 `password` 输入框，需要用**手机号输入框**或**获取验证码按钮**作为定位特征。

### 4.2 手机验证页（activePhone.html）— Phase 1

| 元素      | placeholder / 文本         | 定位策略                                                        | 说明      |
| ------- | ------------------------ | ----------------------------------------------------------- | ------- |
| 手机号输入框  | `请输入手机号`                 | `input[placeholder*='手机号']`                                 | 手机号     |
| 获取验证码按钮 | `点击获取验证码`                | `button/div/span` 文本含 "获取验证码"                             | 点击后触发滑块 |
| 短信验证码输入框  | `请输入手机验证码`               | `input[placeholder*='手机验证码']` → `input[placeholder*='验证码']` | 短信验证码   |
| 隐私条款复选框 | `勾选表示同意《隐私保护条款》`         | 第 1 个 checkbox                                              | 必选      |
| 授权书复选框  | `勾选表示同意《授权书》`            | 第 2 个 checkbox                                              | 必选      |
| 下一步按钮   | `下一步`                    | `button` 文本含 "下一步"                                          | 提交      |
| 提示文本    | `温馨提示: 您首次登录时,需通过手机验证身份` | —                                                           | 仅首次出现 |

### 4.3 登录页（login.html）— Phase 2

| 元素      | placeholder / 文本 | 定位策略                                                                              | 说明            |
| ------- | ---------------- | --------------------------------------------------------------------------------- | ------------- |
| 账号输入框   | `请输入资金账号`        | `input[placeholder*='资金账号']` → `input[placeholder*='账号']` → `input[id='account']` | 资金账号          |
| 密码输入框   | `请输入交易密码`        | `input[placeholder*='交易密码']` → `input[type='password']`                           | 交易密码          |
| 验证码输入框  | `请输入四则运算的运算结果`   | `input[placeholder*='四则运算']` → `input[placeholder*='运算结果']`                       | 填写运算答案        |
| 验证码图片   | 右侧图片（如 `9÷3=?`）  | `img` 标签，宽 50~250px，高 15~80px                                                   | 图片形式四则运算 |
| 隐私条款复选框 | `勾选表示同意《隐私保护条款》` | 第 1 个 `input[type="checkbox"]`                                                    | 必选            |
| 授权书复选框  | `勾选表示同意《授权书》`    | 第 2 个 `input[type="checkbox"]`                                                    | 必选            |
| 登录按钮    | `登 录`            | `button` 文本含 "登录"                                                                 | 点击后触发滑块 |

### 4.4 网易云盾滑块元素

**两个页面的滑块弹出方式和 DOM 结构一致**：

| 元素      | CSS 选择器                                    | 说明          |
| ------- | ------------------------------------------ | ----------- |
| 验证码容器   | `.yidun`, `.yidun_panel`                   | 整个验证码区域     |
| 背景图     | `.yidun_bg-img img`, `.yidun_bgimg`        | 带缺口的背景图     |
| 拼图块     | `.yidun_jigsaw img`, `.yidun_slider__icon` | 需要拖到缺口位置的拼图 |
| 滑块滑道    | `.yidun_slider`                            | 底部滑动条       |
| 滑块按钮    | `.yidun_slider__handler`, `.yidun_btn`     | 可拖动的滑块手柄    |
| 成功/失败提示 | `.yidun_tips__text`                        | 验证结果文字      |

> **关键问题**：滑块弹出后可能在**当前 iframe 内**，也可能在**顶层 document** 或**新的 iframe** 中。检测时需要切换 frame 上下文来查找。

## 5. 滑块验证核心算法

### 5.1 滑块距离计算

采用**灰度模板匹配**算法（SAD）+ **边缘检测**备用方案。

**距离修正**：`实际距离 = 计算距离 × (页面渲染宽度 / 图片原始宽度)`

### 5.2 滑动轨迹生成

S 曲线缓动函数 + 拟人特征（X 抖动 ±2px、Y 正弦波 ±3px、每步 15-25ms）。

## 6. 当前阻塞问题分析与排查方案

### 6.1 问题描述

在手机验证页（activePhone.html），点击「获取验证码」按钮后，网易云盾滑块**没有正确弹出或没有被检测到**。

### 6.2 可能原因分析

| # | 可能原因 | 概率 | 排查方式 |
|---|---------|------|---------|
| 1 | **iframe 上下文错误**：点击按钮后滑块出现在顶层 document，但代码在 iframe 内查找 | 高 | 新增诊断接口：在顶层和所有 iframe 中分别查找 `.yidun` 元素 |
| 2 | **按钮未真正点击**：`safeClick` 点击了错误元素，或被其他元素遮挡 | 中 | 新增诊断接口：点击前后截全页面图对比 |
| 3 | **时序问题**：点击按钮后滑块异步加载，检测时机太早或太晚 | 中 | 新增诊断接口：点击后每秒轮询滑块状态，持续 10 秒 |
| 4 | **反爬检测**：网易云盾检测到 Selenium 特征，拒绝弹出滑块 | 低 | 检查浏览器指纹注入是否生效（`navigator.webdriver` 是否为 undefined） |
| 5 | **协议未勾选**：未勾选复选框导致按钮点击无效 | 低 | 点击前确认复选框状态 |
| 6 | **手机号未输入或格式错误**：表单校验未通过 | 低 | 点击前确认输入框内容 |

### 6.3 诊断接口设计

为排查上述问题，新增以下**诊断接口**（仅用于调试）：

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| DOM 诊断 | GET | `/api/browser/debug/dom-inspect` | 在顶层 + 所有 iframe 中搜索 `.yidun` 元素，返回找到位置 |
| 全页面截图 | GET | `/api/browser/debug/screenshot` | 截取当前页面完整截图保存到 `.tmp/` |
| 滑块轮询 | GET | `/api/browser/debug/slider-poll?seconds=10` | 点击后持续轮询滑块出现，返回每秒检测结果 |
| 指纹检查 | GET | `/api/browser/debug/fingerprint` | 执行 JS 检查 `navigator.webdriver` 等反检测状态 |
| 当前 frame 信息 | GET | `/api/browser/debug/frame-info` | 返回当前所在 frame 路径和页面 URL |
| 表单状态 | GET | `/api/browser/debug/form-state` | 返回所有输入框、复选框、按钮的当前值和状态 |

### 6.4 排查步骤（按优先级执行）

```
步骤 1：确认基础状态
  POST /api/browser/start                       → 启动浏览器
  POST /api/browser/navigate/login              → 访问登录页
  GET  /api/browser/debug/frame-info            → 确认当前页面类型

步骤 2：切入正确 iframe + 填表
  POST /api/browser/frame/switch                → 切入 iframe
  POST /api/browser/login/input-account?account=手机号
  POST /api/browser/login/check-agreements      → 勾选协议
  GET  /api/browser/debug/form-state            → 确认表单填写完整

步骤 3：点击按钮并诊断滑块
  GET  /api/browser/debug/screenshot            → 点击前截图
  POST /api/browser/phone/send-code             → 点击「获取验证码」
  GET  /api/browser/debug/screenshot            → 点击后截图（对比变化）
  GET  /api/browser/debug/slider-poll?seconds=10 → 持续 10 秒检测滑块

步骤 4：如果 slider-poll 检测不到
  GET  /api/browser/debug/dom-inspect           → 在所有 frame 中搜索 .yidun
    → 如果在顶层找到：说明是 iframe 上下文问题（原因 #1）
    → 如果完全找不到：说明滑块没弹出（原因 #2/#4/#5/#6）

步骤 5：如果是 iframe 上下文问题
  → 修改代码：检测滑块时先 switchTo().defaultContent()
  → 拖动滑块时在正确的 frame 上下文中操作

步骤 6：如果滑块确实没弹出
  GET  /api/browser/debug/fingerprint           → 检查反爬检测状态
  → 对比截图前后页面是否有变化（是否有错误提示？按钮是否变灰？）
```

## 7. API 接口

### 7.1 一键登录

```
POST /api/auto-login/login
POST /api/auto-login/login?username=XXX&password=XXX
GET  /api/auto-login/status
POST /api/auto-login/quit
```

### 7.2 分步操作 — 通用

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 启动浏览器 | POST | `/api/browser/start` | 启动 Chrome |

| 5 | POST | `/api/browser/phone/solve-slider` | 执行滑块验证 |
| 6 | POST | `/api/browser/phone/confirm-slider` | 点击「确定」 |
| 7 | POST | `/api/browser/phone/input-code?code=XXXXXX` | 输入短信验证码 |

### 7.4 分步操作 — 登录页（Phase 2）

| 步骤 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 1 | POST | `/api/browser/login/input-account?account=账号` | 输入账号 |
| 2 | POST | `/api/browser/login/input-password?password=密码` | 输入密码 |
| 3 | GET | `/api/browser/login/capture-captcha` | 截取/计算验证码 |

| 5 | POST | `/api/browser/login/check-agreements` | 勾选协议 |

| 8 | POST | `/api/browser/phone/solve-slider` | 执行滑块验证（复用同一逻辑） |
| 9 | GET | `/api/browser/login/check` | 检查登录结果 |

### 7.5 诊断接口（调试用）

## 8. 异常处理与重试

| 操作       | 最大重试次数 | 等待时间              |
| -------- | ------ | ----------------- |
| 完整登录流程   | 3 次    | 3 秒/次             |
| 图片验证码识别  | 5 次    | 首次 300 秒，后续 120 秒 |
| 滑块验证     | 3 次    | 立即重试              |
| Token 同步 | 8 次    | 600ms/次           |

## 9. 部署方案

### 9.1 本地开发

```bash
cd backend && mvn spring-boot:run
```

### 9.2 Docker 部署

```yaml
services:
  chrome:
    image: selenium/standalone-chrome:latest
    ports:
      - "4444:4444"    # Selenium Grid
      - "7900:7900"    # VNC 可视化（密码: secret）
    shm_size: '2gb'

  stock-backend:
    environment:
      - CHROME_REMOTE_URL=http://chrome:4444/wd/hub
      - CHROME_HEADLESS=true
    depends_on:
      - chrome
```

### 9.3 配置项

| 配置                           | 说明                   | 默认值            |
| ---------------------------- | -------------------- | -------------- |
| `chrome.remote.url`          | Remote WebDriver 地址  | 空（使用本地 Chrome） |
