# 动态任务调度设计（固定任务集 + 动态配置）

日期：2026-03-08

## 目标与范围

本设计面向后端固定任务集合的动态调度需求，支持运行时调整 Cron、暂停/恢复、手动触发、查看状态（上次/下次执行时间）。

- **固定任务集合**：任务在代码中预定义（如 syncStockList、syncDailyStockData、runStockSelection 等）。
- **动态配置**：Cron/启停/描述等配置持久化到数据库，运行时生效。
- **管理能力**：提供管理接口以查询、修改、暂停/恢复与手动触发。

非目标：运行时创建全新任务类型、分布式调度一致性（如多节点幂等/锁）、Quartz 集成。

## 架构设计

### 组件与职责

**`com.stock.job` 包**

- `JobConfig`（JPA 实体）
  - 字段：
    - `id`（主键）
    - `jobKey`（唯一业务 key）
    - `cron`（Cron 表达式）
    - `description`（任务描述）
    - `enabled`（是否启用）
    - `lastRunTime`（上次执行时间）
    - `nextRunTime`（下次执行时间）
    - `status`（运行状态：如 NORMAL/PAUSED）
    - `createdAt` / `updatedAt`（审计字段）

- `JobConfigRepository`
  - Spring Data JPA 仓库，提供 `findByJobKey` 等查询。

- `JobSchedulerService`
  - 持有 `ThreadPoolTaskScheduler` 与 `ScheduledFuture` 映射。
  - 提供注册、配置、调度、暂停/恢复、手动触发、状态查询等核心能力。

**现有 Scheduler 改造**

- `DataSyncScheduler` / `StrategyScheduler`：
  - 移除 `@Scheduled` 注解；
  - 在 `@PostConstruct` 中调用 `JobSchedulerService.registerJob` 注册固定任务；
  - 调用 `configureJob(key, defaultCron, description)` 完成配置初始化与调度。

**JobAdminController**

- 提供 REST 接口：列表、详情、更新 cron、暂停/恢复、手动触发、状态查询。

## 核心流程

### 1. 启动注册流程

1) Scheduler 在 `@PostConstruct` 中注册任务：

- `registerJob(jobKey, runnable)`
- `configureJob(jobKey, defaultCron, description)`

2) `configureJob` 从数据库读取配置：

- 若不存在：以默认 cron/描述创建记录（enabled 默认 true）。
- 若存在：以数据库配置为准。

3) enabled 为 true 时立即调度，并维护 `ScheduledFuture`。

### 2. 运行与状态更新

- 任务执行被 try-catch 包裹，确保异常不导致任务停止调度。
- 执行前后更新 `lastRunTime`。
- 每次调度后根据 cron 计算并写入 `nextRunTime`。

### 3. 管理操作

- **更新 Cron**：取消旧任务 → 更新 cron → 重新调度。
- **暂停**：取消任务并设置 enabled=false/status=PAUSED。
- **恢复**：enabled=true/status=NORMAL，并重新调度。
- **Run Now**：立即异步执行 runnable，不影响调度周期。

## 错误处理与并发策略

- 任务内部统一捕获异常并记录，避免任务失活。
- 同一 jobKey 只允许一个 `ScheduledFuture`；更新/恢复前需先 cancel。
- `ThreadPoolTaskScheduler` 使用固定线程池大小配置，避免单线程串行阻塞。

## 数据模型建议

```text
JobConfig
  id: Long
  jobKey: String (unique)
  cron: String
  description: String
  enabled: boolean
  status: String
  lastRunTime: LocalDateTime
  nextRunTime: LocalDateTime
  createdAt: LocalDateTime
  updatedAt: LocalDateTime
```

## 接口建议

```
GET    /api/admin/jobs                # 列表
GET    /api/admin/jobs/{jobKey}       # 详情
PUT    /api/admin/jobs/{jobKey}/cron  # 更新 cron
POST   /api/admin/jobs/{jobKey}/pause # 暂停
POST   /api/admin/jobs/{jobKey}/resume# 恢复
POST   /api/admin/jobs/{jobKey}/run   # 立即执行
```

## 测试与验证

- 构建验证：`mvn compile`
- 修改文件的 `lsp_diagnostics` 确保无错误

## 风险与规避

- **多节点重复执行**：当前设计为单节点内存调度。若未来多实例部署，可引入 ShedLock 或 Quartz 分布式调度。
- **Cron 配置错误**：在更新时校验表达式，避免运行时报错。
- **任务阻塞**：线程池大小需合理配置，避免长任务阻塞其他任务。
