# 动态任务调度 Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** 在后端实现固定任务集合的动态调度配置（Cron 可变、可暂停/恢复、可手动触发、可查看状态）。

**Architecture:** 使用 `ThreadPoolTaskScheduler` + `ScheduledFuture` 实现内存调度，任务配置持久化在 `JobConfig` 表中；启动时注册固定任务，读取或创建配置并调度；管理接口通过 `JobSchedulerService` 操作。

**Tech Stack:** Spring Boot 3.2, Spring Scheduling, Spring Data JPA, Lombok

```java
package com.stock.job.entity;

import jakarta.persistence.*;
import lombok.*;

import java.time.LocalDateTime;

@Entity
@Table(name = "job_config")
@Data
@Builder
@NoArgsConstructor
@AllArgsConstructor
public class JobConfig {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, unique = true, length = 100)
    private String jobKey;

    @Column(nullable = false, length = 100)
    private String cron;

    @Column(length = 255)
    private String description;

    @Column(nullable = false)
    private boolean enabled;

    @Column(length = 20)
    private String status;

    private LocalDateTime lastRunTime;
    private LocalDateTime nextRunTime;

    private LocalDateTime createdAt;
    private LocalDateTime updatedAt;

    @PrePersist
    public void prePersist() {
        LocalDateTime now = LocalDateTime.now();
        this.createdAt = now;
        this.updatedAt = now;
    }

    @PreUpdate
    public void preUpdate() {
        this.updatedAt = LocalDateTime.now();
    }
}
```
