# 02-AI 模型模块 - 设计文档

## 模块概述

AI 模型模块（`com.stock.models`）是系统的智能分析层，负责股票价格预测和财经新闻情感分析，为交易决策提供 AI 支持。

## 核心职责

1. **LSTM 价格预测** - 基于历史 K 线数据预测次日股票价格走势
2. **情感分析推理** - 分析财经新闻对股票的情感倾向 (利好/利空)
3. **股票实体识别** - 识别新闻中提及的股票 (代码/简称/全称)
4. **模型管理** - 模型加载、版本管理、热切换
5. **预测服务** - 为策略模块提供预测结果和置信度

## 核心组件

### 1. StockLSTM (LSTM 网络模型)

- **职责**: 定义 LSTM 神经网络结构
- **输入**: 过去 N 日的 K 线数据 (开高低收量)
- **输出**: 次日涨跌概率
- **特性**: 纯 Java 实现，DJL 框架加载

### 2. LstmInference (LSTM 推理服务)

- **职责**: 加载 LSTM 模型并执行价格预测推理
- **输入**: 股票代码
- **输出**: PredictionResult(涨跌概率，置信度)
- **流程**: 数据获取 → 特征工程 → 模型推理 → 结果输出

### 3. SentimentInference (情感分析服务)

- **职责**: 分析财经新闻的情感倾向
- **模型**: FinBERT (中文金融情感分析模型)
- **输入**: 新闻文本 + 股票代码
- **输出**: SentimentResult(情感得分，利好/利空)

### 4. EntityRecognizer (股票实体识别器)

- **职责**: 识别新闻文本中提及的股票
- **策略**:
  - 股票代码匹配 (6 位数字)
  - 股票全称匹配
  - 股票简称匹配
  - 曾用名匹配
- **输出**: 股票 ID 列表

### 5. NewsProcessor (新闻处理流水线)

- **职责**: 处理新闻的完整流水线
- **流程**:
  1. 新闻文本清洗
  2. 句子分割
  3. 股票实体识别
  4. 句子级情感分析
  5. 股票级情感聚合
  6. 输出结果

## 数据流程

### LSTM 价格预测流程

```
MongoDB (K 线) → 数据预处理 → 特征标准化 → LSTM 模型 → 结果后处理 → 预测结果
                      ↓
               过去 60 日 K 线              涨跌概率
               开高低收量                 置信度
                                         预测价格
```

### 新闻情感分析流程

```
新闻文本 → 文本清洗 → 句子分割 → 实体识别 → 句子级情感 → 聚合 → 股票级情感
                                        ↓
                                 FinBERT 模型
                                 利好/利空/中性
```

### 多股票新闻处理

```
单条新闻 (涉及多股票)
    ↓
句子分割 (拆分为独立语义单元)
    ↓
每句实体识别 (识别提及的股票)
    ↓
句子级情感分析 (针对特定股票)
    ↓
股票级情感聚合 (加权平均)
    ↓
输出: 每只股票的情感得分
```

## 模型设计

### LSTM 网络结构

```
输入层 (60 时间步 × 6 特征)
    ↓
LSTM 层 (128 隐藏单元)
    ↓
Dropout (0.3)
    ↓
全连接层 (64 单元)
    ↓
ReLU 激活
    ↓
输出层 (2 单元：涨/跌概率)
    ↓
Softmax
```

### 特征工程

**输入特征 (6 维):**
1. 开盘价 (归一化)
2. 最高价 (归一化)
3. 最低价 (归一化)
4. 收盘价 (归一化)
5. 成交量 (归一化)
6. 涨跌幅 (%)

**归一化方法:**
- 价格特征：除以基准价 (首日收盘价)
- 成交量：对数变换后归一化
- 涨跌幅：直接使用百分比

### 情感分析模型

**模型选择**: FinBERT (中文金融领域 BERT)

**输入格式:**
```
[CLS] 新闻文本 [SEP]
```

**输出:**
- 利好概率 (0-1)
- 利空概率 (0-1)
- 中性概率 (0-1)

**情感得分计算:**
```
score = 利好概率 - 利空概率
score > 0.1: 利好
score < -0.1: 利空
其他：中性
```

## 技术栈

- **深度学习框架**: DJL 0.36.0 (Deep Java Library)
- **模型引擎**: PyTorch Engine
- **NLP 模型**: FinBERT (中文金融情感)
- **数值计算**: ND4J
- **工具类**: Hutool, Commons Math

## 关键设计决策

### 1. 纯 Java 实现

- **选择**: 使用 DJL 而非 Python 服务
- **原因**: 
  - 简化部署 (无需 Python 环境)
  - 降低延迟 (无进程间通信)
  - 统一技术栈 (全 Java)

### 2. 句子级情感分析

- **问题**: 一条新闻可能涉及多只股票，情感可能相反
- **方案**: 先句子级分析，再股票级聚合
- **优势**: 准确捕捉每只股票的情感倾向

### 3. 模型与代码分离

- **设计**: 模型文件独立于代码
- **优势**:
  - 支持模型热更新
  - 版本管理方便
  - 可以 A/B 测试不同模型

### 4. 置信度输出

- **设计**: 除了预测结果，还输出置信度
- **用途**: 策略模块可以根据置信度调整权重
- **计算**: 基于概率分布的熵或 margin

## 对外接口

### 预测服务接口

```
LstmInference
  - predict(String stockCode): PredictionResult
  - batchPredict(List<String> codes): List<PredictionResult>
  
PredictionResult:
  - stockCode: String
  - riseProbability: double (涨的概率)
  - fallProbability: double (跌的概率)
  - confidence: double (置信度)
  - predictedChange: double (预测涨跌幅)
```

### 情感分析接口

```
SentimentInference
  - analyze(String newsText, String stockCode): SentimentResult
  - batchAnalyze(List<News> newsList): List<SentimentResult>
  
SentimentResult:
  - newsId: String
  - stockCode: String
  - sentimentScore: double (-1 到 1)
  - sentiment: String (利好/利空/中性)
  - confidence: double
```

## 性能指标

### LSTM 预测

- **推理时间**: < 100ms/股票
- **准确率**: ~55-60% (涨跌方向)
- **更新频率**: 每日一次 (收盘后)

### 情感分析

- **推理时间**: < 50ms/条新闻
- **处理速度**: ~20 条/秒
- **更新频率**: 实时 (新闻采集后立即分析)

## 与其他模块的关系

- **数据采集模块**: 获取 K 线数据和新闻数据
- **策略模块**: 提供预测结果和情感得分用于选股
- **执行模块**: 不直接交互
