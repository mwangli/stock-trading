# AI 股票交易系统 - 整体架构设计

## 系统概述

AI 股票自动交易系统是基于 LSTM 神经网络和财经新闻情感分析的智能交易决策系统，采用前后端集成架构，支持自动化交易、实时数据分析和 AI 模型预测。

## 模块划分

| 模块 | 包路径 | 职责 | 关键技术 |
|------|--------|------|----------|
| 数据采集 | com.stock.databus | 股票数据、新闻采集 | Tushare API, ROME, Jsoup |
| AI 模型 | com.stock.models | 价格预测、情感分析 | DJL, LSTM, FinBERT |
| 策略模块 | com.stock.strategy | 选股决策、信号生成 | 双因子模型，排名算法 |
| 执行模块 | com.stock.executor | 风控检查、交易执行 | 风控规则，订单管理 |
| Web 层 | com.stock.web | REST API, 前端服务 | Spring Boot, Static Files |

## 核心业务流程

### 1. 数据采集流程

```
外部数据源 (Tushare/RSS/网页)
    ↓
采集客户端 (TushareClient/RomeClient/JsoupClient)
    ↓
数据清洗和验证
    ↓
存储 (MySQL/MongoDB)
    ↓
供 AI 模型模块使用
```

### 2. AI 预测流程

```
K 线数据 (MongoDB) → LSTM 预测 → 涨跌概率
新闻数据 (MySQL) → 实体识别 → 情感分析 → 情感得分
    ↓
输出到策略模块
```

### 3. 策略决策流程

```
LSTM 预测结果 + 情感分析结果
    ↓
因子计算器 (标准化)
    ↓
综合得分 = LSTM×0.6 + 情感×0.4
    ↓
股票排名 (降序排序)
    ↓
决策引擎 (生成交易信号)
    ↓
输出到执行模块
```

### 4. 交易执行流程

```
交易信号
    ↓
风控检查 (止损/仓位/熔断)
    ↓
通过？─否─→ 拒绝
    │
    是
    ↓
执行交易 (模拟/实盘)
    ↓
更新持仓和账户
    ↓
记录交易流水
```

## 关键设计决策

### 1. 纯 Java 实现

- **决策**: 所有模块使用 Java 实现，包括 AI 推理
- **技术**: DJL 框架加载深度学习模型
- **优势**: 简化部署，统一技术栈，降低延迟

### 2. 前后端集成部署

- **决策**: 前端打包到后端 resources/static
- **技术**: Spring Boot 静态文件服务
- **优势**: 单端口访问，简化部署和运维

### 3. 双因子选股模型

- **决策**: LSTM(60%) + 情感 (40%)
- **原因**: 技术指标 + 市场情绪，互补增强
- **效果**: 提高选股准确性和稳定性

### 4. 多层次风控

- **决策**: 账户级 + 股票级 + 交易级风控
- **规则**: 止损、仓位、熔断等多维度
- **目的**: 全方位控制交易风险

### 5. 模拟/实盘分离

- **决策**: 支持模拟和实盘两种模式
- **优势**: 安全测试策略，无缝切换实盘

## 性能指标

### 数据采集

- **Tushare API**: 50 次/分钟 (频控限制)
- **RSS 采集**: 10 个源/次，约 30 秒
- **新闻采集**: 每 30 分钟一次

### AI 推理

- **LSTM 预测**: < 100ms/股票
- **情感分析**: < 50ms/条新闻
- **批量处理**: 20-30 条/秒

### 策略决策

- **综合选股**: < 5 秒 (全市场)
- **信号生成**: < 1 秒

### 交易执行

- **风控检查**: < 10ms
- **订单生成**: < 50ms
- **持仓更新**: < 100ms
