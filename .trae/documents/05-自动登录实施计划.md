### 1.3 核心难题：Docker Chrome 会话持久化

**问题**：默认 Docker Chrome 每次启动都是全新 session，导致每次都被视为"首次登录"需要手机验证。

**推荐方案：方案C - 长期运行容器 + 单会话复用**

```yaml
# docker-compose.yml 关键配置
chrome:
  image: selenium/standalone-chrome:latest
  ports:
    - "4444:4444"    # Selenium Grid
    - "9222:9222"    # CDP 直连
    - "7900:7900"    # VNC
  mem_limit: 1500m
  environment:
    - SE_NODE_MAX_SESSIONS=1              # 单会话
    - SE_SESSION_REQUEST_TIMEOUT=3600     # 1小时等待验证码
    - SE_NODE_SESSION_TIMEOUT=7200        # 2小时实际超时
  volumes:
    - chrome-userdata:/tmp/chrome-user-data

volumes:
  chrome-userdata:
```

**原理**：

* volume 持久化 Chrome 用户数据（Cookie、LocalStorage）

* 单会话限制确保不竞争

* 超长超时支持等待短信验证码

* 容器重启后会话自动恢复

```java
public String detectPageType(WebDriver driver) {
    // 检查手机验证页：包含"手机验证"标题 + 手机号输入框
    if (!driver.findElements(By.xpath("//*[contains(text(), '手机验证')]")).isEmpty()
        && !driver.findElements(By.xpath("//input[contains(@placeholder, '手机号')]")).isEmpty()) {
        return "PHONE_VERIFY";
    }
    // 检查登录页：包含 password 输入框
    if (!driver.findElements(By.xpath("//input[@type='password']")).isEmpty()) {
        return "LOGIN";
    }
    return "UNKNOWN";
}
```

**验收标准**：`GET /api/browser/debug/page-type` 返回 `PHONE_VERIFY` 或 `LOGIN`

```java
public String captureCaptcha() {
    // 1. 查找验证码图片旁的算式文本
    WebElement captchaText = driver.findElement(
        By.xpath("//div[contains(@class, 'captcha')]//span[contains(@class, 'text')]")

    String expression = captchaText.getText(); // "9÷3=?"

    // 2. 解析算式
```

```java
public Map<String, Object> checkLoginStatus() {
    boolean hasToken = false;
    String tokenValue = null;

    // 1. 尝试从 Cookie 获取
    Cookie tokenCookie = driver.manage().getCookieNamed("token");
    if (tokenCookie != null) {
        hasToken = true;
        tokenValue = tokenCookie.getValue();
    }

    // 2. 尝试从 localStorage 获取
    if (!hasToken) {
        JavascriptExecutor js = (JavascriptExecutor) driver;
        tokenValue = (String) js.executeScript(
            "return localStorage.getItem('token') || sessionStorage.getItem('token');"
        );
        hasToken = tokenValue != null && !tokenValue.isEmpty();
    }

    // 3. 同步到 ZXRequestUtils
    if (hasToken) {
        ZXRequestUtils.setToken(tokenValue);
    }

    return Map.of("success", hasToken, "hasToken", hasToken, "tokenValue", tokenValue);
}
```

**验收标准**：Token 长度 > 0，同步成功

### Phase 4：AutoLoginService 编排（P1）

**目标**：实现 Task 6

**涉及文件**：`AutoLoginService.java`

**实现逻辑**：

```java
public void autoLogin() {
    // 1. 检测页面类型
    String pageType = browserApiController.getPageType();

    if ("PHONE_VERIFY".equals(pageType)) {
        // 流程一：手机验证页
        phaseOne();
    } else if ("LOGIN".equals(pageType)) {
        // 流程二：登录页
        phaseTwo();
    } else {
        throw new RuntimeException("无法识别页面类型: " + pageType);
    }

    // 2. 提取 Token
    Map<String, Object> result = browserApiController.checkLoginStatus();
    if (!(boolean) result.get("success")) {
        throw new RuntimeException("登录失败");
    }
}

private void phaseOne() {
    // 手机验证流程
    ensureLoginFrame();
    inputPhoneAndAgreements();
    sendCode();
    solveSlider();
    confirmSlider();
    inputSmsCode();
    submit();
}

private void phaseTwo() {
    // 登录页流程
    ensureLoginFrame();
    inputAccountAndPassword();
    solveMathCaptcha();
    checkAgreements();
    submit();
    solveSlider();
}
```
