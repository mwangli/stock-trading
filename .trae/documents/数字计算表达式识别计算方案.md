# 数字计算表达式识别计算方案设计

## 1. 背景与目标

### 1.1 现有方案分析

当前 `MathCaptchaService` 采用**百度 OCR + 图像处理双层策略**：

| 层级  | 方法                                       | 依赖        | 缺点               |
| --- | ---------------------------------------- | --------- | ---------------- |
| 第一层 | 百度 `general_basic` OCR                   | 网络（百度云）   | 依赖外部服务，词汇拼接可能出错  |
| 第二层 | 百度 `numbers` OCR + `detectOperator` 灰度检测 | 网络 + 图像算法 | 3位数场景运算符列位置判断不稳定 |

**当前存在的问题：**

1. **准确性不足**：OCR 识别率在图片质量不稳定时下降，表达式解析依赖正则匹配
2. **效率有优化空间**：每次调用都放大图片 + 两次 OCR 请求，网络开销较大
3. **容错机制薄弱**：第二层 fallback 依赖 `findOperatorColumn` 灰度密度分析，3位数场景判断不稳定
4. **无本地备用方案**：完全依赖百度 OCR，网络不可用时直接失败

### 1.2 设计目标

| 指标       | 当前      | 目标     |
| -------- | ------- | ------ |
| 单次识别成功率  | \~85%   | >95%   |
| 单次识别耗时   | \~800ms | <500ms |
| 网络依赖     | 强依赖     | 降级备用   |
| 表达式解析准确性 | 一般      | 高准确    |

### 2.2 优化策略

#### 策略 1：图片预处理优化（原始分辨率优先）

```java
/**
 * 图像预处理（原始分辨率优先策略）
 *
 * 原则：尽量使用原始分辨率处理，放大仅作为降级选项
 * 原因：放大虽增加像素，但会放大噪点和锯齿，反而降低识别准确率
 *
 * @param img  原始图片
 * @param scale 是否放大（默认false，OCR失败时启用）
 */
public ProcessedImage process(BufferedImage img, boolean scale) {
    // 1. 去噪（关键步骤）
    BufferedImage denoised = removeHorizontalNoiseLine(img);

    // 2. 二值化
    boolean[][] dark = binarize(denoised);

    // 3. 仅在需要时放大（降级选项）
    BufferedImage processed = scale ? scaleImage(denoised, 2) : denoised;

    return new ProcessedImage(denoised, dark, processed);
}
```

**优化原理**：

| 对比 | 当前方案（4x放大） | 优化方案（原始优先） |
|------|------------------|-------------------|
| 噪点处理 | 放大时同步放大干扰 | 先去噪再决定是否放大 |
| OCR准确率 | 较低（插值失真+噪点） | 更高（原始清晰度） |
| 计算量 | 4倍像素处理量 | 1倍，效率更高 |
| 降级策略 | 固定放大 | 原始失败→2x放大→本地分析 |

#### 策略 2：干扰线去除（利用已知特征）

```java
/**
 * 去除横向贯穿干扰线
 *
 * 干扰线特征（根据验证码图片分析）：
 * - 方向：横向（从左到右贯穿）
 * - 长度：占图片宽度的 3/4（约75%）
 * - 位置：图片中部或偏上
 * - 灰度：与数字相比较暗
 *
 * @param img 原始图片
 * @return 去噪后的图片
 */
private BufferedImage removeHorizontalNoiseLine(BufferedImage img) {
    int w = img.getWidth();
    int h = img.getHeight();
    boolean[][] dark = binarize(img);

    // 扫描每一行，检测贯穿暗线
    for (int y = 0; y < h; y++) {
        int consecutiveDark = 0;
        int darkStartX = -1;

        for (int x = 0; x < w; x++) {
            if (dark[x][y]) {
                if (consecutiveDark == 0) {
                    darkStartX = x;  // 记录暗线起始位置
                }
                consecutiveDark++;
            } else {
                // 检查之前的暗线是否符合干扰线特征
                if (consecutiveDark > 0) {
                    double ratio = (double) consecutiveDark / w;
                    // 干扰线特征：暗线长度占图片宽度 60%~90% 之间
                    if (ratio >= 0.60 && ratio <= 0.90) {
                        // 这是干扰线，清除该段
                        clearSegment(dark, y, darkStartX, consecutiveDark);
                    }
                }
                consecutiveDark = 0;
            }
        }
    }
    return darkToImage(dark, w, h);
}

/**
 * 清除指定行的暗像素段
 */
private void clearSegment(boolean[][] dark, int y, int startX, int length) {
    for (int x = startX; x < startX + length && x < dark.length; x++) {
        dark[x][y] = false;
    }
}
```

**干扰线处理效果预估**：

| 阶段 | 处理前 | 处理后 |
|------|--------|--------|
| 二值化 | 干扰线与数字混叠 | 干扰线被识别并清除 |
| OCR识别 | 可能误识别干扰线为"-"或数字 | 纯净的数字表达式 |
| 准确率 | ~85% | >95% |

#### 策略 3：表达式区域精确定位

```java
/**
 * 定位表达式区域（数字和运算符的边界）

    //    - 宽度 < 总宽度 15%：运算符或等号

}
```

**优化点：**

* 不依赖 OCR 文字检测，直接通过图像结构定位

* 避免 OCR 拼接错误

### 3.2 重构 `MathCaptchaService`

```java
@Slf4j
@Service
public class MathCaptchaService {

    private final ImagePreprocessor imagePreprocessor;
    private final DigitRecognizer digitRecognizer;
    private final OperatorDetector operatorDetector;

    /**
     * 获取验证码图片并识别计算结果
     *
     * @param account 资金账号/手机号
     * @return 识别结果（表达式、答案、置信度、识别方法）
     */
    public CaptchaRecognitionResult getCaptchaResult(String account) {
        // 1. 获取验证码图片
        BufferedImage rawImage = fetchCaptchaImage(account);

        // 2. 图片预处理
        ImagePreprocessor.ProcessedImage processed = imagePreprocessor.process(rawImage);

        // 3. 表达式区域定位
        ExpressionRegion region = imagePreprocessor.locateExpressionRegion(processed.getBinary());

        // 4. 分层识别
        return recognizeWithFallback(processed, region);
    }

    /**
     * 多层降级识别
     */
    private CaptchaRecognitionResult recognizeWithFallback(
            ImagePreprocessor.ProcessedImage processed,
            ExpressionRegion region) {

        // Level 1: 百度 OCR（优先）
        Result level1 = tryBaiduOcr(processed.getScaled());
        if (level1.isHighConfidence()) {
            return CaptchaRecognitionResult.success(level1, "baidu_ocr");
        }

        // Level 2: 图像分割 + 本地数字识别
        Result level2 = tryLocalDigitRecognition(processed, region);
        if (level2.isAcceptable()) {
            return CaptchaRecognitionResult.success(level2, "local_digit");
        }

        // Level 3: 纯图像分析（无网络时）
        Result level3 = tryPureImageAnalysis(processed, region);
        return CaptchaRecognitionResult.of(level3, "image_analysis");
    }
}
```

### 3.3 关键算法

#### 3.3.1 垂直投影分割

```java
/**
 * 计算垂直投影（每列暗像素占比）
 */
private int[] computeVerticalProjection(boolean[][] dark, int w, int h) {
    int[] projection = new int[w];
    for (int x = 0; x < w; x++) {
        int count = 0;
        for (int y = 0; y < h; y++) {
            if (dark[x][y]) count++;
        }
        projection[x] = count;
    }
    return projection;
}

/**
 * 寻找分割点（波谷位置 = 字符间隔）
 */
private List<Integer> findSplitPoints(int[] projection, int h) {
    List<Integer> points = new ArrayList<>();
    int threshold = h / 10; // 暗像素 < 10% 视为间隔

    boolean inContent = false;
    for (int x = 0; x < projection.length; x++) {
        boolean hasContent = projection[x] > threshold;
        if (!inContent && hasContent) {
            inContent = true; // 进入字符区
        } else if (inContent && !hasContent) {
            inContent = false; // 进入间隔
            points.add(x);     // 记录分割点
        }
    }
    return points;
}
```

#### 3.3.2 数字模板匹配

```java
/**
 * 本地数字识别（模板匹配）
 */
public int recognizeDigit(BufferedImage digitImg, Map<Integer, List<BufferedImage>> templates) {
    // 1. 预处理：统一尺寸、提取骨架
    BufferedImage normalized = normalizeSize(digitImg, TEMPLATE_WIDTH, TEMPLATE_HEIGHT);
    boolean[][] skeleton = extractSkeleton(normalized);

    // 2. 与每个数字的多个模板逐一匹配
    int bestMatch = -1;
    double bestScore = 0;

    for (Map.Entry<Integer, List<BufferedImage>> entry : templates.entrySet()) {
        int digit = entry.getKey();
        for (BufferedImage template : entry.getValue()) {
            double score = compareSkeleton(skeleton, template);
            if (score > bestScore) {
                bestScore = score;
                bestMatch = digit;
            }
        }
    }

    // 3. 置信度检查
    if (bestScore < MIN_MATCH_SCORE) {
        return -1; // 识别失败
    }
    return bestMatch;
}
```

#### 6.4.2 自动化验证接口

```java
/**
 * 验证码识别验证接口
 * 用于测试环境批量验证识别准确率
 */
@GetMapping("/debug/verifyCaptcha")
public ResponseDTO<CaptchaVerificationResult> verifyCaptcha(
        @RequestParam(defaultValue = "10") int count) {

    int successCount = 0;
    List<CaptchaRecognitionResult> results = new ArrayList<>();

    for (int i = 0; i < count; i++) {
        BufferedImage captchaImage = fetchCaptchaImage(account);
        CaptchaRecognitionResult result = mathCaptchaService.getCaptchaResult(account);
        boolean isCorrect = verifyByFormSubmission(result.getAnswer());
        result.setVerifiedCorrect(isCorrect);

        if (isCorrect) successCount++;
        results.add(result);
        Thread.sleep(1000);
    }

    CaptchaVerificationResult summary = new CaptchaVerificationResult();
    summary.setTotalCount(count);
    summary.setSuccessCount(successCount);
    summary.setSuccessRate((double) successCount / count);
    summary.setDetails(results);

    return ResponseDTO.success(summary);
}
```
