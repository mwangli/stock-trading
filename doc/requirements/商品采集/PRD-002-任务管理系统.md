## 1. 需求概述

### 1.1 背景
当前系统已有基础的手动采集任务功能，使用Spring Scheduler实现定时调度，但功能较为简单，缺乏统一的任务管理界面。需要增强任务管理能力，支持：
- 手动触发任务执行（通过API接口）
- 定时任务调度（通过Spring Scheduler）
- 任务执行日志和监控
- 前端任务管理界面

### 1.2 目标
构建任务管理模块，提供统一的商品采集任务管理能力，支持：
- 手动触发任务执行（通过REST API接口）
- 定时任务调度（通过Spring Scheduler）
- 任务执行日志和监控
- 前端任务管理界面

### 1.3 范围
- 后端：增强现有Scheduler调度能力，新增任务管理API
- 前端：创建任务管理页面
- 保留现有采集任务相关实体和Service

## 2. 功能需求

### 2.1 功能列表

| 功能 | 描述 | 优先级 |
|------|------|--------|
| XXL-Job集成 | 将商品采集任务注册为XXL-Job JobHandler | 必须 |
| 手动执行 | 通过XXL-Job管理界面手动触发采集任务 | 必须 |
| 定时执行 | 配置Cron表达式实现定时采集 | 必须 |
| 任务监控 | 查看任务执行日志、状态、耗时等 | 应该 |
| 前端管理页面 | 提供任务管理前端界面 | 应该 |

### 2.2 用户故事

| 角色 | 故事 | 验收条件 |

## 3. 非功能需求

### 3.1 性能要求
- 任务调度延迟 < 1秒
- 支持高并发任务执行（通过XXL-Job分布式能力）

### 3.2 可用性要求
- 任务执行失败可重试
- 执行日志保留30天

### 3.3 依赖要求
- XXL-Job Admin（调度中心）需独立部署
- 本系统作为Executor注册到调度中心

## 4. 技术方案

### 4.1 架构设计
```
┌─────────────────────────────────────────────────────────────┐
│                    任务管理模块                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                    前端层                            │   │
│  │         任务管理页面 (React + Ant Design)            │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ HTTP API
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    后端层 (Spring Boot)                      │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              CollectionTaskController               │   │
│  │    (任务管理API: 创建/执行/查询/启停)                 │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              CollectionTaskService                  │   │
│  │    (任务业务逻辑: 状态管理/日志记录)                   │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              CollectionScheduler                    │   │
│  │    (Spring Scheduler: 定时检查与执行)                 │   │
│  └─────────────────────────────────────────────────────┘   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │           ProductCollectorService                   │   │
│  │           (采集执行器: 实际采集逻辑)                   │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 触发方式

| 触发方式 | 实现方式 | 说明 |
|----------|----------|------|
| 手动触发 | REST API接口 | POST /api/collection-tasks/{id}/execute |
| 定时触发 | Spring Scheduler | 每分钟检查待执行任务，支持Cron表达式 |

### 4.3 配置说明

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| scheduler.enabled | 是否启用定时调度 | true |
| scheduler.check-interval | 任务检查间隔 | 60000 (ms) |
| collection.task.default-max | 默认最大采集数 | 100 |
| collection.log.retention-days | 日志保留天数 | 30 |
