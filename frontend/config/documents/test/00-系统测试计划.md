## 1. 测试策略

### 1.1 测试层次

```
┌─────────────────────────────────────┐
│         验收测试                     │
│   (用户需求验证)                     │
└─────────────────┬───────────────────┘
                  │
┌─────────────────▼─────────────────┐
│         系统测试                   │
│   (端到端流程验证)                 │
└─────────────────┬─────────────────┘
                  │
┌─────────────────▼─────────────────┐
│         集成测试                   │
│   (模块间接口验证)                 │
└─────────────────┬─────────────────┘
                  │
┌─────────────────▼─────────────────┐
│         单元测试                   │
│   (函数/方法级验证)                │
└───────────────────────────────────┘
```

### 1.2 测试类型

| 测试类型 | 说明 | 执行时机 |
|----------|------|----------|
| 单元测试 | 测试单个函数/方法 | 开发阶段 |
| 集成测试 | 测试模块间接口 | 开发完成后 |
| 系统测试 | 测试完整业务流程 | 集成测试通过后 |
| 性能测试 | 测试系统性能指标 | 系统稳定后 |
| 验收测试 | 验证用户需求 | 交付前 |

## 4. 系统测试

### 4.1 端到端流程测试

**测试场景 1: 完整选股流程**
```
1. 数据采集 (股票列表、K 线、新闻)
2. AI 预测 (LSTM、情感分析)
3. 策略计算 (选股、排名)
4. 信号生成 (买入/卖出)
5. 风控检查
6. 交易执行
7. 持仓更新
```

**测试场景 2: 定时任务流程**
```
1. 01:00 股票列表同步
2. 09:00-15:00 实时数据采集
3. 15:30 情感分析
4. 16:00 LSTM 预测
5. 17:00 综合选股
6. 17:30 信号生成
```

### 4.2 性能测试

**性能指标:**

| 指标 | 要求 | 测试方法 |
|------|------|----------|
| 数据采集延迟 | < 1 小时 | 收盘后计时 |
| AI 预测完成时间 | < 2 小时 | 全市场预测 |
| 策略计算时间 | < 10 分钟 | 选股计算 |
| API 响应时间 | < 500ms | 95% 请求 |
| 并发支持 | 10 个请求 | 并发测试 |

**测试工具:**
- JMeter (API 性能)
- Gatling (负载测试)
